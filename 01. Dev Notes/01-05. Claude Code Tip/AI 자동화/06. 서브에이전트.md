Claude Code에서 플랜→코딩→리뷰→테스트→보안점검을 각각 담당하는 서브에이전트 시스템을 구성하는 방법을 설명할게요.

---

## 전체 아키텍처

```
오케스트레이터 (메인 Claude Code)
├── 1. 🗺️  Plan Agent      - 요구사항 분석 & 설계
├── 2. 💻  Code Agent      - 코드 구현
├── 3. 🔍  Review Agent    - 코드 리뷰
├── 4. 🎭  Test Agent      - E2E + 유닛 테스트
└── 5. 🔒  Security Agent  - 보안 취약점 점검
```

## 1. CLAUDE.md 오케스트레이터 정의

```markdown
# CLAUDE.md

## 멀티 서브에이전트 워크플로우

새 기능 요청이 들어오면 반드시 아래 순서로 서브에이전트를 실행:

### 실행 순서 (순차적)
1. `bash scripts/agents/plan-agent.sh "$TASK"`
2. `bash scripts/agents/code-agent.sh`
3. `bash scripts/agents/review-agent.sh`
4. `bash scripts/agents/test-agent.sh`
5. `bash scripts/agents/security-agent.sh`

### 규칙
- 각 에이전트는 이전 에이전트의 결과를 읽고 시작
- 어느 단계든 FAIL이면 해당 단계에서 중단 후 사용자 보고
- 모든 결과는 `agent-results/` 폴더에 저장
- 5단계 모두 PASS시에만 PR 생성
```

## 2. 공통 유틸리티

```bash
# scripts/agents/common.sh
RESULTS_DIR="agent-results"
mkdir -p $RESULTS_DIR

log_result() {
  local AGENT=$1
  local STATUS=$2
  local MESSAGE=$3
  
  echo "{
    \"agent\": \"$AGENT\",
    \"status\": \"$STATUS\",
    \"message\": \"$MESSAGE\",
    \"timestamp\": \"$(date -u +%Y-%m-%dT%H:%M:%SZ)\"
  }" > "$RESULTS_DIR/$AGENT.json"
  
  if [ "$STATUS" = "FAIL" ]; then
    echo "❌ [$AGENT] 실패: $MESSAGE"
    exit 1
  else
    echo "✅ [$AGENT] 통과"
  fi
}

read_result() {
  cat "$RESULTS_DIR/$1.json"
}
```

# scripts/agents/code-agent.sh

# 플랜 읽기
PLAN=$(cat "$RESULTS_DIR/plan.json")

플랜: $PLAN

- 에러 처리 필수

if [ $? -ne 0 ]; then
  log_result "code-agent" "FAIL" "TypeScript 에러 발생"
fi

# 변경된 파일 목록 저장
git diff --name-only > "$RESULTS_DIR/changed-files.txt"

log_result "code-agent" "PASS" "코드 구현 완료"

### 🔍 Review Agent

```bash
# scripts/agents/review-agent.sh
source scripts/agents/common.sh

echo "🔍 Review Agent 시작..."

CHANGED_FILES=$(cat "$RESULTS_DIR/changed-files.txt")

# 변경된 파일의 diff 추출
DIFF=$(git diff HEAD~1 -- $CHANGED_FILES)

REVIEW=$(claude --print "
당신은 엄격한 시니어 코드 리뷰어입니다.
아래 코드 변경사항을 리뷰하고 JSON으로 결과를 반환하세요.

변경사항:
$DIFF

리뷰 기준:
1. 코드 가독성
2. 성능 문제 (N+1 쿼리, 불필요한 루프 등)
3. 중복 코드
4. 단일 책임 원칙 위반
5. 에러 처리 누락

응답 형식:
{
  \"verdict\": \"PASS|FAIL\",
  \"score\": 0-100,
  \"critical_issues\": [],
  \"warnings\": [],
  \"suggestions\": []
}
")

echo "$REVIEW" > "$RESULTS_DIR/review.json"

VERDICT=$(echo $REVIEW | node -e "
  const d=require('fs').readFileSync('/dev/stdin','utf8');
  const r=JSON.parse(d);
  console.log(r.verdict);
")

if [ "$VERDICT" = "FAIL" ]; then
  log_result "review-agent" "FAIL" "코드 리뷰 실패 - critical issues 발견"
fi

log_result "review-agent" "PASS" "코드 리뷰 통과"
```

### 🎭 Test Agent

```bash
# scripts/agents/test-agent.sh
source scripts/agents/common.sh

echo "🎭 Test Agent 시작..."

CHANGED_FILES=$(cat "$RESULTS_DIR/changed-files.txt")

# 변경된 파일에 대한 테스트 자동 생성
claude --print "
변경된 파일들에 대한 유닛 테스트를 생성하세요:
$CHANGED_FILES

- Jest + Testing Library 사용
- 각 함수당 최소 3개 케이스 (정상/경계값/에러)
- 테스트 파일은 __tests__/ 폴더에 생성
"

# 유닛 테스트 실행
echo "--- 유닛 테스트 ---"
npm run test -- --coverage --json \
  --outputFile="$RESULTS_DIR/unit-test.json"
UNIT_EXIT=$?

# E2E 테스트 실행
echo "--- E2E 테스트 ---"
npx playwright test --reporter=json \
  --output="$RESULTS_DIR/e2e-test.json"
E2E_EXIT=$?

# 커버리지 체크
COVERAGE=$(node -e "
  const r = require('./$RESULTS_DIR/unit-test.json');
  console.log(r.coverageMap ? 
    Object.values(r.coverageMap)
      .reduce((a,f) => a + f.s.covered/f.s.total, 0) / 
    Object.keys(r.coverageMap).length * 100 
    : 0
  );
")

echo "커버리지: $COVERAGE%"

if [ $UNIT_EXIT -ne 0 ] || [ $E2E_EXIT -ne 0 ]; then
  log_result "test-agent" "FAIL" "테스트 실패"
fi

if (( $(echo "$COVERAGE < 80" | bc -l) )); then
  log_result "test-agent" "FAIL" "커버리지 80% 미달: $COVERAGE%"
fi

log_result "test-agent" "PASS" "테스트 통과 (커버리지: $COVERAGE%)"
```

### 🔒 Security Agent

```bash
# scripts/agents/security-agent.sh
source scripts/agents/common.sh

echo "🔒 Security Agent 시작..."

CHANGED_FILES=$(cat "$RESULTS_DIR/changed-files.txt")
DIFF=$(git diff HEAD~1 -- $CHANGED_FILES)

# 1. npm audit
echo "--- 의존성 취약점 검사 ---"
npm audit --json > "$RESULTS_DIR/npm-audit.json" 2>&1
AUDIT_EXIT=$?

# 2. Claude로 보안 취약점 분석
SECURITY_REVIEW=$(claude --print "
당신은 보안 전문가입니다.
아래 코드를 OWASP Top 10 기준으로 분석하세요.

코드 변경사항:
$DIFF

점검 항목:
1. SQL Injection
2. XSS (Cross-Site Scripting)
3. 인증/인가 취약점
4. 민감 정보 노출 (API 키, 비밀번호 하드코딩)
5. CSRF
6. 안전하지 않은 직접 객체 참조
7. 보안 설정 오류

응답 형식:
{
  \"verdict\": \"PASS|FAIL\",
  \"critical\": [],
  \"high\": [],
  \"medium\": [],
  \"low\": [],
  \"recommendations\": []
}
")

echo "$SECURITY_REVIEW" > "$RESULTS_DIR/security.json"

# 3. 하드코딩된 시크릿 검사
echo "--- 시크릿 스캔 ---"
SECRETS=$(grep -rn \
  -E "(password|secret|api_key|token)\s*=\s*['\"][^'\"]{8,}" \
  src/ --include="*.ts" --include="*.js" || true)

if [ -n "$SECRETS" ]; then
  echo "$SECRETS" >> "$RESULTS_DIR/security.json"
  log_result "security-agent" "FAIL" "하드코딩된 시크릿 발견"
fi

VERDICT=$(echo $SECURITY_REVIEW | node -e "
  const d=require('fs').readFileSync('/dev/stdin','utf8');
  console.log(JSON.parse(d).verdict);
")

if [ "$VERDICT" = "FAIL" ] || [ $AUDIT_EXIT -ne 0 ]; then
  log_result "security-agent" "FAIL" "보안 취약점 발견"
fi

log_result "security-agent" "PASS" "보안 점검 통과"
```

## 4. 오케스트레이터 메인 실행 파일

```bash
#!/bin/bash
# scripts/run-pipeline.sh
# 전체 파이프라인 실행

TASK="$1"
RESULTS_DIR="agent-results"
mkdir -p $RESULTS_DIR

print_header() {
  echo ""
  echo "================================================"
  echo " $1"
  echo "================================================"
}

print_header "🚀 멀티 에이전트 파이프라인 시작"
echo "태스크: $TASK"
echo "시작: $(date)"

# ---- 순차 실행 ----

print_header "1/5 🗺️  Plan Agent"
bash scripts/agents/plan-agent.sh "$TASK" || exit 1

print_header "2/5 💻  Code Agent"
bash scripts/agents/code-agent.sh || exit 1

print_header "3/5 🔍  Review Agent"
bash scripts/agents/review-agent.sh || exit 1

print_header "4/5 🎭  Test Agent"
bash scripts/agents/test-agent.sh || exit 1

print_header "5/5 🔒  Security Agent"
bash scripts/agents/security-agent.sh || exit 1

# ---- 최종 리포트 ----

print_header "📊 최종 결과"
node scripts/generate-report.js

echo ""
echo "✅ 모든 에이전트 통과 - PR 생성 시작"
bash scripts/create-pr.sh
```

## 5. 최종 리포트 생성

```javascript
// scripts/generate-report.js
const fs = require('fs');
const path = require('path');

const agents = [
  'plan-agent', 'code-agent', 
  'review-agent', 'test-agent', 'security-agent'
];

const results = agents.map(agent => {
  try {
    return JSON.parse(
      fs.readFileSync(`agent-results/${agent}.json`, 'utf-8')
    );
  } catch {
    return { agent, status: 'NOT_RUN' };
  }
});

const report = `
# 🤖 멀티 에이전트 파이프라인 결과

| 에이전트 | 상태 | 메시지 |
|---------|------|--------|
${results.map(r => 
  `| ${r.agent} | ${r.status === 'PASS' ? '✅' : '❌'} | ${r.message || '-'} |`
).join('\n')}

## 상세 결과
${results.map(r => `### ${r.agent}\n\`\`\`json\n${JSON.stringify(r, null, 2)}\n\`\`\``).join('\n\n')}

생성: ${new Date().toISOString()}
`;

fs.writeFileSync('agent-results/final-report.md', report);
console.log(report);
```

## 전체 흐름 요약

```
태스크 입력
    ↓
🗺️  Plan Agent   → plan.json (설계 문서)
    ↓
💻  Code Agent   → 코드 구현 + 린터
    ↓
🔍  Review Agent → 코드 품질 점검
    ↓
🎭  Test Agent   → 유닛 + E2E + 커버리지
    ↓
🔒  Security Agent → OWASP + 시크릿 스캔
    ↓
📊  최종 리포트 → PR 자동 생성 ✅
```

각 단계에서 실패하면 즉시 중단되고 사용자에게 상세 리포트를 제공합니다. 추가로 궁금한 부분 있으면 말씀해주세요!
