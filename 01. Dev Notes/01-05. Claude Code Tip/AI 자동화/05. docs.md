Claude Code에서 `docs/` 폴더에 명세를 작성하고 필요할 때만 참조하는 Lazy Loading 방식을 구성하는 방법을 설명할게요.

---

## 핵심 개념

```
❌ Eager Loading  - CLAUDE.md에 모든 문서를 한번에 로드
                   → 컨텍스트 낭비, 토큰 초과 위험

✅ Lazy Loading   - 필요한 시점에 필요한 문서만 참조
                   → 컨텍스트 효율적 사용
```

---

## 3. CLAUDE.md - Lazy Loading 규칙 정의

```markdown
# CLAUDE.md

## 📚 문서 참조 규칙 (Lazy Loading)

### 기본 원칙
- 항상 `docs/index.md`만 먼저 읽을 것
- 작업 관련 문서만 그때그때 참조
- 불필요한 문서는 절대 읽지 말 것

### 자동 참조 트리거

#### 트리거 → 참조 문서 매핑
```

"DB 수정" 언급 → docs/architecture/database.md 읽기 "API 추가" 언급 → docs/architecture/api.md 읽기 "로그인/인증" 언급 → docs/features/auth.md 읽기 "결제/환불" 언급 → docs/features/payment.md 읽기 "테스트 작성" 시작 → docs/conventions/testing.md 읽기 "보안 점검" 시작 → docs/conventions/security.md 읽기 Plan Agent 실행 → docs/agents/plan-agent.md 읽기 Code Agent 실행 → docs/agents/code-agent.md 읽기

```

### 참조 순서
1. `docs/index.md` 확인 (목차)
2. 작업 관련 문서 경로 확인
3. 해당 문서만 읽기
4. 작업 완료 후 다음 문서로

### 금지 사항
- docs/ 전체 폴더 한번에 읽기 금지
- 관련 없는 문서 미리 읽기 금지
```

## 4. 에이전트 명세 문서 예시

````markdown
# docs/agents/plan-agent.md

## Plan Agent 명세

### 역할
요구사항을 분석하고 구현 계획 수립

### 시작 전 참조 문서
- `docs/architecture/overview.md`  ← 반드시 읽기
- `docs/architecture/database.md`  ← DB 관련 시에만

### 수행 작업
1. 태스크 분석
2. 영향받는 파일 목록 작성
3. 구현 순서 결정
4. 리스크 식별

`agent-results/plan.json` 저장
```json

  "files_to_modify": [],
  "files_to_create": [],
  "risks": [],
  "docs_needed": [        ← 다음 에이전트가 참조할 문서 목록
    "docs/features/payment.md"
  ]
}
````

### 완료 조건

- [ ] plan.json 생성
- [ ] docs_needed 목록 작성
- [ ] TODO.md 업데이트

````

## 5. Lazy Loading 실행 스크립트

```bash
#!/bin/bash
# scripts/doc-loader.sh
# 필요한 문서만 동적으로 로드

CONTEXT=$1  # 현재 작업 컨텍스트

# 컨텍스트 기반 문서 자동 선택
select_docs() {
  local DOCS=()
  
  # 항상 포함
  DOCS+=("docs/index.md")

  # 키워드 기반 추가
  case $CONTEXT in
    *"auth"*|*"login"*|*"인증"*)
      DOCS+=("docs/features/auth.md")
      ;;
    *"payment"*|*"결제"*|*"환불"*)
      DOCS+=("docs/features/payment.md")
      ;;
    *"database"*|*"DB"*|*"스키마"*)
      DOCS+=("docs/architecture/database.md")
      ;;
    *"api"*|*"API"*|*"endpoint"*)
      DOCS+=("docs/architecture/api.md")
      ;;
    *"test"*|*"테스트"*)
      DOCS+=("docs/conventions/testing.md")
      ;;
    *"security"*|*"보안"*)
      DOCS+=("docs/conventions/security.md")
      ;;
  esac

  echo "${DOCS[@]}"
}

# 선택된 문서만 출력
SELECTED=$(select_docs)
echo "📚 로드할 문서:"

for DOC in $SELECTED; do
  echo "  - $DOC"
  echo "=== $DOC ===" 
  cat "$DOC"
  echo ""
done
````

## 6. plan.json의 docs_needed로 연쇄 로딩

```javascript
// scripts/lazy-doc-resolver.js
// Plan Agent 결과에서 다음 단계 필요 문서 자동 결정

const fs = require('fs');

const plan = JSON.parse(
  fs.readFileSync('agent-results/plan.json', 'utf-8')
);

// plan.json의 docs_needed를 읽어 다음 에이전트에 전달
const docsNeeded = plan.docs_needed || [];

console.log('📚 다음 단계 필요 문서:');
docsNeeded.forEach(doc => {
  console.log(`  - ${doc}`);
  const content = fs.readFileSync(doc, 'utf-8');
  // 각 에이전트 컨텍스트에 추가
  fs.appendFileSync(
    'agent-results/context.md',
    `\n\n## ${doc}\n${content}`
  );
});
```

## 7. 실제 사용 흐름

```bash
# Claude Code에 요청
"결제 환불 기능 추가해줘"

# Claude Code 내부 동작:
# 1. docs/index.md 읽기 (항상)
# 2. "결제" 키워드 감지
# 3. docs/features/payment.md만 읽기
# 4. Plan Agent 시작 → docs/agents/plan-agent.md 읽기
# 5. plan.json의 docs_needed 확인
# 6. 필요한 문서만 추가 로드
```

---

## 전체 흐름 요약

```
태스크 입력
    ↓
docs/index.md 읽기 (목차만)
    ↓
키워드 분석 → 관련 docs만 선택
    ↓
Plan Agent → docs/agents/plan-agent.md
    ↓ plan.json (docs_needed 포함)
Code Agent → docs/agents/code-agent.md
           + plan.json의 docs_needed
    ↓
Review Agent → docs/agents/review-agent.md
    ↓
Test Agent  → docs/agents/test-agent.md
            + docs/conventions/testing.md
    ↓
Security Agent → docs/agents/security-agent.md
              + docs/conventions/security.md
    ↓
PR 생성 ✅
```

## 핵심 요약

|방법|역할|
|---|---|
|`docs/index.md`|항상 로드되는 목차|
|`CLAUDE.md` 트리거 규칙|키워드 → 문서 자동 매핑|
|`plan.json docs_needed`|에이전트간 필요 문서 전달|
|`doc-loader.sh`|동적 문서 선택 스크립트|

**`docs/index.md`를 목차 전용으로 유지하고, CLAUDE.md에 트리거 규칙을 명시**하는 것이 Lazy Loading의 핵심입니다.
