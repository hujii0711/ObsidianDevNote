Claude Code에서 진척 사항을 체크리스트/Todo로 관리하는 방법을 설명할게요.

---

## 핵심 방법 3가지

```
1. CLAUDE.md Todo 섹션     - 프로젝트 전체 진척
2. TODO.md 전용 파일       - 에이전트간 상태 공유
3. claude --todo 플래그    - Claude Code 내장 기능
```

---

## 1. Claude Code 내장 Todo 기능

```bash
# Claude Code는 작업 시작 시 자동으로 Todo 생성
# 대화 중에 직접 요청

"결제 모듈 구현하면서 진척 사항을 
 Todo로 추적해줘"
```

Claude Code가 자동으로 내부 Todo를 생성하고 각 단계마다 업데이트합니다. 대화창에서 실시간으로 확인 가능합니다.

## 3. CLAUDE.md에 Todo 규칙 정의

```markdown
# CLAUDE.md

## Todo 관리 규칙

### 작업 시작 시
1. TODO.md에 태스크 목록 작성
2. 각 항목은 `- [ ]` 형식 사용
3. 완료 시 `- [x]`로 즉시 업데이트

### 업데이트 시점
- 각 서브에이전트 시작 전: 상태를 "진행 중..."으로
- 완료 후: [x]로 체크 + 완료 시간 기록
- 실패 시: [!]로 표시 + 실패 이유 기록

### 형식
- [ ] 대기
- [>] 진행 중
- [x] 완료
- [!] 실패
- [-] 스킵
```

## 4. Todo 자동 업데이트 스크립트

```bash
# scripts/todo-manager.sh

ACTION=$1   # check / uncheck / progress / fail
ITEM=$2     # 업데이트할 항목 텍스트
TODO_FILE="TODO.md"

update_todo() {
  local FROM=$1
  local TO=$2
  local ITEM=$3
  sed -i "s/- $FROM $ITEM/- $TO $ITEM/" $TODO_FILE
}

case $ACTION in
  "start")
    update_todo "[ ]" "[>]" "$ITEM"
    echo "🔄 시작: $ITEM"
    ;;
  "done")
    TIME=$(date +%H:%M)
    update_todo "[>]" "[x]" "$ITEM"
    sed -i "s/- \[x\] $ITEM/- [x] $ITEM ($TIME)/" $TODO_FILE
    echo "✅ 완료: $ITEM"
    ;;
  "fail")
    update_todo "[>]" "[!]" "$ITEM"
    echo "❌ 실패: $ITEM"
    ;;
esac

# 진척률 계산 및 출력
TOTAL=$(grep -c "\- \[" $TODO_FILE)
DONE=$(grep -c "\- \[x\]" $TODO_FILE)
echo "📊 진척률: $DONE/$TOTAL ($(( DONE * 100 / TOTAL ))%)"
```

## 5. 파이프라인에 Todo 통합

```bash
# scripts/run-pipeline.sh 수정

init_todo() {
  cat > TODO.md << 'EOF'
# 파이프라인 진척 현황

## 에이전트 상태
- [ ] 🗺️  Plan Agent
- [ ] 💻  Code Agent  
- [ ] 🔍  Review Agent
- [ ] 🎭  Test Agent
- [ ] 🔒  Security Agent

## 세부 작업
- [ ] 플랜 생성
- [ ] 파일 구현
- [ ] 린터 통과
- [ ] 리뷰 통과
- [ ] 테스트 통과
- [ ] 보안 통과
- [ ] PR 생성
EOF
  echo "📋 TODO.md 초기화 완료"
}

# 파이프라인 실행
init_todo

bash scripts/todo-manager.sh "start" "🗺️  Plan Agent"
bash scripts/agents/plan-agent.sh "$TASK" \
  && bash scripts/todo-manager.sh "done" "🗺️  Plan Agent" \
  || bash scripts/todo-manager.sh "fail" "🗺️  Plan Agent"

bash scripts/todo-manager.sh "start" "💻  Code Agent"
bash scripts/agents/code-agent.sh \
  && bash scripts/todo-manager.sh "done" "💻  Code Agent" \
  || bash scripts/todo-manager.sh "fail" "💻  Code Agent"

# ... 반복
```

## 6. 실시간 진척 확인

```bash
# 터미널에서 실시간 모니터링
watch -n 2 'cat TODO.md'

# 또는 진척률만 확인
node -e "
const fs = require('fs');
const todo = fs.readFileSync('TODO.md', 'utf-8');
const total = (todo.match(/- \[/g) || []).length;
const done  = (todo.match(/- \[x\]/g) || []).length;
const fail  = (todo.match(/- \[!\]/g) || []).length;
console.log('✅ 완료:', done);
console.log('❌ 실패:', fail);
console.log('⏳ 대기:', total - done - fail);
console.log('📊 진척:', Math.round(done/total*100) + '%');
"
```

## 7. GitHub Issue와 연동

```bash
# Todo 항목을 GitHub Issue 체크리스트로 자동 동기화
gh issue create \
  --title "결제 환불 기능 구현" \
  --body "$(cat TODO.md)"

# PR 본문에도 자동 포함
gh pr create \
  --title "feat: 결제 환불 기능" \
  --body "## 진척 현황
$(cat TODO.md)

Closes #$ISSUE_NUM"
```

---

## 전체 흐름

```
태스크 시작
    ↓
TODO.md 자동 생성 (모두 [ ])
    ↓
각 에이전트 실행 시 → [>] 진행 중
    ↓
완료 시 → [x] + 시간 기록
실패 시 → [!] + 중단 보고
    ↓
PR 본문에 TODO.md 포함 ✅
```

## 핵심 요약

|방법|용도|
|---|---|
|Claude Code 내장 Todo|대화 중 실시간 추적|
|`TODO.md` 파일|에이전트 간 상태 공유|
|`todo-manager.sh`|자동 업데이트 스크립트|
|GitHub Issue 연동|팀과 진척 공유|

**가장 중요한 것은 CLAUDE.md에 "항상 TODO.md를 업데이트하라"고 명시**하는 것입니다. Claude Code가 이 규칙을 따라 자동으로 진척을 추적합니다.
