### 플러그인이란?

플러그인은 Claude Code를 커스텀 기능으로 확장할 수 있는 단위로, 프로젝트와 팀 전체에서 공유 가능합니다. 스킬(Skills), 에이전트(Agents), 훅(Hooks), MCP 서버 등을 묶어서 배포할 수 있습니다.

### 플러그인의 구조

공식 플러그인의 디렉토리 구조는 다음과 같습니다:

```
plugin-name/
├── .claude-plugin/
│   └── plugin.json     # 플러그인 메타데이터 (필수)
├── .mcp.json           # MCP 서버 설정 (선택)
├── commands/           # 슬래시 커맨드 (선택)
├── agents/             # 에이전트 정의 (선택)
├── skills/             # 스킬 정의 (선택)
└── README.md           # 문서
```

### 플러그인에 포함될 수 있는 구성 요소

플러그인은 다음을 포함할 수 있습니다: 슬래시 커맨드, 특화된 에이전트, 훅(Hook), MCP 서버, LSP 서버, 백그라운드 모니터 등.

### 설치 및 마켓플레이스

Anthropic의 공식 페이지(`claude.com/plugins`)에서 다양한 플러그인을 브라우징하고 원클릭으로 설치할 수 있습니다. Figma, Vercel, Jira, Slack, Supabase 등 수많은 서비스와의 통합 플러그인이 있습니다.

## 스킬(Skill)이란?

스킬은 **에이전트에게 특정 작업을 가르치는 재사용 가능한 명령어 세트**입니다. 단일 커맨드로 설치 가능합니다. 반면 플러그인은 여러 스킬, MCP 서버, 커맨드를 하나로 묶은 번들이고, 마켓플레이스는 여러 플러그인을 하나의 레지스트리로 배포하는 GitHub 저장소입니다.

스킬은 `SKILL.md` 파일 하나로 구성되며, Claude가 **자동으로 컨텍스트를 파악해서 사용**하는 방식입니다. 스킬은 모델이 호출하는 방식(model-invoked)으로, Claude가 작업 컨텍스트에 따라 자동으로 사용합니다. 슬래시 커맨드처럼 명시적으로 호출하는 게 아닙니다.

---

## 플러그인 vs 스킬 비교

|구분|스킬(Skill)|플러그인(Plugin)|
|---|---|---|
|**구성**|`SKILL.md` 파일 하나|여러 구성요소의 번들|
|**호출 방식**|Claude가 컨텍스트 파악 후 자동 사용|슬래시 커맨드, 에이전트 등 다양|
|**범위**|단일 작업/행동 지침|복합 기능 (커맨드+스킬+MCP 등)|
|**공유**|단독으로도 배포 가능|팀/프로젝트 단위 배포에 최적화|
|**외부 연동**|없음|MCP 서버를 통해 외부 서비스 연동 가능|

---

### 한 줄 요약

> **스킬** = Claude에게 "어떻게 행동할지" 가르치는 지침서  
> **플러그인** = 스킬 + 커맨드 + MCP + 에이전트를 하나로 묶은 확장 패키지

실제로 플러그인 안에 스킬이 포함되는 경우가 많고, 간단한 행동 지침만 필요하면 스킬 단독으로, 외부 서비스 연동이나 복잡한 워크플로우가 필요하면 플러그인으로 구성하는 것이 일반적입니다.
