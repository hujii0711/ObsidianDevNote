#### 1. 워크스페이스에 생성되는 파일
온보딩이 끝나면 ~/.openclaw/ 폴더 안에 생성되는 주요 파일

| 파일/폴더                 | 역할                        |
| --------------------- | ------------------------- |
| openclaw.json         | 오픈클로의 모든 설정이 담긴 핵심 파일     |
| openclaw.db           | 대화 기록이 저장되는 SQLite 데이터베이스 |
| .env                  | 발급 받은 API 키 등 환경변수 저장     |
| memory/               | 에이전트의 기억(일일 로그, 장기 기억)    |
| workspace/            | 에이전트의 작업 디렉터리             |
| workspace/SOUL.md     | 에이전트의 행동 철학과 규칙           |
| workspace/IDENTITY.md | 에이전트의 이름. 말투. 페르소나        |
| workspace/USER.md     | 사용자 정보(이름, 타임존, 선호 등)     |

#### 2. DM 접근 정책

| 모드         | 설명                  |
| ---------- | ------------------- |
| paring(기본) | 코드 승인 필요. 가장 안전     |
| allowlist  | 미리 허용한 사용자만 대화 가능   |
| open       | 누구나 대화 가능. 공개 봇에 적합 |
| disabled   | DM 비활성화             |

#### 3. 안전한 셋팅
- 도구 확인 프롬프트: 사용자 승인 거친후 작업 진행
- 샌드박스 기본 활성화: 에이전트가 셀 명령어를 실행할 때 시스템 전체가 아니라 격리된 공간에서 실행합니다. 도커 컨테이너를 사용하며, 실수로 중요한 파일을 건드리는 것을 막는다.
- DM 접근 제어: 텔레그램을 연결할 때 페어링 모드가 여기에 해당한다. 승인 코드 없이는 아무도 에이전트에게 말을 걸 수가 없다.
- 도구 프로필 기본값: minimal 프로필로 시작하면 에이전트가 할 수 있는 일이 극히 제한되어 실수가 줄어든다.(행동 범위 규제)
- ~/.openclaw/workspace/SOUL.md에 금지 사항 섹션 추가
- 에이전트 전용 이메일 계정 생성
- 기본 브라우저 프로필을 openclaw로 설정
- 워크스페이스 폴더에 중요한 개인 파일이 들어 있지 않은지 확인
- 워크스페이스 폴더 안에서만 작업하도록 설정(다른 폴더의 파일 건드리지 못하게)


#### 4. 도구 프로필

| 프로필       | 사용 가능한 도구                            | 언제 쓰나                 | 이 질문에서 일어 나는 일                         |
| --------- | ------------------------------------ | --------------------- | -------------------------------------- |
| minimal   | 세션 상태 확인, 메모리 읽기(쉽게 말해 없음)           | 처음 설치 직후(기본값)         | 학습 데이터에서만 답변, 최신 뉴스를 가져올 수 없음          |
| coding    | + 파일 읽기/쓰기. 셀 명령어                    | 코드 작성이나 파일 작업이 필요할 때  | curl로 웹 요청을 시도할 수 있지만 비효율적             |
| messaging | + 이메일 전송, 메신저 메시지                    | 에이전트가 대신 메시지를 보내야 할 때 |                                        |
| full      | 모든 도구 활성화<br>web_search, web_fetch 등 | 브라우저 자동화, 외부 API 호출 등 | 브레이브 서치로 최신 뉴스 검색<br>-> 기사 본문 읽기 -> 요약 |

#### 5. 커스텀 스킬 만들기
스킬을 만드는 스킬인 skill-creator가 기본으로 내장되어 있다.
```
my-skill
	SKILL.md # 에이전트가 읽는 지침서
	scripts/ # 에이전트가 실행하는 코드
		generate.py
```


#### 6. 오픈클로의 메모리
- 일일 로그: memory/ 폴더에 날짜별로 저장되는 파일이다. 2026-02-05.md처럼 날짜가 파일 이름이 된다.
에이전트가 하루 동안 처리한 주요 작업, 대화 요약, 중요한 결정 사항이 기록된다. 업무 일지와 비슷하다.
- 장기 기억: MEMORY.md 파일에 저장되는 중요 정보이다. 사용자의 선호, 자주 쓰는 형식, 반복되는 요청 패턴처럼 날짜와 무관하게 계속 기억해야할 내용이 들어간다.
> 일일 로그는 에이전트가 자동으로 작성한다. 별도 설정 없이도 하루가 끝나면 그날의 활동을 정리한다.
> 장기 기업은 에이전트가 "이건 오래 기억해야겠다"고 판단한 내용을 저장한다.
- 대화 정리
```
지금까지 대화 정리해줘.
```
/compact와 같은 동작이다.
에이전트가 대화 이력을 요약하고, 핵심 맥락만 남긴 뒤 나머지를 버린다. 중요한 내용은 저장되므로 걱장할 필요가 없다. 세부 대화는 사라지지만 핵심 맥락은 유지된다.
새로운 주제로 넘어갈 때 컴팩션을 해주면 이전 대화의 토큰을 아낄 수 있다. "이제 다른 얘기 할 건데, 앞에 내용 정리하고 시작하자"처럼 자연스럽게 말하면 된다.
- 프롬프트 캐싱: AI 프로바이더가 이전 요청의 입력을 기록해두는 경우가 있다. 앤트로픽 API의 프롬프트 캐싱이 대표적이다. 캐시된 토큰은 원래 가격의 10%만 과금된다.


#### 7. 비용 절감 방법
- 주제 전환시 컴팩션: 대화 이력 토큰 절감(가장 효과가 큼)
- 스킬 구성 고정(캐시 유지): 캐시 토큰 90% 할인
- 겹치는 스킬 정리: 오인 호출 방지, 캐시 안정성 확보
- 경량 모델로 단순 작업 처리: 모델 비용 자체 절감(오퍼스를 쓰는 건 변호사를 고용해 우편함 확인을 시키는 것과 같다.)
- 모델 라우팅: 오픈클로는 기본적으로 모든 요청을 agents.defaults.model.primary에 설정된 모델로 보낸다. 모든 작업을 전부 같은 모델을 쓴다. 비용을 줄이는 가장 간단한 방법은 기본 모델을 경량 모델로 낮추고 어려운 작업은 비싼 모델을 써야 한다.
- 자동으로 복잡도를 판단하여 모델 라우팅: 오픈 라우터의 Auto 모델을 프라이머리로 설정하면 오픈 라우터가 프롬프트를 보고 적절한 모델을 자동으로 선택한다.
#### 8. 멀티 에이전트
- 오픈클로는 하나의 게이트웨이에서 여러 에이전트를 동시에 운영할 수 있다. 각 에이전트는 독립된 워크스페이스, 세션, 인증 정보를 갖습니다.
- 멀티 에이전트는 상시 운영되는 별도의 에이전트이다.
###### (1) 새 에이전트 등록 방법
```
openclaw agents add <새 에이전트의 이름>
```
###### (2) 대화형 마법사가 시작되고, 모델과 프로바이더를 선택하면 루시 전용 워크스페이스가 생성된다.
별도의 모델과 인증을 설정할 수 있다. 에이전트 성격에 따라 주요 모델을 설정해도 좋다.

###### (3) 채널을 설정한다. 별도의 봇 계정을 만들어 연결할 수 있다.

###### (4) 새 에이전트의 워크스페이스에 SOUL.md 넣고 새 대화를 시작하자.

###### (5) 채널별 라우팅
에이전트별로 어떤 채널을 사용할지 정하면 openclaw.json에 설정이 반영된다.


#### 9. 서브 에이전트
- 서브 에이전트는 메인 에이전트가 임시로 생성하는 에이전트이다.
- 한 가지 일이 끝나야 다음 일을 시작하는 순차 처리 대신, 여러 작업을 동시에 진행할 수 있다.
- 메인 에이전트가 지시를 하면 별도의 세션에서 독립적으로 작업하는 에이전트가 서브 에이전트이다.
  (회사에서 팀장이 팀원에게 업무를 분배하는 것과 비슷하다.)
- 메인 에이전트는 다른 일을 계속하고, 팀원은 맡은 일을 끝내면 결과를 보고한다.
- 서브 에이전트는 '임시 알바'이다. 메인 에이전트가 큰 작업을 받으면, 부분 작업을 서브 에이전트에게 나눠준다. 서브 에이전트는 자기 일만 하고 결과를 돌려준 뒤 사라진다. 메인 에이전트는 그 사이에 다른 일을 계속할 수 있다.
- 주요 특징: 독립 세션(메인 에이전트와 분리된 세션), 병렬 처리(여러 서브 에이전트 동시 사용), 결과 보고(메인 에이전트에 결과 보고), 경량 구성(서브 에이전트는 SOUL.md, IDENTITY.md, USER.md를 상속 받지 않는다. 대신 AGENT.md에 정의된 규칙을 따른다. 시스템 프롬프트가 가벼우므로 토큰 비용이 적다.)
- 단순한 질문을 메인 에이전트가 직접 처리하는 것이 효율적이고, 시간이 오래 걸리거나 조사나 독립적인 작업에만 서브 에이전트를 활용하자.
- 서브 에이전트에 작업 시키는 프롬프트
```
일본 도쿄 여행을 준비 중이야. 다음 3가지를 동시에 조사해줘
1. ...
2. ...
3. ...
각각 서브 에이전트로 나눠서 처리해줘
```
메인 에이전트가 3개의 서브 에이전트를 생성한다. 각 서브 에이전트가 독립적으로 웹 검색을 하고, 결과를 정리해서 돌아온다.
- AGENTS.md로 규칙 정하기: 에이전트의 행동 규칙은 AGENTS.md 파일에서 정의한다. SOUL.md가 메인 에이전트의 성격이라면, AGENTS.md는 에이전트의 업무 규정이다.
```
AGENTS.md에 이 규칙들을 추가해줘.
- 서브 에이전트는 외부 서비스에 글을 쓰거나 메일을 보내지 않는다.
- 조사 결과는 마크다운 표로 정리한다.
- 작업 시간이 5분을 넘기면 중간 보고를 한다.
```

#### 10. 고급 설정
- 게이트웨이 설정, 메신저 채널, 에이전트 설정, AI 프로바이더, 도구 정책, 브라우저 프로필, 하트비트 주기, 크론 등록, 모델 페일오버, 세션관리 등을 설정한다.
```
~/.openclaw/openclaw.json
```
- openclaw config set
- 파일을 직접 열어서 항목 교체
- 에이전트에세 설정을 바꿔줘
  이상 3가지 방법으로 바꿀 수 있고 어떤 방법을 쓰든 결국 같은 파일이 수정된다.

---
- 매신저 채널 --- 내 컴퓨터(게이트웨이) --- AI 프로바이더
- 게이트웨이: 대화 기록, 메모리, 워크스페이스, 설정, 환경변수, 페르소나
- AI 모델에게 질문을 보내는 순간에는 데이터가 인터넷을 통해 프로바이더 서버로 전송된다. 설정 파일과 대화 기록은 내 컴퓨터에 남지만, 에이전트가 생각하는 과정은 클라우드에서 이루어진다.
- 샌드박스: 셀 명령어를 격리된 환경에서 실행하는 기능이다. 샌드박스 안에서 실행한 명령어는 시스템 전체에 영향을 주지 않는다. 도커 컨테이너 기반으로 에이전트별, 세션별로 격리할 수 있다.
- 오픈 클로 설치시 기본 셋팅: AI 모델 선택, 메신저 선택, 웹 검색 도구(Brave Search), 커뮤니티가 만든 스킬 선택(선택), 훅 설정
- 마크다운: 간단한 기호로 서식을 표현하는 텍스트 형식이다.
- AI 프로바이더: AI 모델을 제공하는 회사이다.
- 캐시 토큰: 같은 시스템 프롬프트가 반복되면, 프로바이더가 이전에 처리한 결과를 재사용한다. 이때 캐시된 입력 토큰은 정가의 10 ~ 25%만 과금된다.
- 시스템 프롬프트: 사용자의 대화가 시작되기 전에 AI에게 미리 전달되는 "배경 지시서"이다. 오픈 클로에서는 SOUL.md, IDENTITY.md 같은 설정 파일이 매번 시스템 프롬프트에 포함되어 에이전트가 자신의 성격과 규칙을 기억한다.
- 에이전트 내장 브라우저는 개인 크롬과 쿠키, 로그인 세션, 북마크가 모두 분리되어 있다.
- 워크스페이스: 에이전트가 파일을 일고 쓰는 공간이다. 기본 위치는 ~/.openclaw/workspace이다. 이 폴더 안에서만 작업하도록 설정하면 에이전트가 다른 디렉터리의 파일을 건드릴 일이 없다.
- 스킬은 에이전트에세 새로운 능력을 가르치는 설명서이다. 스킬을 설치하면 에이전트가 그 분야의 작업 방법을 알게 되고, 도구를 조합해서 더 정교한 일을 처리한다. 오픈클로에는 약 50개의 기본 스킬(번들 스킬)이 포함되어 있다.
- 스킬은 에이전트의 시스템 프롬프트에 포함된다. 에이전트가 대화를 시작할 때마다 활성화된 스킬 목록을 읽으며 사용자의 요청을 바탕으로 어떤 스킬을 사용해야 하는지 결정한다.
- 스킬 사용 규칙을 SOUL.md에 추가하자.
- 클로허브에 원하는 스킬이 없으면 직접 만들 수 있다. 에이전트에게 스킬을 만들도록 요청할 수 있다.
  오픈클로에는 스킬을 만드는 스킬인 skill-creator가 기본으로 내장되어 있다.
