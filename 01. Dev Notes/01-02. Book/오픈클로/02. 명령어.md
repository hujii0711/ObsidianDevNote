
온보딩 마법사 시작
```
openclaw onboard
```

모델 변경
```
openclaw models set google/gemini-3-flash-preview
```

모델 별칭 등록
```
openclaw models aliases add gemini google/gemini-3-flash-preview
```

게이트웨이를 재시작(설정 적용)
```
openclaw gateway restart
```

오픈클로 컨트롤 UI 접속(http://localhost:18789)
```
openclaw dashboard
```

메인 모델 실패시 자동으로 다른 모델 전환(failover chain)
```
openclaw models fallbacks add google/gemini-3-flash-preview
```

모델 상태 확인
```
openclaw models status
```

에이전크가 개인 크롬과 완전히 분리된 전용 브라우저 환경을 사용
```
openclaw config set browser.enabled true --json
openclaw config set browser.defaultProfile "openclaw"
```

에이전트 상태 점검
게이트웨이 상태, 연결된 채널, 활성 모델, 최근 로그를 한 번에 보여줌
```
openclaw status -all
```
게이트웨이 상태 확인
```
openclaw status
```

에이전트 문제 진단
openclaw.json의 스키마를 검증하고, 문법 오류나 잘못된 키를 찾아낸다.
-- fix 옵션을 붙이면 자동으로 수정 가능한 문제는 고쳐준다.
 ```
 openclaw doctor --fix
 ```

게이트웨이 재시작
```
openclaw gateway restart
```

설치된 스킬 목록 확인
```
openclaw skills list
```

스킬 설치
```
npx clawhub@latest install <스킬 이름>
```
설치가 끝나면 워크스페이스의 skills/ 폴더에 스킬 파일이 생긴다. 재시작 필요가 없다.

스킬이 제대로 설치했는지 확인
```
openclaw skills info <스킬 이름>
```

로그 확인
```
openclaw logs --limit 50
```
최근 50줄의 로그를 출력한다. 에러 메시지의 구체적인 내용을 확인할 수 있다.

크론 상태 확인
```
openclaw cron list
```

오픈클로 업데이트
```
openclaw update
```

오픈클로 버전 확인
```
openclaw --version
```

## 슬래시 명령어

- /usage cost-: 사용량 확인
- /model opus: 모델을 opus로 변경
- /tool-profile coding: 도구 프로필을 coding으로 변경
- /tool-profile full
- /clear: 이전 맥락을 지우고 새로 시작
- /compact: 대화의 맥락은 유지하면서 불필요한 디테일을 정리, 지금까지의 대화 요약, 토큰 비용 절약
- /new: 대화 기록을 완전히 지우고 새로 시작(새출발)
- /markdown-converter: 유투브 영상을 마크다운 파일로 변환하는 스킬을 활성화
- /compact: 대화 내용을 요약해서 토큰을 줄임
- /status: 토큰 사용량 확인
- /usage cost: 이번 세션에서 사용한 금액을 표시
- /clawhub install notion: 에이전트에 클로허브 명령어를 입력해 노션 스킬을 설치
