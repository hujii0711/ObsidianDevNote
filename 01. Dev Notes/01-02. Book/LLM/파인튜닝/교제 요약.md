### ★ 데이터 전처리
덱스트 전처리는 자연어 처리를 위해 원본 텍스트를 컴퓨터가 더 쉽게 분석할 수 있는 형태로 바꾸는 과정이다.
이는 우리가 일상에서 사용하는 언어, 즉 자연어를 다루는 작업이다. 텍스트 전처리는 <font color="#0070c0">데이터 클렌징</font>이라는 단계부터 시작한다. 데이터 클렌징은 데이터에서 잘못되거나 불완전한 부분을 찾아 고치거나 없애는 과정이다. 이렇게 하면 데이터의 품질이 좋아지고 분석 결과도 더 정확해진다. (중복 데이터 제거, 오타 교정, 잘못된 정보 교정, 개인정보 제거)
이런 데이터 클렌징 작업을 마친 후에야 텍스트는 본격적인 전처리 단계로 넘어 간다.
전처리의 첫단계인 토큰화에서는 텍스트를 의미 있는 작은 단위로 나눈다. 이어서 정규화 과정을 통해 인공지능 학습에 용이한 형태로 텍스트를 일관된 형식으로 만든다.
허깅페이스에서 제공하는 각 모델별 토크나이저는 텍스트를 효율적으로 처리할 수 있는 다양한 기능을 갖추고 있다.

4) 단어를 숫자로 바꾸는 것

### ★ GPU 병렬화

##### 1. 데이터 병렬화 (DP)
데이터 병렬 처리는 딥러닝에서 대규모 데이터셋을 효과적으로 다루는 병렬화 기법이다. 이 방법은 전체 데이터를 작은 덩러리 여러 개로 나눠 각각을 서로 다른 GPU와 같은 계산 장치에서 동시에 처리함으로써 전체 학습 과정의 속도를 높인다. DP의 장점은 여러 GPU로 데이터를 동시에 처리해 학습 시간을 크게 단축할 수 있고, 메모리 제약 없이 대규모 데이터섹을 효과적으로 다룰 수 있다는 점이다.
##### 2. 모델 병렬화 (MP)
모델 병렬화는 대규모 신경망 모델을 여러 GPU에 나누어 처리하는 방식이다. 이 방법은 모델을 수직으로 분할해 각 부분을 서로 다른 GPU에서 실행한다. 주로 .to() 메서드를 활용해 모델의 각 레이어를 특정 GPU에 직접 할당한다.
##### 3. 파이프라인 병렬화 (PP)
파이프라인 병렬화는 모델은 여러 단계로 분할 각 단계를 다른 계산 장치에서 동시에 실행함으로써 전체 모델의 실행 속도를 향상시키는 기법이다. 파이프라인 병렬화는 MP와 거의 동일하지만, 들어오는 배치를 미니 배치로 나누고 인위적으로 파이프라인을 생성해 여러 GPU가 계산 프로세스에 동시에 참여할 수 있도록 한다.
##### 4. 텐서 병렬화 (TP)
텐서 병렬 처리는 대규모 딥러닝 모델의 주요 계산 부하를 분산하는데 쓰이는 방법이다.
이 방식은 모델의 큰 텐서, 예를 들어 가중치 행렬을 여러 GPU에 나눠 할당한다. 각 GPU가 전체 텐서의 일부만 처리하므로 메모리 요구량이 줄고 계산 효율이 높아진다. 이 기법의 구현은 각 텐서를 작은 부분 행렬로 나누고, 이를 각기 다른 GPU에 할당하는 과정으로 시작된다.
##### 5. FSDP(Fully Sharded Data Parallel)
딥러닝에서 사용되는 고급 데이터 병렬 처리 기법중 하나로, 특히 대규모 모델 학습에 효과적이다. 이 방식은 각 GPU가 모델의 일부만을 메모리에 보유하도록 해 전체 모델의 메모리 요구 사항을 줄이는 것을 목표로 한다.

##### (3) 데이터 전처리
모델이 쉽게 이해할 수 있도록 대화 형식으로 데이터를 전처리한다.
주어진 텍스트 데이터를 사용자와 모델 간의 대화 형식으로 바꾼다.
- 특수 토큰을 도입하면서 모델이 대화의 구조를 더 잘 이해하고 처리할 수 있도록 한다. 이 과정을 통해 대화 데이터는 모델에 적합한 형태로 변환된다.

	3) `<end_of_turn>`: 사용자 또는 모델의 말이 끝났음을 모델에 알려주는 토큰
	4) `<eos>`: End of sequence의 약자로, 대화가 끝을 모델에 알려주는 토큰

##### (5) 콜레이터 설정
모델이 중점적으로 학습해야 할 부분을 알려주는 역할을 콜레이터가 한다. 콜레이터는 데이터 처리 과정에서 다양한 역할을 수행한다. 주요 기능으로는 <font color="#de7802">데이터 배치화(데이터 묶음),</font> <font color="#de7802">패딩(데이터의 길이 맞춤)</font>, <font color="#de7802">텐서 변환(데이터 타입 변환)</font>이 있으며, 중요하게는 <font color="#de7802">라벨링 작업</font>도 담당한다.
라벨을 달아 정확히 어느 부분을 학습해야 하는지 지정해 주는 역할을 한다. 이후의 대화에 모델이 더 집중하도록 도와준다. 텍스트를 형광펜으로 칠한 것처럼 표시해준다. 이렇게 표시해주면 모델은 이 부분이 특별히 중요하다는 것을 알게 된다. 데이터 콜레이터는 이 형광펜 칠하기 작업을 자동으로 해주는 도구라고 생각하면 된다. 학습 과정에서 이 도구는 입력된 대화 데이터에서 모델이 집중해야 할 부분을 자동으로 골라내고 표시해준다.
TRL 라이브러리에서 제공하는 <font color="#7030a0">DataCollatorForCompletionOnlyLM</font> 클래스를 사용해 `<start_of_turn>model` 이후의 대화에서 모델이 더 집중하도록 도와준다.

##### (7) 평가 메트릭 정의
평가 매트릭은 모델의 성능을 객관적으로 측정할 수 있는 중요한 지표이다. 이를 통해 우리는 모델이 얼마나 잘 작동하는지, 어떤 부분에서 개선이 필요한지를 파악할 수 있다. 평가 메트릭은 작업의 특성에 따라 다양하게 선택될 수 있으며, 예를 들어 분류 문제에서는 정확도(Accuracy), 정밀도(Precision), 재현율(Recall) 등을 회귀 문제에서는 평균 제곱 오차 값 등을 사용한다.
주로 기계 번역 품질을 특정하는 BLEU 점수와 모델 예측 결과가 실제 정답과 얼마나 정확하게 일치하는지를 측정하는 정확도(ACC)를 평가 지표로 사용한다.

##### (8) 모델 학습
모델, 토크나이저, 콜레이터, 데이터셋, 평가 지표를 모두 모아 학습을 도와주는 Trainer 클래스가 있다. 이 클래스는 이러한 요소들을 하날 묶어 효율적인 학습 과정을 제공한다.
trainer.train() 함수는 미리 정해진 스텝 수만큼 모델을 훈련하며, 설정된 배치 크기에 맞춰 데이터를 처리한다.
학습이 진행되는 동안 손실값을 계산하고 역전파를 통해 모델의 가중치를 업데이트한다.

##### (9) 모델 평가
trainer.evaluate() 함수는 모델의 성능을 평가하낟. 이 함수는 미리 정해둔 평가 데이터셋으로 모델의 예측을 만들고 compute_metrics 함수를 통해 정한 평가 지표를 계산한다. 평가할 때는 학습과 달리 모델의 가중치를 바꾸지 않고 현재 모델의 성능만 측정한다.

### 3. LoRA
LoRA는 적은 수의 파라미터만 조정해 모델을 효과적으로 튜닝하는 기법이다. 이와 함께 모델을 양자화해 메모리 사용량을 크게 줄이면서도 성능 저하를 최소화하는 기법이 QLoRA이다.
LoRA는 기존 가중치를 변경하지 않으면서 모델의 성능을 개선하는 방식으로 작동한다.
LoRA의 핵심은 A와 B라는 두 개의 작은 행렬을 사용하는 것이다. 이 행렬들은 새로운 정보나 수정 사항을 반영하기 위해 추가된다. A 행렬은 무작위로 초기화되고, B 행렬은 처음에 모든 값이 0으로 설정된다. 이 작은 행렬들을 사용함으로써 전체 모델을 조정하는 것보다 훨씬 더 효율적으로 학습할 수 있다.
LoRA는 대규모 모델의 전체 가중치를 변경하지 않고도 효과적으로 모델을 특정 작업에 맞게 조정할 수 있다.

### 4. QLoRA
양자화를 이용하면 모델이 사용하는 VRAM을 최소화하여 모델을 로드할 수 있다.
FP32, BF16, FP16 이는 모두 부동소수점 방식을 사용해 숫자를 표현하는 데이터 타입이다.
부동소수점 방식은 컴퓨터에서 실수를 표현하는 중요한 방법으로, 매우 큰 수부터 아주 작은 소수까지 다양한 범위의 숫자를 효율적으로 나타낼 수 있다. FP32(3.1415927401125732), FP16(3.140625)
더 많은 비트를 사용할수록 더 정밀한 값 표현이 가능하다.
비트의 수에 따라 표현할 수 있는 숫자의 범위와 정밀도가 달라진다.
부동소수점 표현에서 더 많은 비트를 사용할수록, 특히 0에 가까운 값들을 표현할 때 더 높은 정밀도를 얻을 수 있다. 

★ BF16
	BF16(bfloat16)은 16비트 부동소수점 형식의 한 변형이다. 이 형식은 일반적인 32비트 부동소수점 형식(FP32)과 동일한 범위의 수를 표현할 수 있다. 하지만 BF16의 정밀도는 FP32 보다는 낮다.
	그럼에도 불구하고 BF16은 일반적인 16비트 부동소수점 형식(FP16)보다는 높은 정밀도를 제공한다. 이러한 특성 덕분에 BF16은 딥러닝 모델 훈련에서 메모리 사용량을 줄이면서도 수치 안정성을 유지할 수 있는 좋은 대안으로 사용된다. BF16은 특히 큰 규모의 언어 모델 훈련에서 자주 활용되는데, 이는 계산 효율성과 모델 성능 사이의 균형을 잘 맞출 수 있기 때문이다.
	BF16은 FP32와 동일한 8비트의 지수를 유지하면서 가수만 23비트에서 7비트로 축소했습니다. 이러한 접근 방식 덕분에 BF16은 FP32와 동일한 +-3.4e38의 넒은 범위를 그대로 유지할 수 있다.
	BF16은 숫자를 매우 정밀하게 표현하는 능력은 일부 포기하는 대신, 매우 크거나 작은 숫자도 표현할 수 있게 해준다. 즉, BF16은 FP32만큼 다양한 크기의 숫자를 다룰 수 있으면서도 메모리 사용량을 줄이고 계산 속도도 향상할 수 있는 데이터 타입이다.
	→ 양자화는 컴퓨터가 다루는 숫자의 표현 방식을 조정하는 기술이다. 이 기술은 숫자를 표현하는 비트 수를 줄여 더 적은 메모리로 비슷한 정보를 저장할 수 있게 한다. (숫자의 표현 범위를 조절해 메모리 사용량을 줄이고 연산 속도를 높임) 결과적으로 양자화를 통해 대규모 AI 모델을 더 적은 컴퓨팅 자원으로 실행할 수 있게 되며, 이는 모델의 접근성과 활용도를 크게 높일 수 있다.₩

```python
# Quantization config 세팅 -> 모델이 사용하는 vram을 최소화하기
bnb_config = BitsAndBytesConfig(
	# 모델을 4비트 정밀도로 로드한다. 이는 메모리 사용량을 크게 줄여준다.
	load_in_4bit=True,
	
	# 이중 양자화 기법을 사용해 양자화로 인한 정확도 손실을 줄인다.
	bnb_4bit_use_double_quant=True,
	
	# 다양한 양자화 종류 중 nf4를 선택. 이는 정규화된 부동 소수점 4비트 양자화를 의미
	bnb_4bit_quant_type="nf4",
	
	# 계산 시 bfloat16 형식을 사용한다.
	# Llama 모델은 16비트 부동 소수점으로 학습됐기 때문에 이 설정이 적합
	bnb_4bit_compute_dtype=torch.bfloat16
)
```
