에러의 원인은 **줄 바꿈(Line Ending) 형식의 차이** 때문입니다.

`^M`이라는 문자는 윈도우 스타일의 줄 바꿈 기호인 **CRLF**를 의미합니다. 아마도 윈도우 환경에서 작성되거나 수정된 스크립트 파일을 맥(macOS, Unix 계열)에서 실행하려고 할 때 발생하는 전형적인 문제입니다.

### 상세 원인

- **윈도우(CRLF):** 줄 끝을 `\r\n` (Carriage Return + Line Feed)으로 표시합니다.
    
- **맥/리눅스(LF):** 줄 끝을 `\n` (Line Feed)으로만 표시합니다.
    
- **문제 발생:** 스크립트 첫 줄의 `#!/bin/bash` 뒤에 눈에 보이지 않는 `\r`(`^M`)이 붙어 있어, 시스템이 `/bin/bash`가 아닌 `/bin/bash\r`이라는 존재하지 않는 실행 파일을 찾으려고 하기 때문에 에러가 발생합니다.
    

---

### 해결 방법

가장 간단한 방법은 터미널에서 `sed` 명령어를 사용하여 `\r` 문자를 제거하는 것입니다.

#### 1. 터미널 명령어로 해결 (추천)

아래 명령어를 터미널에 입력하여 파일을 변환하세요.

Bash

```
sed -i '' 's/\r$//' setup.sh
```

그 후 다시 실행해 보세요.

Bash

```
./setup.sh
```

#### 2. VS Code에서 해결

만약 VS Code를 사용 중이시라면 더 직관적으로 고칠 수 있습니다.

1. VS Code로 `setup.sh` 파일을 엽니다.
    
2. 오른쪽 하단 상태 표시줄을 보면 **CRLF**라고 적힌 부분이 있습니다.
    
3. 그 부분을 클릭하여 **LF**로 변경합니다.
    
4. 파일을 저장(`Cmd + S`)하고 다시 터미널에서 실행합니다.
    

#### 3. vim 에디터에서 해결

Bash

```
vi setup.sh
```

명령어 모드에서 아래를 입력하고 저장(`:wq`)하세요.

Vim Script

```
:set ff=unix
```

### 💡 추가 팁

현재 폴더에 있는 모든 `.sh` 파일에 같은 문제가 있을 가능성이 높습니다. 아래 명령어로 한 번에 모든 쉘 스크립트의 줄 바꿈을 수정할 수 있습니다.

Bash

```
sed -i '' 's/\r$//' *.sh
```
