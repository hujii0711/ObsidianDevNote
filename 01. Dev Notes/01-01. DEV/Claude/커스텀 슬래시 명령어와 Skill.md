
> 결론: 거의 같은 것 — Skills이 슬래시 명령어를 통합/확장한 형태입니다

  >공식 문서 표현 그대로 옮기면

```
"Custom commands have been merged into skills. A file at .claude/commands/deploy.md and a skill at
.claude/skills/deploy/SKILL.md both create /deploy and work the same way."
```

즉 커스텀 슬래시 명령어 = Skills의 부분집합입니다.

### 핵심 차이 한 줄 요약

**슬래시 명령어**는 **사용자가 직접 호출**할 때 실행되고, **Skill**은 **Claude가 문맥을 보고 자동으로 판단**해서 로드합니다.

---

### 차이점

**1. 실행 주체 (가장 중요한 차이)**

슬래시 명령어는 사용자가 `/command`를 타이핑해야만 실행됩니다.
Skill은 대화 맥락을 분석해서 Claude가 자동으로 로드합니다. 이 차이가 작아 보이지만, 실제 워크플로우를 구성하는 방식 자체를 바꿉니다.

**2. 파일 구조**

슬래시 명령어는 `.claude/commands/`에 단일 파일로 저장되어 터미널 자동완성(`/...`)이 잘 작동합니다. Skill은 보통 `.claude/skills/` 하위에 디렉토리 단위로 구성되며, 지원 파일(템플릿, 스크립트, 예시 등)을 함께 포함할 수 있습니다.

**3. 컨텍스트 비용**

슬래시 명령어는 호출할 때마다 전체 내용이 컨텍스트에 삽입됩니다. Skill은 description만 먼저 로드되고, 실제 활성화될 때 전체 내용이 lazy-load 방식으로 불러와집니다.

**4. 인수(Arguments) 처리**

슬래시 명령어는 인수를 명시적으로 받습니다. 예: `/draft-linkedin-post [topic] [bullets]`. Skill은 대화 내용과 코드베이스에서 맥락을 스스로 추론합니다.

**5. 복잡성과 확장성**

Skill은 서브에이전트를 포크해서 실행하거나, 특정 에이전트 타입(Explore, Plan 등)을 지정하는 등 더 복잡한 워크플로우를 지원합니다.

---

### 공통점

둘 다 `/`로 시작하는 슬래시 인터페이스로 호출할 수 있습니다. 실제로 Claude Code는 예전에 commands와 skills를 별도 시스템으로 운영했지만, 현재는 두 위치의 파일 모두 동일한 슬래시 명령어 인터페이스를 만들어냅니다.

정리하면:

- 둘 다 마크다운 파일로 정의
- 둘 다 `/명령어`로 수동 호출 가능
- 둘 다 팀과 공유 가능
- 둘 다 반복 작업을 자동화하는 목적

---

### 언제 뭘 쓸까?

슬래시 명령어는 "지금 이 순간 판단해서 쓰는" 도구입니다. Skill은 "패턴을 미리 정의해두고 Claude가 알아서 꺼내 쓰게 하는" 도구입니다. 어떤 게 더 좋다기보다, 상황에 따라 맞는 걸 고르는 게 핵심입니다.

실용적인 기준으로는:

| 상황                                 | 추천      |
| ---------------------------------- | ------- |
| 특정 명령을 내가 직접 트리거하고 싶을 때            | 슬래시 명령어 |
| 반복적인 단순 작업 단축키가 필요할 때              | 슬래시 명령어 |
| 특정 언어/프레임워크 작업 시 자동으로 컨텍스트 주입 원할 때 | Skill   |
| 스크립트, 템플릿 등 여러 파일을 함께 묶고 싶을 때      | Skill   |
| 서브에이전트를 활용한 복잡한 멀티스텝 워크플로우         | Skill   |
