`gh`는 **GitHub CLI(Command Line Interface)**의 약자로, 터미널(Terminal)에서 웹 브라우저를 열지 않고도 GitHub의 다양한 기능을 직접 제어할 수 있게 해주는 도구입니다.

일반적인 `git` 명령어가 소스 코드의 이력(Commit, Push, Pull)을 관리한다면, `gh` 명령어는 **GitHub 플랫폼 자체의 협업 기능**(Issue, Pull Request, Release, Actions 등)을 관리하는 데 초점이 맞춰져 있습니다.

---

## 1. 주요 기능 및 명령어

`gh`는 직관적인 명령 구조를 가지고 있어 배우기 쉽습니다.

### 핵심 카테고리

- **`gh auth`**: GitHub 계정 로그인 및 인증 관리.
    
- **`gh repo`**: 저장소 생성, 복제(Clone), 포크(Fork), 설정 변경.
    
- **`gh pr`**: 풀 리퀘스트(PR) 생성, 목록 확인, 체크아웃, 리뷰 및 머지(Merge).
    
- **`gh issue`**: 이슈 생성, 할당, 상태 변경 및 조회.
    
- **`gh workflow` / `gh run`**: GitHub Actions 워크플로우 실행 상태 확인 및 수동 실행.
    
- **`gh secret`**: CI/CD 등에 필요한 환경 변수(Secrets) 및 변수(Variables) 관리.

## 2. 어떤 상황에서 유용한가?

### ① 컨텍스트 스위칭(Context Switching) 최소화

개발 중 PR을 보내거나 이슈를 확인하기 위해 브라우저로 이동하는 흐름을 끊지 않아도 됩니다.

- **상황:** 코딩을 마치고 바로 PR을 올리고 싶을 때.
    
- **명령:** `gh pr create --title "기능 구현 완료" --body "상세 내용..."` 한 줄로 브라우저 없이 PR 생성이 완료됩니다.
    

### ② 로컬에서 즉시 PR 확인 및 리뷰

동료의 PR 코드를 내 로컬 환경에서 직접 돌려보고 싶을 때 매우 강력합니다.

- **상황:** "이 PR 코드가 내 로컬에서도 잘 빌드되나?" 확인하고 싶을 때.
    
- **명령:** `gh pr checkout <PR번호>`를 입력하면, 해당 PR의 브랜치를 자동으로 로컬에 생성하고 체크아웃해 줍니다. 브랜치 이름을 일일이 찾을 필요가 없습니다.
    

### ③ 자동화 스크립트 작성 (DevOps 및 CI/CD)

반복적인 운영 업무를 스크립트화할 때 필수적입니다.

- **상황:** 새 프로젝트를 시작할 때마다 저장소를 만들고, 기본 Secret 값을 설정하고, 팀원을 초대해야 하는 경우.
    
- **활용:** 쉘 스크립트에 `gh repo create`, `gh secret set` 등을 포함하여 환경 구축을 완전히 자동화할 수 있습니다.
    

### ④ GitHub Actions 모니터링

배포 스크립트가 잘 돌아가는지 확인하기 위해 웹 페이지를 새로고침할 필요가 없습니다.

- **상황:** Push 후 배포가 성공했는지 터미널에서 계속 지켜보고 싶을 때.
    
- **명령:** `gh run watch`를 사용하면 실시간으로 워크플로우 진행 상태가 터미널에 표시됩니다.

## 3. `git` vs `gh` 한눈에 비교

|**구분**|**Git**|**GitHub CLI (gh)**|
|---|---|---|
|**관리 대상**|소스 코드 파일, 커밋 히스토리|저장소 설정, PR, 이슈, 팀 협업|
|**저장 위치**|로컬 및 원격 저장소|GitHub 서버 플랫폼 기능|
|**주요 작업**|`add`, `commit`, `push`, `merge`|`pr create`, `issue list`, `repo fork`|
|**웹 브라우저**|상관없음|브라우저 역할의 상당 부분을 대체|

## 4. 설치 및 시작하기

macOS, Windows, Linux 환경을 모두 지원하며 설치 후 간단한 인증 절차만 거치면 됩니다.

1. **설치 (macOS 기준):** `brew install gh`
    
2. **로그인:** `gh auth login` (브라우저나 토큰을 통해 한 번만 인증하면 됩니다.)
    
3. **상태 확인:** 현재 디렉토리의 저장소 정보를 보려면 `gh repo view`
    

> **Tip:** `gh` 명령어는 `git`과 함께 사용할 때 시너지가 가장 큽니다. 예를 들어 `git push` 후 바로 `gh pr create`를 실행하는 식의 워크플로우를 구성하면 개발 속도를 비약적으로 높일 수 있습니다.
