**Git Worktree**는 하나의 Git 저장소(Repository)에서 **여러 개의 작업 디렉토리(Working Tree)를 동시에 유지**할 수 있게 해주는 기능입니다.

보통 Git은 하나의 저장소 당 하나의 브랜치만 체크아웃하여 작업할 수 있지만, `git worktree`를 사용하면 별도의 폴더에 다른 브랜치를 동시에 펼쳐놓고 작업할 수 있습니다.

---

## 1. Git Worktree의 핵심 개념

일반적인 Git 환경에서는 브랜치를 변경(`git checkout`)할 때마다 현재 작업 중인 파일들이 바뀌거나, 작업 중인 내용을 `git stash`로 임시 저장해야 합니다. 하지만 Worktree를 사용하면 저장소의 `.git` 데이터는 공유하면서, 물리적인 폴더만 여러 개 생성하여 각기 다른 브랜치를 할당할 수 있습니다.

### 주요 명령어

- **추가:** `git worktree add <경로> <브랜치명>` (새 폴더에 특정 브랜치 체크아웃)
    
- **목록 확인:** `git worktree list` (현재 연결된 모든 워크트리 확인)
    
- **제거:** `git worktree remove <경로>` (작업이 끝난 워크트리 삭제)
    

---

## 2. 어떤 상황에서 유용한가?

### ① 긴급한 버그 수정 (Hotfix)

기능 개발(Feature Branch)을 위해 수천 개의 파일을 수정하던 중, 운영 서버에 긴급한 버그가 발생했다고 가정해 봅시다.

- **기존 방식:** 진행 중인 작업을 `stash`하거나 커밋한 뒤 브랜치를 전환해야 합니다.
    
- **Worktree 방식:** 현재 작업은 그대로 두고, 새 폴더에 `hotfix` 브랜치를 위한 워크트리를 생성하여 즉시 수정하고 배포할 수 있습니다.
    

### ② 코드 리뷰 및 대조

동료의 Pull Request(PR)를 로컬에서 직접 구동하며 리뷰해야 할 때 유용합니다.

- 내 작업 브랜치를 닫지 않고도, 옆 폴더에서 동료의 코드를 빌드하고 실행해 볼 수 있습니다.
    
- 두 브랜치의 실행 결과나 UI를 **동시에 띄워놓고 비교**할 때 매우 강력합니다.
    

### ③ 시간이 오래 걸리는 작업 (빌드, 테스트)

대규모 프로젝트에서 빌드나 테스트가 10분 이상 걸린다면?

- 한쪽 워크트리에서 테스트를 실행해 두고, 다른 워크트리 폴더로 이동하여 막힘없이 코딩을 계속할 수 있습니다.
    

### ④ 의존성(Dependencies) 충돌 방지

브랜치마다 `node_modules`나 `venv` 같은 라이브러리 버전이 크게 달라지는 경우, 브랜치를 바꿀 때마다 새로 설치(install)하는 시간을 아낄 수 있습니다. 각 워크트리 폴더마다 해당 브랜치에 맞는 의존성 환경이 유지되기 때문입니다.

---

## 3. `git stash` vs `git worktree` 비교

|**구분**|**git stash**|**git worktree**|
|---|---|---|
|**목적**|현재 작업의 임시 보관|여러 브랜치의 동시 작업|
|**흐름 끊김**|브랜치 전환 시 빌드/설정 초기화 발생 가능|각 폴더에서 독립적인 상태 유지|
|**동시성**|한 번에 하나의 브랜치만 확인 가능|여러 브랜치를 동시에 띄우고 비교 가능|
|**복잡도**|가볍고 빠름|관리를 위한 별도의 디렉토리 공간 필요|

---

## 4. 실전 사용 예시

새로운 기능을 개발하다가 긴급 수정을 해야 할 때의 워크플로우입니다.

1. **새 워크트리 생성:**
    
    Bash
    
    ```
    git worktree add ../emergency-fix main
    ```
    
    (부모 디렉토리의 `emergency-fix` 폴더에 `main` 브랜치를 펼칩니다.)
    
2. **이동 및 작업:**
    
    Bash
    
    ```
    cd ../emergency-fix
    # 버그 수정 및 커밋, 푸시 진행
    ```
    
3. **작업 완료 후 삭제:**
    
    Bash
    
    ```
    cd ../original-project  # 원래 작업 폴더로 복귀
    git worktree remove ../emergency-fix
    ```
    

> **주의사항:** Git의 무결성을 위해, **동일한 브랜치를 두 개 이상의 워크트리에 동시에 체크아웃할 수는 없습니다.** 각 워크트리는 서로 다른 브랜치를 바라보고 있어야 합니다.
