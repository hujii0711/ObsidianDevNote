`git log`는 한마디로 **"프로젝트의 타임라인을 확인하는 일지"**입니다. 저장소에 커밋된 기록들을 시간 역순(최신순)으로 나열하여, 누가, 언제, 어떤 메시지와 함께 코드를 저장했는지 한눈에 보여주는 도구입니다.

---

### 1. 기본적인 개념

터미널에 `git log`를 입력하면 기본적으로 다음과 같은 정보가 표시됩니다.

- **Commit ID (Hash):** 각 커밋을 식별하는 고유한 문자열입니다.
    
- **Author:** 코드를 작성한 사람의 이름과 이메일입니다.
    
- **Date:** 커밋이 이루어진 날짜와 시간입니다.
    
- **Commit Message:** 작업 내용에 대한 요약 설명입니다.
    

---

### 2. 어떤 상황에서 유용한가?

#### ① 최근 작업 내역 확인

"내가 어제 어디까지 코딩했더라?" 혹은 "동료가 마지막으로 수정한 내용이 뭐지?"라고 궁금할 때 가장 먼저 사용합니다. 프로젝트의 전체적인 흐름을 파악하기에 좋습니다.

#### ② 특정 파일의 수정 이력 추적

특정 파일에 언제부터 버그가 생겼는지, 누가 마지막으로 그 파일을 건드렸는지 확인하고 싶을 때 유용합니다.

> `git log [파일명]`

#### ③ 커밋 ID를 찾을 때 (`reset`이나 `revert`용)

앞서 배운 `git reset` 등으로 과거로 돌아가고 싶다면, 돌아갈 지점의 **커밋 해시(ID)**를 알아야 합니다. 이때 `git log`를 통해 정확한 주소를 복사할 수 있습니다.

#### ④ 브랜치 간의 차이 비교

내 브랜치에는 없고 `main` 브랜치에만 반영된 커밋이 무엇인지 필터링해서 볼 수 있습니다.

> `git log main..feature/login`

---

### 💡 더 유용하게 사용하는 옵션들

`git log`를 그냥 쓰면 내용이 너무 길어 가독성이 떨어질 수 있습니다. 개발자들이 자주 쓰는 꿀팁 옵션들을 챙겨가세요.

- **`--oneline`**: 커밋 하나를 딱 한 줄로 요약해서 보여줍니다. (전체 흐름 파악 최적)
    
- **`--graph`**: 브랜치가 갈라지고 합쳐지는 모습을 나뭇가지 모양의 그래프로 그려줍니다.
    
- **`-p`**: 각 커밋에서 **실제로 어떤 코드가 바뀌었는지(diff)**까지 함께 보여줍니다.
    
- **`-n [숫자]`**: 최근 몇 개의 커밋만 볼지 정합니다. (예: `git log -n 5`)
    

> **강력 추천 조합:**
> 
> `git log --oneline --graph --all`
> 
> (모든 브랜치의 흐름을 한 줄로, 그래프와 함께 보여주는 마법의 명령어입니다.)
