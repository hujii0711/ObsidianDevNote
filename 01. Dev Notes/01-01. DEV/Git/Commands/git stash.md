`git stash`는 한마디로 **"하던 작업을 잠시 서랍에 넣어두는 기능"**입니다. 현재 브랜치에서 수정 중인 내역이 있지만, 아직 커밋할 단계는 아닐 때 그 변경 사항들을 별도의 임시 공간(Stack)에 안전하게 보관하고 작업 디렉토리를 깨끗한 상태(마지막 커밋 상태)로 되돌려줍니다.

---

### 1. 기본적인 개념

작업을 하다가 급하게 브랜치를 전환(Checkout)해야 할 때, 수정한 내용이 있으면 Git은 충돌 방지를 위해 브랜치 전환을 막습니다. 이때 `stash`를 사용하면 수정한 코드를 날리지 않고도 깨끗한 상태에서 다른 업무를 볼 수 있습니다.

**주요 명령어:**

- **`git stash`**: 현재 작업 내용을 서랍에 넣고 작업 디렉토리를 비웁니다.
    
- **`git stash list`**: 서랍에 들어있는 작업 목록을 확인합니다.
    
- **`git stash apply`**: 서랍에서 가장 최근 작업 내용을 꺼내와서 현재 브랜치에 적용합니다. (서랍에는 기록이 남음)
    
- **`git stash pop`**: 작업 내용을 적용함과 동시에 서랍에서 삭제합니다. (가장 많이 사용)
    
- **`git stash drop`**: 서랍에 있는 특정 작업 내용을 삭제합니다.
    

---

### 2. 어떤 상황에서 유용한가?

#### ① 긴급한 버그 수정 요청이 왔을 때 (Context Switching)

새로운 기능을 열심히 개발하고 있는데, 운영 서버에 치명적인 버그가 터져서 즉시 수정해야 한다는 요청이 왔습니다. 현재 개발 중인 코드는 엉망이라 커밋하기 싫을 때, `stash`로 잠시 치워두고 `main` 브랜치로 가서 버그를 고친 뒤 다시 돌아와서 작업을 이어갈 수 있습니다.

#### ② `git pull`을 받으려는데 내 코드와 충돌날 때

원격의 최신 코드를 가져오고 싶은데, 내가 수정한 내용 때문에 `pull`이 거절당하는 경우가 있습니다. 이때 `stash`로 내 코드를 잠시 숨기고 `pull`을 받은 뒤, 다시 `pop`으로 내 코드를 꺼내어 합치면 충돌 해결이 훨씬 수월합니다.

#### ③ 코드를 갈아엎기 전 '백업'용으로

지금 작성 중인 로직이 마음에 안 들어서 다른 방식으로 시도해보고 싶을 때, 혹시 모를 상황을 대비해 현재 상태를 `stash`에 저장해두고 새로운 시도를 해볼 수 있습니다.

#### ④ 잘못된 브랜치에서 작업했을 때

`feature-A` 브랜치에서 작업해야 하는데 실수로 `main` 브랜치에서 코드를 수정했다면? 당황하지 말고 `stash` -> `git checkout feature-A` -> `git stash pop` 순서대로 실행하면 작업 내용을 그대로 옮겨올 수 있습니다.

`stash`를 여러 번 하면 나중에 어떤 게 무슨 작업인지 헷갈립니다. 이때 메시지를 남겨두면 관리하기 편합니다.

> **명령어:** `git stash save "결제 모듈 작업 중"`

Git의 이 마법 같은 기능까지 익히셨으니 이제 어떤 돌발 상황이 생겨도 유연하게 대처하실 수 있을 거예요!

가장 권장되는 방법은 Git의 **`stash`** 기능을 사용하는 것입니다. 작업 중인 내용을 "임시 저장소(Stack)"에 잠시 치워두는 방식입니다.

### 1. 터미널(Git CLI)을 사용하는 방법

가장 빠르고 표준적인 방법입니다.

1. **현재 변경사항 임시 저장**
    
    Bash
    
    ```
    git stash
    ```
    
    - 만약 추적되지 않은 파일(Untracked files, 새로 만든 파일 등)까지 포함하려면 `-u` 옵션을 붙이세요: `git stash -u`
        
2. **다른 브랜치로 체크아웃**
    
    Bash
    
    ```
    git checkout <대상-브랜치명>
    # 또는 최신 버전의 Git을 사용한다면
    git switch <대상-브랜치명>
    ```
    
3. **원래 브랜치로 돌아와서 복구하기** (나중에 작업 재개 시)
    
    Bash
    
    ```
    git checkout <원래-브랜치명>
    git stash pop
    ```
    
    - `pop`은 임시 저장된 내용을 가져오면서 스택에서 삭제합니다. 내용을 유지하고 싶다면 `git stash apply`를 사용하세요.
        

---

### 2. VS Code 인터페이스를 사용하는 방법

이전 대화에서 VS Code 환경을 다루고 계셨으니, UI에서 클릭 몇 번으로 처리할 수도 있습니다.

1. **Stash 실행:** 왼쪽 사이드바의 **소스 제어(Source Control)** 아이콘을 클릭합니다.
    
2. **메뉴 선택:** 상단의 `...` (View and More Actions) 아이콘을 누릅니다.
    
3. **Stash 선택:** `Stash` -> `Stash (Include Untracked)`를 클릭하고 식별할 수 있는 메시지를 입력합니다.
    
4. **브랜치 변경:** 하단 상태 표시줄의 현재 브랜치 이름을 클릭하여 다른 브랜치를 선택합니다.
    
5. **복구:** 나중에 다시 `...` 메뉴에서 `Stash` -> `Pop Latest Stash`를 선택하면 작업이 복구됩니다.

### 3. 임시 커밋(Temporary Commit) 방법

Stash가 익숙하지 않다면, 그냥 임시로 커밋을 해두는 방법도 있습니다.

1. **임시 커밋:** `git add .` -> `git commit -m "temp"`
    
2. **브랜치 이동:** `git checkout <대상-브랜치명>`
    
3. **복구 후 커밋 취소:** 원래 브랜치로 돌아온 뒤, 바로 이전 커밋을 취소하고 작업 상태로 되돌립니다.
    
    Bash
    
    ```
    git reset HEAD~1
    ```
    

**💡 팁:** 작업 내용이 많거나 복잡할 때는 `git stash push -m "작업내용설명"` 처럼 메시지를 남겨두면 나중에 `git stash list`로 확인했을 때 헷갈리지 않아 좋습니다.
