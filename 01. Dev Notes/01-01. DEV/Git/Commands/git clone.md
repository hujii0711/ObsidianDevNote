`git clone`은 말 그대로 원격 저장소(GitHub, GitLab 등)에 있는 프로젝트의 **모든 내용과 이력을 그대로 내 컴퓨터로 복제(Clone)**해오는 명령어입니다.

단순히 파일만 다운로드하는 것이 아니라, 그 프로젝트가 탄생한 순간부터 지금까지 쌓아온 **모든 커밋 기록과 브랜치 정보**를 통째로 가져오는 것이 특징입니다.

---

### 1. 기본적인 개념

`git clone`을 실행하면 내부적으로 다음과 같은 일들이 한꺼번에 일어납니다.

1. **디렉토리 생성**: 프로젝트 이름과 같은 폴더를 만듭니다.
    
2. **데이터 다운로드**: 원격 저장소의 모든 데이터를 로컬로 가져옵니다.
    
3. **원격 연결**: 가져온 원격 저장소를 자동으로 `origin`이라는 이름의 리모트로 등록합니다.
    
4. **체크아웃**: 가장 최신 상태의 코드를 작업 디렉토리에 펼쳐줍니다.
    

**기본 명령어:**

- `git clone [저장소 URL]`: 현재 폴더에 프로젝트 복제
    
- `git clone [저장소 URL] [폴더명]`: 지정한 폴더 이름으로 복제
    

---

### 2. 어떤 상황에서 유용한가?

#### ① 새로운 프로젝트에 투입되었을 때

회사에 입사하거나 새로운 팀 프로젝트에 참여하게 되었을 때, 기존에 쌓여있는 소스 코드를 내 컴퓨터로 처음 가져와서 개발 환경을 구축할 때 가장 먼저 사용합니다.

#### ② 다른 컴퓨터에서 작업을 이어가야 할 때

회사에서 작업하던 코드를 집에서도 보고 싶거나, 새 노트북을 구매해서 기존 프로젝트들을 옮겨와야 할 때 사용합니다. 굳이 USB로 옮길 필요 없이 URL 하나로 모든 상태를 복원할 수 있습니다.

#### ③ 오픈 소스 프로젝트를 분석하거나 기여하고 싶을 때

GitHub에서 마음에 드는 오픈 소스를 발견했을 때, 내 로컬 환경에서 직접 실행해보고 코드를 분석하기 위해 사용합니다. (기여를 하고 싶다면 보통 `Fork`를 먼저 한 뒤 자신의 저장소를 `Clone` 합니다.)

#### ④ 특정 시점의 코드를 안전하게 복구하고 싶을 때

내 로컬 저장소가 알 수 없는 이유로 완전히 망가졌을 때, 서버에 올라가 있는 최신 상태를 기반으로 다시 시작하기 위해 기존 폴더를 지우고 새로 `Clone` 받는 방식을 쓰기도 합니다.

---

### 💡 `git init`과의 차이점

- **`git init`**: 빈 폴더에서 **"이제부터 이 폴더를 Git으로 관리하겠어!"**라고 시작하는 것 (무에서 유를 창조)
    
- **`git clone`**: 이미 누군가(혹은 과거의 나)가 만들어 둔 **성 완성본과 설계도를 그대로 복사**해오는 것
    

---

### 🏁 요약

- **`git clone`**: 원격 저장소의 전체 이력을 내 로컬로 복제하는 첫 단추입니다.
    
- **장점**: 복제와 동시에 원격 저장소(`origin`)가 자동으로 연결되므로 바로 `pull`이나 `push`를 할 수 있습니다.
