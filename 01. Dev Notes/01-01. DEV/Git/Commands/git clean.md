`git clean`은 한마디로 **"Git이 추적하지 않는(Untracked) 파일들을 로컬 작업 디렉토리에서 말끔히 삭제하는 명령어"**입니다.

보통 `git reset --hard`가 이미 '추적 중인' 파일을 과거로 되돌린다면, `git clean`은 Git이 아예 모르는 파일(새로 생성된 파일, 빌드 결과물 등)을 정리할 때 사용합니다.

---

### 1. 기본적인 개념

Git 저장소에는 세 종류의 파일이 있습니다.

1. **Tracked**: 이미 커밋되었거나 스테이징된 파일
    
2. **Ignored**: `.gitignore`에 등록되어 무시되는 파일
    
3. **Untracked**: 새로 만들었지만 아직 `git add` 하지 않은 파일
    

`git clean`은 바로 이 **3번(Untracked)** 파일들을 삭제합니다. 기본적으로는 위험 방지를 위해 그냥 실행하면 동작하지 않으며, 특정 옵션을 붙여야 합니다.

- **`git clean -n` (Dry Run)**: 삭제될 파일 목록을 미리 보여줍니다. (가장 권장!)
    
- **`git clean -f` (Force)**: 파일을 실제로 삭제합니다.
    
- **`git clean -fd`**: 파일뿐만 아니라 빈 디렉토리까지 삭제합니다.
    

---

### 2. 어떤 상황에서 유용한가?

#### ① 빌드 결과물 및 임시 파일 청소

컴파일 언어나 프레임워크를 사용하다 보면 `dist`, `build`, `out` 같은 폴더나 수많은 임시 파일들이 생성됩니다. `.gitignore`에 등록되지 않은 이런 파일들이 너무 많아져서 저장소를 초기 상태처럼 깨끗하게 만들고 싶을 때 유용합니다.

#### ② 테스트 후 생성된 더미 파일 삭제

로컬에서 코드를 테스트하기 위해 수십 개의 텍스트 파일이나 로그 파일을 생성했을 때, 이를 일일이 손으로 지우는 대신 명령어 한 줄로 정리할 수 있습니다.

#### ③ 잘못 생성된 디렉토리 구조 초기화

스크립트 오류 등으로 원치 않는 폴더 구조가 대량으로 생성되었을 때, `git clean -fd`를 사용하면 Git 이력에 없는 모든 디렉토리를 한꺼번에 날려버릴 수 있습니다.

#### ④ `git merge`나 `rebase` 실패 후 뒷정리

복잡한 머지 과정에서 생긴 임시 파일들이나 충돌 해결 중에 생성된 백업 파일들을 정리할 때 사용합니다.

---

### ⚠️ 주의사항

**"삭제된 파일은 복구할 수 없습니다."**

`git clean`으로 삭제된 파일은 Git의 추적 대상이 아니었기 때문에 `git checkout`이나 `git reset`으로 되살릴 수 없습니다. 휴지통으로 가는 것이 아니라 영구 삭제되므로, 실행 전 반드시 **`-n` 옵션**으로 목록을 확인하는 습관이 필요합니다.

> **꿀팁:** `.gitignore`에 등록된 파일까지 지우고 싶다면 `-x` 옵션을 추가하세요 (`git clean -fdx`). 빌드 환경을 완전히 공장 초기화 상태로 만들 때 자주 사용됩니다.
