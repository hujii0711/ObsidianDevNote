```python
args = TrainingArguments(
	output_dir="code-llama-7b-text-to-sql", # 모델 저장 및 허브 업로드를 위한 디렉토리 지정 합니다.
	
	num_train_epochs=1, # number of training epochs
	
	# max_steps=100, # 100스텝 동안 훈련 수행합니다.

	tf32=True,

	warmup_ratio=0.03, # 워밍업 비율 0.03으로 설정 (QLoRA 논문 기반)합니다.
	
	lr_scheduler_type="constant", # 일정한 학습률 스케줄러 사용합니다.
	
	push_to_hub=True, # 훈련된 모델을 Hugging Face Hub에 업로드합니다.
	
	report_to="wandb", # wandb로 매트릭 관찰합니다.
)
```

1. output_dir
	학습중 만들어지는 모델 체크포인트를 저장할 폴더를 지정한다. 학습이 중간에 종료되거나 나중에 이어서 학습할 때. 이 디렉터리에 저장된 체크포인트를 불러와 작업을 재개할 수 있다.

2. num_train_epochs
	에폭은 전체 데이터셋을 한 번 학습하는 주기를 의미한다. 1로 설정하면 데이터셋을 한 번만 학습한다. 에폭 수를 늘리면 모델이 데이터를 더 많이 학습하지만, 과적합의 위험도가 증가할 수 있다.

3. per_device_train_batch_size
	학습할 때 GPU나 CPU에서 한 번에 처리할 데이터 샘플의 수를 정한다.
	배치 크기가 크면 한 번에 더 많은 데이터를 처리하지만, 그만큼 메모리 사용량도 커진다. 이는 학습 속도와 메모리 사용량 간의 균형을 맞추는 데 중요한 역할을 한다.

4. gradient_accumulation_steps
	그레이디언트 누적 기법을 사용해 가상의 더 큰 배치 크기 효과를 얻는 방법이다. 이번 실습에서는 2로 설정했으며, 모델이 2번의 순전파와 역전파를 수행한 후 한 번씩 가중치를 업데이트한다. 즉, 2개의 작은 배치에서 계산된 그레이디언트를 모아 한 번에 적용하는 것이다. 이 기법은 GPU 메모리 사용량을 줄이면서도 큰 배치 크기의 효과를 얻을 수 있어 메모리 효율성이 높다. 실제 배치 크기에 누적 스텝 수를 곱한 만큼의 유효 배치 크기 효과를 얻을 수 있어 더 안정적인 학습이 가능하다. 이는 GPU 메모리가 제한적이거나 매우 큰 모젤을 학습할 때 특히 유용하지만, 학습 시간이 약간 늘어날 수 있다는 점을 고려해야 한다.

5. gradient_checkpointing
	그레이디언트 체크포인팅 기법을 활성화한다. 이는 대규모 신경망 모델 학습 시 메모리 사용을 최적화하는 고급 기술이다. 일반적으로 역전파 과정에서 모든 중간 활성화 값을 메모리에 저장하는 대신, 그레이디언트 체크포인팅은 전략적으로 선택된 일부 활성화만 저장한다. 나머지 활성화는 역전파 중에 필요할 때 재계산된다. 이 방식은 메모리 사용량을 크게 줄이는 대신 약간의 계산 오버헤드를 발생시킨다. 결과적으로 더 큰 배치 크기를 사용하거나 더 깊은 모델을 학습하거나, 또는 더 긴 시퀀스를 처리할 수 있게 된다.

6. optim="adamw_torch_fused"
	이는 파이토치에서 제공하는 최적화된 AdamW 구현이다. AdamW는 널리 사용되는 AdamW는 널리 사용되는 AdamW 옵티마이저의 변형으로, 적응적 학습률 조정과 모멘텀을 결합한 방식이다.
	여기에 가중치 감쇠를 효과적으로 적용해 과적합을 줄이는 특징이 있다.
	"fused" 버전은 이러한 AdamW의 장점을 유지하면서도 계산 효율을 높인 것이다. 이 옵티마이저는 여러 연산을 하나로 합쳐(fused) GPU에서 더 빠르게 실행될 수 있도록 최적화됐다.

7. logging_steps
	지정한 크기 스텝마다 학습이 어떻게 진행되고 있는지 기록한다. 일정한 주기마다 학습 상태, 손실값 등을 기록해 학습 과정을 추적할 수 있게 도와준다.

8. save_strategy
	모델 체크 포인트를 저장하는 전략을 설정한다. "epoch"으로 설정하면 각 에폭이 끝날 때 마다 모델을 저장한다. 다른 옵션으로 "steps"(특정 스텝마다 저장) 또는 "no"(저장하지 않음)등이 있다.

9. max_grad_norm
	그레이디언트 클리핑의 임계값을 지정한다. 그레이디언트 클리핑은 학습 과정에서 그레이디언트의 크기가 지나치게 커지는 것을 방지하는 기법이다. 0.3이라는 값은 모든 파라미터의 그레이디언트를 합한 노름(norm)이 0.3을 초과할 경우, 그 비율에 맞춰 그레이디언트를 축소한다. 이는 급격한 가중치 변화를 억제해 학습의 안정성을 높이고, 그레이디언트 폭발 문제를 완화한다.

10. warmup_ratio
	학습률 웜업 기법을 제어한다. 웜업은 학습 초기에 학습률을 매우 작은 값에서 시작해 점진적으로 목표 학습률까지 증가시키는 방법이다. 0.03이라는 값은 전체 학습 스텝의 3% 동안 이 웜업 과정이 진행됨을 의미한다. 이 기법의 주요 목적은 학습 초기의 급격한 파라미터 변화를 방지하고, 모델이 안정적으로 학습을 시작할 수 있게 하는 것이다. 초기 학습 단계에서의 그레이디언트 불안정성을 줄이고, 모델이 더 나은 지역 최적점으로 수렴할 가능성을 높인다. 그러나 최적의 웜업 비율은 모델 크기, 데이터셋 특성, 전체 학습 에폭 수 등에 따라 달라질 수 있으며, 실험을 통해 조정이 필요할 수 있다.

11. lr_scheduler_type
	학습률 스케줄러이며 학습 과정에서 학습률의 변화를 관리한다. "constant"로 설정하면 학습 전체에 걸쳐 일정한 학습률을 유지한다. 다른 옵션으로는 "linear", "cosine", "cosine_with_restarts" 등이 있으며, 각각 다른 방식으로 학습률을 조절한다.

12. report_to
	어떤 도구를 사용해 학습 현황을 관찰할지 정하는 곳이다. 이를 통해 학습 과정을 모니터링 할 수 있다.
	예를 들어, wanb나 tensorboard와 같은 툴을 지정할 수 있으며, 이를 통해 시각적으로 학습 과정을 모니터링할 수 있다.
