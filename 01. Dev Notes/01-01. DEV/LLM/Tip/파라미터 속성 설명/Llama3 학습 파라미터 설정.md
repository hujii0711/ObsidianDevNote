
```yaml
model_name: "meta-llama/Meta-Llama-3.1-8B-Instruct"
dataset_path: "."
max_seq_length: 512
output_dir: "./llama-3.1-korean-8b-hf-20-epoch"
report_to: "wandb"
learning_rate: 0.00005
lr_scheduler_type: "linear"
num_train_epochs: 5
per_device_train_batch_size: 2
per_device_eval_batch_size: 2
gradient_accumulation_steps: 4
optim: "adamw_torch_fused"
logging_steps: 10
save_strategy: "epoch"
weight_decay: 0.01
max_grad_norm: 0.5
warmup_ratio: 0.03
bf16: true
tf32: true
gradient_checkpointing: true
fsdp: "full_shard auto_wrap"
fsdp_config:
  backward_prefetch: "backward_pre"
  forward_prefetch: "false"
  use_orig_params: "false"
```

1. model_name
	사용하고자하는 모델의 ID를 나타내며, 허깅페이스에 등록된 모델 이름을 적는다.

2. dataset_path
	학습에 사용할 데이터셋이 위치한 경로를 지정한다.

3. max_seq_length
	모델이 처리할 수 있는 최대 시퀀스 길이를 설정하는 파라미터이다. 이 파라미터 값을 모델이 한 번에 처리할 수 있는 입력 텍스트의 최대 길이를 결정하며, 더 긴 시퀀스는 자동으로 max_seq_length 길이로
	잘리고, 더 짧은 시퀀스는 패딩된다. 이 설정은 모델의 성능과 메모리 사용량에 직접적인 영향을 미치므로 사용 가능한 하드웨어 자원과 작업의 특성을 고려해 적절히 조정해야 한다.

4. output_dir
	모델 학습 과정에서 생성되는 학습된 모델, 학습 중간중간에 저장되는 체크 포인트 파일들을 저장할 디렉터리이다.

5. report_to
	모델 학습 과정에서 발생하는 로그와 성능 지표를 어떤 플랫폼에 기록할지 지정하는 옵션이다. 위에서는 wandb를 사용한다.
	이러한 도구들은 학습 손실, 학습률 변화 등의 중요한 지표들을 실시간 모니터링이 가능하게 도와준다.

6. learning_rate
	신경망의 가중치를 얼마나 크게 조정할지 결정하는 중요한 하이퍼파아미터이다. 일반적으로 전체 파인 튜닝은 1e-5에서 5e-5 사이의 값이 자주 사용되며, 이번 학습에서는 0.00005를 사용할 예정이다.
	학습률의 선택은 모델의 크기, 데이터셋의 특성, 학습 목표 그리고 학습 방법에 따라 달라진다. 높은 학습률은 최적점을 빠르게 찾을 수 있지만, 학습이 불안정해지거나 발산할 위험이 있다.
	반면, 너무 낮은 학습률은 안정적이지만 학습이 매우 느리거나 지역 최적점에 갇힐 수 있다.

7. lr_scheduler_type
	학습률 스케줄러이며, 학습 과정에서 학습률의 변화를 관리한다.
	constant로 설정하면 학습 전체에 걸쳐 일정한 학습률을 유지한다. 다른 옵션으로는 linear, cosine, consine_with_restarts 등이 있으며, 각각 다른 방식으로 학습률을 조절한다.

8. num_train_epochs
	에폭이며 전체 데이터셋을 한 번 학습하는 주기를 의미한다.
	1로 설정하면 데이터셋을 한 번만 학습한다. 에폭 수를 늘리면 모델이 데이터를 더 많이 학습하지만, 과적합의 위험도 증가할 수 있다.

9. per_device_train_batch_size
	각 GPU 또는 CPU에서 한 번에 처리할 수 학습 데이터의 개수이다.
	일반적으로 컴퓨터의 메모리와 처리 능력이 감당할 수 있는 가장 큰 배치 크기를 사용하면 더 빠르게 학습 할 수 있다.

10. per_device_eval_batch_size
	평가 시 각 디바이스에서 처리할 데이터의 개수이다.

11. gradient_accumulation_steps
	그레이디언트 누적 기법을 사용해 가상의 더 큰 배치 크기 효과를 얻는 방법이다.
	위에서는 4로 설정했고, 이는 모델이 4번의 순전파와 역전파를 수행한 후 한 번씩 가중치를 업데이트 한다. 즉 4개의 작은 배치에서 계산된 그레이디언트를 모아 한 번에 적용하는 것이다.
	이 기법은 GPU 메모리 사용량을 줄이면서도 큰 배치 크기의 효과를 얻을 수 있어 메모리 효율성이 높다.
	실제 배치 크기에 누적 스텝 수를 곱한 만큼의 유효 배치 크기 효과를 얻을 수 있어 더 안정적인 학습이 가능하다.
	이는 GPU 메모리가 제한적이거나 매우 큰 모델을 학습할 때 특히 유용하지만, 학습 시간이 약간 늘어날 수 있다는 점을 고려해야한다.

12. optim
	모델 학습에 사용할 최적화 알고리즘을 지정한다.
	위에서는 adamw_torch_fused를 사용한다. 이는 파이토치에서 제공하는 최적화된 AdamW 구현한다. AdamW는 널리 사용되는 Adam 옵티마이저의 변형으로, 적응적 학습률 조정과 모멘텀을 결합한 방식이다.
	여기에 가중치 감쇠(weight decay)를 효과적으로 적용해 과적합을 줄이는 특징이 있다. fused 버전은 이러한 AdamW의 장점을 유지하면서도 계산 효율을 높인 것이다.
	이 옵티마이저는 여러 연산을 하나로 합쳐(fuse) GPU에서 더 빠르게 실행될 수 있도록 최적화됐다.

13. logging_steps
	로그를 기록할 주기를 스텝 단위로 설정한다. 10으로 설정하면 10스텝마다 학습 진행 상황과 성능 지표가 기록된다.

14. save_strategy
	모델 체크포인트를 저장하는 전략을 설정한다. epoch으로 설정하면 각 에폭이 끝날때 마다 모델을 저장한다. 다른 옵션으로는 steps(특정 스텝마다 저장) 또는 no(저장하지 않음) 등이 있다.

15. weight_decay
	과적합을 방지하고 모델의 일반화 성능을 향상시키는 데 사용된다.

16. max_grad_norm
	그레이디언트 클리핑의 임계값을 지정한다.
	그레이디언트 클리핑은 학습 과정에서 그레이디언트의 크기가 지나치게 커지는 것을 방지하는 기법이다.
	0.3이라는 값은 모든 파라미터의 그레이디언트를 합한 노름이 0.3을 초과할 경우, 그 비율에 맞춰 그레이디언트를 축소한다.
	이는 급격한 가중치 변화를 억제해 학습의 안정성을 높이고 그레이디언트 폭발 문제를 완화한다.

17. warmup_ratio
	학습률 웜업 기법을 제어한다. 웜업은 학습 초기에 학습률을 매우 작은 값에서 시작해 점진적으로 목표 학습률까지 증가시키는 방법이다.
	0.03이라는 값은 전체 학습 스텝의 3% 동안 이 웜업 과정이 진행됨을 의미한다.
	이 기법의 주요 목적은 학습 초기의 급격한 파라미터 변화를 방지하고, 모델이 안정적으로 학습을 시작할 수 있게 하는 것이다.
	초기 학습 단계에서의 그레이디언트 불안정성을 줄이고, 모델이 더 나은 지역 최적점으로 수렴할 가능성을 높인다.
	그러나 최적의 웜업 비율은 모델 크기, 데이터셋 특성, 전체 학습 에폭 수 등에 따라 달라질 수 있으며, 실험을 통해 조정이 필요할 수 있다.

18. bf16
	bfloat16 정밀도를 사용하도록 활성화한다. bfloat16은 32비트 부동소수점(float32)의 지수 부분은 그대로 유지하면서 가수 부분을 줄인 16비트 형식이다.
	이는 넓은 동적 범위를 유지하면서 메모리 사용량을 절반으로 줄이고 연산 속도를 높인다. 특히 딥러닝 모델 학습에서 float32와 유사한 수치 안정성을 제공하면서 더 효율적인 학습을 가능하게 한다.

19. tf32
	NVDIA의 TensorFloat-32(TF32) 형식을 활성화한다. TF32는 NVDIA의 Ampere 아키텍처 이상의 GPU에서 제공되는 특별한 내부 표현 방식이다. 이는 19비트 부동소수점 형식으로,
	float32의 정밀도와 float16의 속도 사이의 균형을 제공한다. 이를 통해 프로그램의 변경 없이도 행렬 곱셈과 같은 주요 연산의 속도를 크게 향상할 수 있다.

20. gradient_checkpointing
	그레이디언트 체크포인팅 기법을 활성화한다. 이는 대규모 신경망 모델 학습 시 메모리 사용을 최적화하는 고급 기술이다.
	일반적으로 역전파 과정에서 모든 중간 활성화 값을 메모리에 저장하는 대신, 그레이디언트 체크포인팅은 전략적으로 선택된 일부 활성화만 저장한다.
	나머지 활성화는 역전파 중에 필요할 때 재계산된다. 이 방식으로 메모리 사용량을 크게 줄이는 대신 약간의 계산 오버헤드를 발생시킨다.
	결과적으로 더 큰 배치 크기를 사용하거나, 더 깊은 모델을 학습하거나 또는 더 긴 시퀀스를 처리할 수 있게 된다.
	한편, gradient_checkpointing을 사용하다 보면 use_cache와 함께 쓸 수 없다는 경고문이 뜬다.
	use_cache는 모델의 키와 값을 캐시해 추론 시 계산 효율을 높이는 기능이다.
	이는 주로 텍스트 생성 작업에서 사용되며, 이전에 계산된 키와 값을 재사용해 연속적인 토큰 생성 속도를 높인다.
	그러나 gradient_checkpointing과 use_cache는 동시에 사용할 수 없다. 그 이유는 gradient_checkpointing이 메모리 절약을 위해 중간 활성화를 재계산하는 반면, use_cache는 이러한 중간 결과를 저장하고
	재사용하기 때문이다. 두 기술의 작동 방식이 서로 상충되어 함께 사용할 경우 오류가 발생하거나 의도한 효과를 얻을 수 없다.

21. fsdp
	full_shard auto_wrap는 Fully Sharded Data Pararallel(FSDP)을 활성화해 모델을 여러 GPU에 분산시켜 메모리 효율을 높이는 기술을 적용한다.
	full_shard는 모든 파라미터를 분산시키고, auto_wrap은 자동으로 모델을 FSDP로 래핑한다.
	FSDP란 각 GPU가 전체 모델의 일부만을 저장하고 필요할 때 다른 GPU와 통신해 데이터를 주고받는 방식으로 작동한다.

22. fsdp_config
	fsdp_config의 하위 속성 설정들은 이 분산 처리의 효율성을 더욱 높여준다.

23. backward_prefetch
	backward_pre 설정은 FSDP에서 역전파 과정을 최적화하는 중요한 기능이다.
	이 설정은 역전파 계산이 시작되기 직전에 필요한 모델 파라미터를 미리 가져오는 전략을 활성화한다.
	FSDP에서 모델 파라미터가 여러 GPU에 분산돼 있기 때문에 이 설정은 각 역전파 단계 시작 전에 필요한 파라미터를 효율적으로 준비한다.
	이를 통해 대기 시간이 감소하고, 파라미터를 가져오는 통신 작업과 실제 계산 작업을 부분적으로 병렬화할 수 있으며, 메모리 사용을 최적화한다.

24. forward_prefetch
	forward_prefetch를 false로 설정하면 순전파 과정 중 추가적인 데이터 프리페치(미리 가져오기)를 비활성화한다.
	일반적으로 프리페치는 성능을 향상시키기 위해 사용되지만, FSDP 컨텍스트에서는 오히려 불필요할 수 있다.
	순전파 중 프리페치를 비활성화함으로써 모델은 현재 필요한 데이터만을 처리하게 된다.
	이는 GPU 간 불필요한 데이터 전송을 줄이고, 메모리 사용을 최적화하며, 전체적인 계산 효율을 높일 수 있다.

25. use_orig_params
	use_orig_params를 false로 설정하면 원본 파라미터 대신 분할된 파라미터를 직접 사용해 메모리 효율성을 극대화한다.
