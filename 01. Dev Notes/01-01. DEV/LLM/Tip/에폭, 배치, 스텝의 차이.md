
### 1. 에폭 (Epoch)
- 전체 데이터셋을 한 번 모두 학습하는 과정을 의미.
- 데이터셋이 10,000개이고 에폭 1번이면, 10,000개 데이터를 한 번 학습하는 것.
- 에폭 수가 많을수록 모델이 데이터에서 학습할 기회가 많지만, 너무 많으면 과적합(Overfitting) 위험이 있음.

### 2. 배치 (Batch)
한 번의 학습에서 모델이 한꺼번에 처리하는 데이터 개수.
데이터셋을 나누는 단위로 사용되며, 일반적으로 GPU 메모리 한계 때문에 모든 데이터를 한 번에 학습시키지 않고 배치 단위로 학습함.
예시)
전체 데이터 10,000개, 배치 크기 32 → 한 번에 32개 샘플을 학습.
따라서 한 에폭 동안 총 (10,000 ÷ 32) ≈ 312 스텝 필요.

### 3. 스텝 (Step)
모델이 한 번의 배치를 학습하는 과정.
즉, 한 스텝은 한 배치 크기만큼의 데이터를 학습하고 가중치를 업데이트하는 과정.
예시)
배치 크기 32, 전체 데이터 10,000개 → 한 에폭 동안 312 스텝 필요.

### 4. 예제 시뮬레이션
1) 데이터셋 크기: 10,000개
- 배치 크기(Batch Size): 32
- 에폭(Epoch): 3

2) 학습 과정
- 한 에폭에서 필요한 스텝 수
	→ 10,000 ÷ 32 ≈ 312 Steps per Epoch
- 전체 학습에서 필요한 스텝 수 (Epoch 3번)
	→ 312 × 3 = 936 Steps

3) 정리
- 에폭: 데이터셋 전체를 몇 번 반복할지.
- 배치 크기: 한 번에 몇 개씩 학습할지.
- 스텝: 한 배치를 학습하는 과정(즉, 배치의 수만큼 스텝이 필요).
