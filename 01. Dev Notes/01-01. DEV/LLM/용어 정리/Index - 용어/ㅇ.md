- 어텐션 메커니즘
	어텐션은 트랜스포머만이 아니라 다양한 가중치를 입력의 다른 부분에 할당하려는 딥러닝 모델에서 사용되는 메커니즘입니다.
	이를 통해 모델은 번역이나 요약과 같은 작업을 수행하면서 가장 중요한 정보를 우선시하고 강조할 수 있습니다.
	본질적으로 모델이 동적으로 입력의 다른 부분에 집중할 수 있게 하여, 성능 향상과 더 정확한 결과를 이끌어냅니다.
	
	트랜스포머 아키텍처의 논문 [어텐션만 있으면 됩니다]에서 소개된 어텐션 메커니즘은 LLM이 입력 시퀀스 의 다양한 부분에 동적으로 집중할 수 있게 하여, 
	예측을 하는 각 부분의 중요성을 결정합니다.
	
	이전의 신경망이 모든 입력을 동등하게 처리했던 것과 달리, 주의력을 갖춘 LLM은 예측 정확도를 혁명적으 로 변화시켰습니다.
	어텐션 메커니즘은 주로 LLM이 내부 세계 모델과 사람이 인식할 수 있는 규칙을 학습하거나 인식할 수 있도록 하는 역할을 담당합니다.
	어떤 연구에 따르면 LLM은 오델로 게임과 같은 합성 작업에 대한 일련의 규칙을 과거 게임 데이터에 대한 훈련만으로도 배울 수 있습니다.
	이를 통해 LLM이 사전 훈련과 파인튜닝으로 어떤 다른 종류의 '규칙'을 배울 수 있는지 연구하는 새로운 길이 열렸습니다.

- 인간 피드백 기반 강화 학습(Reinforcement Learning from Human Feedback, RLHF)
	RLHF는 인공지능 모델을 사람 감독자의 피드백에 기반하여 훈련하는 미신러닝에서 사용되는 정렬 기술입니다.
	사람은 모델의 답변에 따라 보상이나 벌칙을 제공하여 훈려 과정을 효과적으로 안내합니다.
	목적은 모델의 답변이 사람의 기대 및 필요와 더 밀접하게 일치하도록 모델의 행동을 개선하는 것입니다.

- AI 피드백 기반 강화 학습(Reinforcement Learning from AI Feedback, RLAIF)
	RLAIF는 훈련하는 동안 AI를 사용하여 모델에 피드백을 제공하는 모델 정렬 방법입니다.
	AI는 모델의 출력을 평가하고 보상이나 벌칙을 제공하는 데 사용됩니다.
	RLAIF와 유사하게, 목표는 모델의 성능을 최적화하고 그 답변을 원하는 결과에 더 밀접하게 일치시켜 특정 작업에 대한 유용성을 향상시키는 것입니다.

- 양자화(Quantization)
	양자화(Quantization)는 연속적인 값을 제한된 이산적인 값들로 변환하는 과정입니다. 
	주로 신호 처리나 머신러닝에서 사용되며, 예를 들어, 아날로그 신호를 디지털로 변환하거나, 딥러닝 모델에서 숫자 범위를 축소해 계산을 가볍게 하는 데 활용됩니다.
	신경망 모델의 가중치와 활성화값을 낮은 정밀도로 변환해 연산 효율성과 메모리 사용을 최적화하는 기술(언어모델에서) 양자화는 신경망의 가중치와 편향의 정밀도를 줄이는 기술입니다.
	이는 더 작은 모델 크기와 빠른 추론 시간을 결과로 가져오며, 모델 정확도에는 약간의 감소가 있습니다.

- 원 핫 인코딩(One-Hot Encoding)
	딥러닝에서 원 핫 인코딩(One-Hot Encoding)은 범주형 데이터를 수치형 데이터로 변환하는 기법 중 하나입니다.
	이는 딥러닝 모델이 범주형 데이터를 처리할 수 있도록 돕기 위해 사용됩니다. 원 핫 인코딩의 주요 특징과 작동 방식은 다음과 같습니다:
	
	1. 범주형 데이터의 표현
	범주형 데이터는 특정 범주(또는 클래스)로 이루어진 데이터입니다.
	예를 들어 색상("빨강", "초록", "파랑")이나 동물 종류("고양이", "개", "새") 등이 있습니다. 이러한 데이터를 수치형으로 변환해야 모델이 처리할 수 있습니다.
	
	2. 인코딩 방식
	원 핫 인코딩은 각 범주를 고유한 이진 벡터로 변환합니다. 이 벡터는 해당 범주에 해당하는 위치에만 1을 갖고, 나머지 위치에는 0을 갖습니다.
	 
	예를 들어, 세 가지 클래스가 있을 때:
	------------------------------
	   - "고양이"는 [1, 0, 0]
	   - "개"는        [0, 1, 0]
	   - "새"는        [0, 0, 1]
	------------------------------
	이렇게 각 클래스는 고유한 1과 0으로 이루어진 벡터로 표현됩니다.
	
	1. 장점
	원 핫 인코딩은 범주형 데이터를 수치형 데이터로 변환하는 간단하고 효과적인 방법입니다.
	또한, 각 범주가 독립적임을 명확히 표현하므로, 모델이 범주 간의 순서를 잘못 해석하는 것을 방지합니다.
	
	2. 단점
	범주의 수가 많을 경우, 원 핫 인코딩으로 생성된 벡터의 길이가 매우 길어질 수 있습니다.
	이는 메모리 사용량이 증가하고, 계산 효율성이 떨어질 수 있는 단점으로 작용할 수 있습니다.
	
	3. 사용 사례
	원 핫 인코딩은 텍스트 데이터에서 단어를 벡터로 표현하는 데 사용되거나,
	범주형 특성을 가진 데이터셋에서 범주형 변수를 모델에 입력하기 전에 변환할 때 사용됩니다.
	
	딥러닝에서 원 핫 인코딩은 모델이 범주형 데이터를 효과적으로 처리할 수 있도록 돕고, 데이터의 구조적 특 성을 유지하면서 수치형으로 변환하는 데 유용한 방법입니다.

- OOV(Out-of-Vocabulary)
	자연어 처리(NLP)에서 "OVV 문제"는 "Out-of-Vocabulary" 문제를 의미합니다.
	이는 모델이 학습되지 않은 단어나 토큰을 처리할 때 발생하는 문제입니다. 모델이 학습 중에 보지 못한 단언
	OVV 문제는 특히 고정된 어휘(vocabulary)를 사용하는 모델에서 빈번하게 발생합니다.
	
	▶︎ 정의
		OVV 문제는 모델이 학습 중에 보지 못한 단어를 입력으로 받았을 때 그 단어를 이해하거나 처리하지 못하는 상황을 말합니다.
		이는 모델의 성능 저하로 이어질 수 있습니다.
	
	▶︎ 원인
		• 고정된 어휘: 대부분의 NLP 모델은 사전에 정의된 어휘 집합을 사용하여 텍스트 데이터를 처리합니다. 학습 시 사용된 데이터에 없는 단어는 OVV로 간주됩니다.
		• 언어의 다양성: 새로운 단어, 속어, 오타, 또는 드물게 사용되는 단어 등은 학습 데이터에 포함되지 않을 수 있습니다.
	
	▶︎ 영향
		• OVV 문제는 특히 드문 단어 또는 특정 도메인에 특화된 용어가 많은 경우 모델의 성능을 저하시킬 수 있습니다.
		• 모델이 새로운 단어를 인식하지 못하면 해당 문장의 의미를 제대로 파악하지 못할 수 있습니다.
	
	cf.) 서브워드 토큰화를 통한 OVV 문제 완화
	
	▶︎ 서브워드 토큰화(Subword Tokenization)
		• 서브워드 토큰화는 단어를 더 작은 단위인 서브워드(subword)로 분할하여 처리하는 방법입니다. 이는 새로운 단어라도 그 단어를 구성하는 서브워드가 기존 어휘에 포함되어 있을 가능성을 높입니다.
		• 대표적인 서브워드 토큰화 기법으로는 BPE(Byte Pair Encoding)와 WordPiece가 있습니다.
	
	▶︎ 장점
		• 유연성: 새로운 단어를 그 구성 요소인 서브워드로 분해하여 처리할 수 있으므로 OVV 문제를 크게 완화합니다.
		• 어휘 크기 감소: 서브워드 단위로 어휘를 구성하면 모델이 처리해야 하는 어휘의 크기를 줄일 수 있습니다.
		• 일반화: 비슷한 형태의 단어들이 비슷한 서브워드를 공유하게 되어, 모델이 더 잘 일반화할 수 있습니다.
	
	서브워드 토큰화는 자연어 처리에서 OVV 문제를 해결하는 데 매우 효과적인 방법으로, 특히 BERT, GPT 등 현대의 많은 NLP 모델에서 사용되고 있습니다. 이를 통해 모델은 더 많은 단어를 효과적으로 다룰 수 있으며, 새로운 단어에도 더 잘 대응할 수 있습니다.

- 인코더
	인코더는 텍스트를 이해하는 데 뛰어나다.
	원시 텍스트를 받아들여 핵심 구성 요소로 분리하고 해당 구성 요소를 벡터로 변환하는 업무를 담당하며, 어텐션을 사용하여 텍스트의 맥락을 이해한다.
	
	cf.) 디코더
	디코더는 텍스트를 생성하는 데 뛰어남
	디코더는 수정된 형식의 어텐션을 사용하여 다음에 올 최적의 토큰을 예측함으로써 텍스트를 생성하는 데 뛰어나다.
	
	cp.) 인코더와 디코더를 함께 사용하는 모델은 전체 모델이 시퀀스-투-시퀀스 모델이 된다.
	
	cf.) 바이-인코더
	바이-인코더는 두 개의 BERT 모델을 훈련시키는 과정을 포함합니다.
	하나는 입력 텍스트를 인코딩하기 위한 것이고 다른 하나는 출력 텍스트를 인코딩하기 위한 것입니다.
	이 두 모델은 입력과 출력 텍스트의 쌍이 서로가 최대한 유사하도록 큰 텍스트 데이터 말뭉치에서 동시에 훈련됩니다.
	결과적으로 얻어진 임베딩들은 입려과 출력 텍스트 사이의 의미적 관계를 포착합니다.
	바이-인코더는 단일 LLM의 복제 두 개가 문서 간의 유사성을 배우기 위해 병렬로 학습하는 고유한 방식으로 훈련됩니다.
	예를 들어, 바이-인코더는 질문을 단락과 연관시켜 벡터 공간에서 서로 가깝게 표시되도록 학습할 수 있습니다.

- 임베딩
	임베딩은 고차원 공간에서의 단어, 구절, 또는 토큰의 수학적 표현입니다. 자연어 처리에서 임베딩은 다른 단어와의 의미와 관계를 포착하는 방식으로 단어, 구절 또는 토큰을 나타냅니다.
	LLM은 사전 훈련을 기반으로 토큰에 대한 다양한 임베딩을 학습하며, 파인튜닝 중에 이러한 임베딩을 다시 업데이트할 수 있습니다.
	
	cp.) 벡터는 다시 텍스트로 되돌릴 수 없지만, 인코딩된 상태에서 점수를 비교할 수 있는 추가적인 이점을 가진 텍스트의 표현입니다.
	LLM이 활용한 텍스트 임베딩을 통해 단어와 문구의 표면적인 구문이나 철자를 넘어서는 의미를 포착할 수 있습니다.
	이러한 텍스트 임베딩을 이용하면 언어 사용에 관한 풍부한 정보를 활용하여 LLM의 사전 훈련과 파인튜닝이 기반이 되는 애플리케이션을 거의 무한하게
	구축할 수 있습니다.
	그리고 언어와 문맥의 깊은 이해를 할 수 있게 해주며, 다양한 언어 작업에 있어서 더욱 정교하고 의미적으로 정확한 모델을 구축할 수 있습니다.

- 엔드 투 엔드(end-to-end)
	엔드 투 엔드(end-to-end)는 시스템이나 프로세스의 시작부터 끝까지 모든 단계를 포괄하는 접근 방식을 의미합니다.
	이 용어는 다양한 분야에서 사용되며, 각각의 맥락에 따라 약간씩 다른 의미를 가질 수 있습니다. 몇 가지 일반적인 예를 들어보겠습니다:
	
	1) 소프트웨어 개발: 엔드 투 엔드 테스트는 애플리케이션의 모든 기능을 처음부터 끝까지 실제 사용자의 관점에서 테스트하는 것을 의미합니다. 이는 개별 기능이나 모듈 테스트와 달리, 시스템 전체가 예상대로 작동하는지를 확인하는 데 중점을 둡니다.
	
	2) 비즈니스 프로세스: 엔드 투 엔드 프로세스는 고객의 요구가 발생하는 시점부터 그 요구가 완전히 충족되는 시점까지의 모든 단계를 포함합니다. 예를 들어, 주문 처리 시스템에서 고객의 주문 접수부터 제품 배송까지의 모든 과정을 포함합니다.
	
	3) 인공지능 및 머신러닝: 엔드 투 엔드 학습은 데이터 입력부터 최종 출력까지의 모든 단계를 하나의 모델로 학습하는 방법을 의미합니다. 이는 여러 단계로 나뉜 전통적인 접근 방식과 대조적입니다.

- ONNX
	ONNX를 사용하면 한 프레임워크(예, 파이토치)에서 모델을 내보내고 다른 프레임워크(예, 텐서플로)로 가져와 추론할 수 있습니다.
	이러한 프레임워크 간 호환성은 다양한 환경과 플랫폼에서 모델을 배포하는 데 매우 유용합니다.
