- 트랜스포머 아키텍처
	최신 LLM의 기본 아키텍처인 트랜스포머 아키텍처는 2017년에 도입되었으며, 주로 두 가지 주요 구성 요소로 이루어진 시퀀스 투 시퀀스 모델입니다.
	인코더와 디코더, 인코더는 원시 텍스트를 처리하고, 그것을 핵심 구성 요소로 분할하며, 이들을 백터로 변환하고, 문맥을 파악하기 위해 어텐션 메커니즘을 사용하는
	역할을 합니다. 디코더는 수정된 어텐션 메커니즘을 사용하여 다음에 나올 최적을 토큰을 예측함으로써 텍스트를 생성하는 데 뛰어납니다.
	복잡함에도 트랜스포머와 BERT, GPT 같은 변형들은 자연어 처리(NLP)의 텍스트를 이해하고 생성하는 능력을 혁명적으로 변화시켰습니다.

- 토큰
	토큰은 의미를 가지는 가장 작은 단위로, 문장이나 텍스트를 작은 단위로 나누어 생성된다. 토큰은 LLM의 기본 입력이다. 토큰은 단어일 수 있지만 이 책 전반에 걸쳐 더 깊게 볼 것처럼 '하위 단어' 일 수도 있다.
	
	cf.) n-gram
	n-gram은 n개의 연속적인 토큰의 시퀀스를 나타낸다.
	
	• 토큰화
	토큰화는 텍스트를 가장 작은 이해 단위인 토큰을 분해하는 과정입니다.
	이 토큰들은 의미를 내포한 정보 조각이며, 어텐션 계산에 입력으로 사용되어 LLM이 실제로 학습하고 작동하게 됩니다.
	거의 모든 LLM에는 모델에 특정 의미를 가진 특별한 토큰이 있습니다.
	예를 들어, BERT 모델에는 특별한 [CLS] 토큰이 있으며, BERT는 이를 모든 입력의 첫 번째 토큰으로 자동으로 삽입하며, 이는 전체 입력 시퀀스에 대한 인코딩된 의미를 나타내기 위한 것입니다.
	
	cf.) 토큰 개수 (Token Count)
	토큰 개수는 텍스트가 토큰화된 후 생긴 개별 토큰들의 총 개수를 의미합니다.
	예를 들어, "I love AI models."이라는 문장을 토큰화하면,
	["I", "love", "AI", "models", "."] → 토큰 개수: 5개
	LLM에서 입력할 수 있는 최대 토큰 개수는 모델마다 다름 (예: GPT-3.5는 4096개, GPT-4는 8192개 등).
	
	토큰 개수는 입력된 텍스트의 길이와 관련됨.
	차원(Dims)은 각 토큰이 임베딩된 벡터의 크기를 의미하며, 모델이 정한 고정된 값임.
	예를 들어, 512개의 토큰이 입력되고, 모델의 차원이 768이라면 → 최종적으로 (512, 768) 크기의 행렬이 생성됨.

- 텐서
	텐서는 딥러닝에서 데이터를 표현하는 기본 단위로, 다차원 (숫자)배열 형태로 데이터를 저장하고 처리한다.
	문자열을 숫자로 인코딩하고 이를 텐서로 변환하는 과정은 모델이 텍스트 데이터를 이해하고 학습할 수 있도록 하는 필수적인 전처리 단계이다.
	
	cf.) 텐서의 속성 확인
	• tokenized_data.shape: 텐서 차원과 크기를 나타냄
	• tokenized_data.dtype: 텐서에 저장된 데이터 타입을 나타냄
	
	2차원 텐서: 전통적인 행렬 연산을 수행 (torch.Size([A, B]))
	3차원 텐서: 배치(batch) 연산이 가능하며, NLP / 이미지 / 시계열 분석에서 사용됨 (torch.Size([A, B, C]))
