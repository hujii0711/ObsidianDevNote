- 과대적합(Overfitting)
	머신러닝에서 과적합 또는 과대적합은 모델이 훈련 데이터를 너무 잘 학습하여 보지 못한 데이터나 테스트 데이터에서 성능이 떨어지는 상태를 말합니다.
	모델은 본질적으로 훈련 데이터의 노이즈나 무작위 변동을 암기하기 때문에 새로운 데이터에 대한 학습을 일반화하는 데 실패합니다.
	LLM의 경우, 모델이 훈련 데이터의 특정한 사항에 지나치게 조정되어, 보지 못한 프롬프트에 대한 합리적인 답변을 생성하는 능력을 잃는 경우에 과적합이 발생할 수 있습니다.
	이로인해 모델이 새로운 프롬프트를 올바르게 대응하지 못하는, 너무 특정적이거나 좁게 맞춰진 답변을 생성할 수 있습니다.
	▶︎ 모델이 학습 데이터는 잘 예측하지만, 조금이라도 다른 상황에서는 제대로 대응하지 못하는 현상을 말한다.

	cf.) 재앙적 망각 현상
	파인튜닝 과정에서 모델이 이전에 학습한 일반적인 지식을 잃어버리는 현상을 말한다.
	이 현상이 발생하는 주된 이유는 모델이 새로운 작업을 학습하는 과정에서 신경망의 가중치가 변화하기 때문이다. 즉, 가중치의 변화로 모델이 이전에 잘 수행하던 작업에 대한 능력을 잃어버리는 현상이다.

- 과소적합(Underfitting)
	머신러닝에서 과소적합은 모델이 훈련 데이터에 있는 기본 패턴을 포착하기에 너무 단순하여 훈련 및 테스트 데이터 모두에서 성능이 떨어지는 상태를 말합니다.
	일반적으로 모델이 충분한 복잡성을 갖지 못하거나 충분히 오래 훈련되지 않았을 때 발생합니다.
	LLM의 경우, 모델이 훈련 데이터의 맥락이나 애매함을 파악하지 못하여, 프롬프트에 대한 답변으로 너무 일반적이거나 주제에서 벗어나거나 무의미한 출력을 생성하는 경우에 과소적합이 발생할 수 있습니다.

- 개체명 인식(Named Entity Recognition, NER)

	개체명 인식(Named Entity Recognition, NER)은 자연어 처리(NLP)의 한 과제로, 텍스트에서 특정한 개체명(예: 사람, 장소, 조직, 날짜 등)을 식별하고 분류하는 작업을 의미합니다.
	NER은 문장에서 중요한 정보를 추출하여 구조화된 데이터로 변환하는 데 사용됩니다. 이는 정보 검색, 질문 응답 시스템, 문서 요약 등 다양한 응용 분야에서 핵심적인 역할을 합니다.
	
	개체명 인식의 주요 특징과 과정은 다음과 같습니다
	
	1. 개체명 식별
	텍스트 내에서 특정 개체명을 식별합니다. 예를 들어, "애플은 새로운 아이폰을 출시했다."라는 문장에서 "애플"과 "아이폰"을 식별하는 것입니다.
	
	2. 개체명 분류
	식별된 개체명을 사전 정의된 카테고리로 분류합니다. 예를 들어, "애플"은 조직(Organization), "아이폰"은 제품(Product)으로 분류될 수 있습니다.
	
	3. 딥러닝 모델의 활용
	전통적으로 NER은 규칙 기반 접근이나 통계적 방법으로 수행되었지만, 최근에는 딥러닝 모델이 널리 사용되고 있습니다.
	특히, RNN(Recurrent Neural Networks), LSTM(Long Short-Term Memory), 그리고 Transformer 기반의 모델(BERT 등)이 NER 작업에 효과적으로 사용됩니다.
	
	4. 언어 모델의 중요성
	BERT와 같은 사전 훈련된 언어 모델은 NER 작업에서 뛰어난 성능을 보입니다.
	이러한 모델들은 대량의 텍스트 데이터를 통해 사전 훈련되어, 문맥을 이해하고 다양한 언어적 패턴을 학습합니다. 이를 통해 NER 작업에서의 정밀도와 정확도를 크게 향상시킬 수 있습니다.
	
	5. 응용 분야
	NER은 정보 추출 시스템, 자동 문서 분류, 검색 엔진 최적화, 소셜 미디어 분석 등 다양한 분야에서 활용됩니다. 예를 들어, 뉴스 기사에서 주요 인물과 사건을 자동으로 식별하여 요약하거나, 고객 리뷰에서 특정 제품명을 추출하는 데 사용될 수 있습니다.
	
	개체명 인식은 텍스트 데이터를 구조화된 정보로 변환하는 중요한 과정으로, 다양한 산업에서 유용한 정보를 추출하는 데 기여하고 있습니다.
	딥러닝 기술의 발전은 NER의 정확성과 효율성을 크게 향상시키고 있습니다.
	
	cp) 토큰 분류
	토큰 분류 모젤은 자연어 처리 분야에서 문장을 구성하는 각 토큰(단어 또는 토큰화된 단위)에 레이블을 할당하는 모델을 의미한다. 이는 개체명 인식과 같은 과제에 활용된다.

- 교차 엔트로피 손실 함수(Cross-Entropy Loss)
	교차 엔트로피 손실 함수는 분류 문제에서 모델의 예측과 실제 레이블 간의 차이를 측정하는 데 사용되는 손실 함수입니다. 이 함수는 특히 다중 클래스 분류 문제에서 널리 사용됩니다.
	교차 엔트로피 손실은 모델이 예측한 확률 분포와 실제 레이블의 확률 분포 간의 차이를 계산하여, 모델이 얼마나 잘 예측하고 있는지를 평가합니다.

	수식)
	이진 분류의 경우, 교차 엔트로피 손실은 다음과 같이 정의됩니다:
    [
    L(y, hat{y}) = -left(y cdot log(hat{y}) + (1-y) cdot log(1-hat{y})right)
    ]
    여기서 ( y )는 실제 레이블 (0 또는 1), ( hat{y} )는 모델의 예측 확률입니다.

	다중 클래스 분류의 경우, 소프트맥스 함수와 함께 사용되며, 손실은 다음과 같이 정의됩니다:
    [
    L(y, hat{y}) = -sum_{i=1}^{C} y_i cdot log(hat{y}_i)
    ]
    여기서 ( C )는 클래스의 수, ( y_i )는 실제 클래스에 대한 원-핫 인코딩 벡터, ( hat{y}_i )는 소프트맥스 함수로 계산된 예측 확률입니다.

	작동 원리)
	  교차 엔트로피 손실은 모델이 예측한 확률이 실제 레이블과 얼마나 일치하는지를 측정합니다. 예측 확률이 실제 레이블과 가까울수록 손실 값은 작아집니다. 반대로, 예측이 실제 레이블과 많이 다를수록 손실 값은 커집니다.
	
	교차 엔트로피 손실 함수의 특징)
	
	확률적 해석: 교차 엔트로피는 확률 분포 간의 차이를 측정하므로, 예측을 확률로 해석할 수 있는 장점을 제공합니다.
	  
	효율적 학습: 특히 소프트맥스와 함께 사용될 때, 모델은 각 클래스에 대한 확률을 출력하기 때문에 다중 클래스 분류 문제에서 효과적으로 작동합니다.
	
	불확실성 처리: 모델이 예측하는 확률이 0에 가까울 때 손실이 무한대로 증가할 수 있으므로, 모델이 불확실한 예측을 하지 않도록 유도합니다.
	
	교차 엔트로피 손실 함수는 분류 문제에서 모델의 성능을 최적화하는 데 매우 유용하며, 특히 딥러닝 모델에서 널리 사용됩니다. 이를 통해 모델은 예측의 정확도를 높이고, 더 나은 일반화 성능을 발휘할 수 있습니다.

- 그레이디언트
	모델이 학습할 때 사용하는 일종의 방향 지시등이라고 생각하면 된다.
	모델이 정답에 가까워지려면 어느 방향으로 얼마나 움직여야 하는지를 알려주는 값이다.

	cf.) RMSNorm(Root Mean Square Normalization)
	그레이디언트 값이 너무 커지거나 작아지면 마치 내비케이션이 갑자기 이상한 방향을 가리키는 것처럼 학습에 문제가 생길수 있다.
	RMSNorm은 학습 과정에서 그레이디언트가 너무 커지거나 작아지는 문제를 해결해 모델이 더 안정적으로 학습할 수 있도록 돕는다.
	▶︎ 그레이디언트라는 방향 지시등이 항상 적절한 크기를 유지하도록 조절해주는 장치이다.
