- 레이블 지정 데이터(Labeled Data)
	레이블 지정 데이터는 일반적으로 특정 작업을 위한 하나 또는 복수의 레이블로 주석이 달린 데이터 요소 또는 데이터 샘플입니다.
	이러한 레이블은 해당 데이터 요소에 대한 올바른 출력이나 답을 나타냅니다.
	지도 학습의 맥락에서 레이블이 지정된 데이터는 훈련 과정의 기초 역할을 합니다. LLM을 포함한 모델들은 이 데이터를 사용하여 올바른 패턴과 연관성을 학습합니다.
	데이터 레이블링은 일반적으로 사람이 직접 원시 데이터를 검토하고 적절한 레이블을 지정합니다. 레이블링 과정은 주석자의 이해, 해석, 주관적 편견에 영향을 받을 수 있으며,
	이로 인해 레이블이 지정된 데이터에 편향이 생길 수 있고, 훈련된 모델들은 이러한 편향을 반영할 수 있습니다.
	그러므로 편향을 최소화하기 위해 레이블링 과정을 신중하게 관리하는 것이 중요합니다.

- logits(모델 출력)
	모델 학습의 결과로서, 로짓은 확률로 변환되기 전의 원시 점수 값이다.
	이 로짓은 각 가능한 출력 클래스에 대한 상대적인 점수를 나타내며, 일반적으로 소프트맥스 함수를 통해 확률로 변환된다.
	이 로짓은 모델의 최종 출력으로, 이를 바탕으로 언어 모젤에서는 로짓이 각 단어가 다음에 올 확률을 나타내는 점수가 되며, 이 중 가장 높은 점수를 가진 단어가 다음 단어로 예측된다.
	
	허깅페이스(Hugging Face) 라이브러리에서 제공하는 파이프라인(pipeline) 기능은 다양한 자연어 처리 작업을 간편하게 수행할 수 있도록 도와줍니다.
	이 과정에서 "logits"라는 용어는 모델의 출력 중 하나로, 신경망이 입력 데이터에 대해 계산한 비정규화된 예측값을 의미합니다.

	Logits의 역할과 특징은 다음과 같습니다.
	
	1. 비정규화된 출력값
	Logits는 모델이 각 클래스에 대해 계산한 원시 점수입니다. 이 값들은 아직 확률로 변환되지 않은 상태로, 보통 실수 범위의 값을 가집니다.
	
	2. 확률로 변환
	Logits는 소프트맥스(softmax) 함수와 같은 활성화 함수를 통해 확률 분포로 변환됩니다.
	소프트맥스 함수는 logits를 입력으로 받아 각 클래스에 대한 확률을 출력하며, 이 확률의 합은 1이 됩니다.
	예를 들어, 분류 문제에서 소프트맥스를 적용하여 각 클래스에 속할 확률을 계산할 수 있습니다.
	
	3. 의사결정
	모델의 최종 예측은 보통 이 확률 값들 중 가장 높은 값을 가지는 클래스를 선택하는 방식으로 이루어집니다.
	따라서, logits는 모델의 예측 과정에서 중요한 중간 단계입니다.
	
	4. 활용 사례
	Logits는 손실 함수 계산에도 사용됩니다. 예를 들어, 크로스 엔트로피 손실 함수는 logits와 실제 레이블을 사용하여 모델의 예측 성능을 평가합니다.
	이 과정에서 logits는 확률로 변환되기 전의 비정규화된 점수로 사용됩니다.
	
	허깅페이스의 파이프라인은 다양한 NLP 작업을 간편하게 수행할 수 있도록 하며, logits는 이러한 작업에서 모델의 예측을 이해하고 활용하는 데 중요한 역할을 합니다.
	Logits는 모델이 입력 데이터에 대해 어떤 결정을 내리려 하는지를 보여주는 중요한 지표로, 이를 통해 모델의 예측 성능을 개선하거나 분석할 수 있습니다.
