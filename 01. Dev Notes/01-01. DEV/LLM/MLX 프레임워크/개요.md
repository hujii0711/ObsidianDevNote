## MLX 프레임워크 개요

---

### 🍎 MLX란?

MLX는 Apple Silicon을 위해 처음부터 새로 설계된 오픈소스 배열 프레임워크입니다. 기본 수치 연산부터 대규모 프론티어 머신러닝 모델 실행까지 유연하게 활용할 수 있습니다. [Apple Developer](https://developer.apple.com/videos/play/wwdc2025/315/)

---

### 🔑 핵심 특징 5가지

**1. 통합 메모리 (Unified Memory)**
MLX의 가장 큰 차별점은 통합 메모리 모델입니다. MLX 배열은 공유 메모리에 존재하며, 데이터 전송 없이 CPU와 GPU 어디서든 연산이 가능합니다. [GitHub](https://github.com/ml-explore/mlx)

**2. 지연 연산 (Lazy Computation)**
MLX의 연산은 지연(lazy) 방식으로 동작하며, 배열은 실제로 필요할 때만 메모리에 구체화됩니다. [GitHub](https://github.com/ml-explore/mlx)

**3. 동적 그래프 (Dynamic Graph)**
연산 그래프가 동적으로 구성되어, 함수 인자의 shape이 바뀌어도 느린 재컴파일이 없고 디버깅이 직관적입니다. [GitHub](https://github.com/ml-explore/mlx)

**4. 친숙한 API**
Python API는 NumPy를 밀접하게 따르며, `mlx.nn`과 `mlx.optimizers`는 PyTorch 스타일로 복잡한 모델 구축을 단순화합니다. [GitHub](https://github.com/ml-explore/mlx)

**5. 다양한 언어 지원**
Python, Swift, C++, C API를 모두 지원합니다. [Apple Developer](https://developer.apple.com/videos/play/wwdc2025/298/)

### 🚀 주요 활용 사례

텍스트 생성 및 LLM 파인튜닝을 위한 **MLX LM**, OpenAI Whisper 기반 음성 인식 **MLX Whisper**, 이미지/오디오/비디오 생성 등 다양한 예제를 공식 지원합니다. [Mlx-framework](https://mlx-framework.org/)

### 📱 최신 동향 (2025~2026)

M5 칩의 Neural Accelerators를 활용하기 위해 macOS 26.2 이상이 필요하며, Metal 4의 TensorOps를 통해 행렬 연산을 전용 하드웨어로 가속합니다. [Apple Machine Learning Research](https://machinelearning.apple.com/research/exploring-llms-mlx-m5)

Ollama도 Apple Silicon에서 MLX 기반으로 전환하여 통합 메모리 아키텍처를 활용한 대폭적인 성능 향상을 이루었습니다. [Ollama](https://ollama.com/blog/mlx)

### 📊 한눈에 비교

| 항목 | MLX | PyTorch |
|---|---|---|
| 최적화 대상 | Apple Silicon 전용 | 크로스 플랫폼 |
| 메모리 방식 | 통합 메모리 (자동) | 디바이스 명시 필요 |
| API 스타일 | NumPy + PyTorch 혼합 | PyTorch 스타일 |
| LLM 로컬 실행 | 매우 효율적 | MPS 지원 (제한적) |
| 클라우드/Linux | ❌ 미지원 | ✅ 지원 |

**결론:** Mac에서 LLM 실행, 파인튜닝, 온디바이스 AI 개발을 한다면 MLX가 현재 가장 효율적인 선택입니다.
