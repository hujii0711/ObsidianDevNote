## MLX에서 torch / numpy / pandas 대체 여부

결론부터 말씀드리면, **완전 대체가 아니라 역할 분담**입니다.

---

### 🔵 NumPy → `mlx.core` (거의 대체 가능)

MLX의 Python API는 NumPy를 매우 유사하게 따르도록 설계되었습니다. [GitHub](https://github.com/ml-explore/mlx) 따라서 대부분의 배열 연산은 그대로 사용 가능합니다.

```python
# NumPy
import numpy as np
a = np.array([1.0, 2.0, 3.0])

# MLX (거의 동일)
import mlx.core as mx
a = mx.array([1.0, 2.0, 3.0])
```

단, NumPy는 bfloat16을 지원하지 않아서 변환 시 float32로 먼저 바꿔야 하는 예외가 있습니다. [Github](https://ml-explore.github.io/mlx/build/html/usage/numpy.html)

---

### 🟠 PyTorch → `mlx.nn` + `mlx.optimizers` (대체 가능)

`mlx.nn`과 `mlx.optimizers`는 PyTorch API를 밀접하게 따라 복잡한 모델 구축을 단순화합니다. [GitHub](https://github.com/ml-explore/mlx)

```python
# PyTorch
import torch.nn as nn
layer = nn.Linear(128, 64)

# MLX
import mlx.nn as nn
layer = nn.Linear(128, 64)  # 거의 동일
```

가장 큰 차이는 MLX는 통합 메모리를 자동으로 사용하지만, PyTorch는 디바이스를 명시적으로 지정해야 한다는 점입니다. [MetalCloud](https://metalcloud.space/blog/mlx-vs-pytorch-comparison/)

---

### 🔴 Pandas → **대체 라이브러리 없음**

MLX는 데이터 전처리/테이블 조작 도구가 없습니다. 그래서 실무에서는 이렇게 씁니다:

```python
import pandas as pd       # 데이터 로딩/전처리는 그대로 pandas 사용
import mlx.core as mx

df = pd.read_csv("data.csv")
X = mx.array(df.values)   # MLX 배열로 변환해서 학습
```

---

### 정리표

| 기존 라이브러리 | MLX 대체 | 비고 |
|---|---|---|
| `numpy` | `mlx.core` | 거의 동일한 API |
| `torch.nn` | `mlx.nn` | PyTorch 스타일 유지 |
| `torch.optim` | `mlx.optimizers` | 동일 패턴 |
| `pandas` | ❌ 없음 | pandas 그대로 사용 후 변환 |

---

**결론:** MLX는 numpy와 torch의 학습/추론 부분은 대체하지만, pandas는 데이터 전처리용으로 계속 함께 사용하는 구조입니다. Apple Silicon에 최적화되어 있어 Mac에서 LLM 파인튜닝이나 추론 시 특히 효율적입니다.
