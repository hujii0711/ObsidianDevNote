> 주제: Claude Code의 CLAUDE.md 파일과 시스템 프롬프트의 관계 정리

## 핵심 결론

**CLAUDE.md는 시스템 프롬프트 자체가 아닙니다.** 시스템 프롬프트에 **주입되는 추가 컨텍스트**입니다.

## 정확한 구조

| 실제 시스템 프롬프트 = 매 요청마다 전송                                                                                                                                                                                                                  |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1. Anthropic 기본 시스템 프롬프트<br>- Claude Code 역할 정의<br>- 도구 사용 규칙, 응답 스타일 등<br>(/context의 "System prompt" 7.1k)<br><br>2. 환경 정보<br>- 작업 디렉터리, OS, git 상태 등<br><br>3. CLAUDE.md 내용 (있을 경우)<br>- 프로젝트별 지침<br>- 사용자별 지침<br><br>4. 도구 정의 (Tools) |

## CLAUDE.md의 종류

| 위치                     | 범위                                | 우선순위       |
| ------------------------ | ----------------------------------- | -------------- |
| `~/.claude/CLAUDE.md`    | 모든 프로젝트에 적용 (사용자 글로벌) | 낮음           |
| `{프로젝트}/CLAUDE.md`   | 해당 프로젝트에만 적용              | 중간           |
| `{하위폴더}/CLAUDE.md`   | 해당 폴더 작업 시 적용              | 높음 (로컬 우선) |

## 시스템 프롬프트와의 차이점

| 항목            | 기본 시스템 프롬프트            | CLAUDE.md                   |
| --------------- | ------------------------------- | --------------------------- |
| 작성자          | Anthropic                       | 사용자/팀                   |
| 역할            | Claude Code의 **기본 동작 정의** | 프로젝트 **맥락 제공**      |
| 수정 가능       | 불가                            | 자유롭게 편집               |
| 위치            | Claude Code 내부                | 파일 시스템                 |
| 역할 우선순위   | 더 강함                         | 기본 규칙 위반 불가         |

## 예시로 이해

### 기본 시스템 프롬프트의 역할 (수정 불가)

- "너는 Claude Code다"
- "파일 수정 전에 Read 도구를 호출해라"
- "커밋은 사용자가 명시적으로 요청할 때만 해라"
- "위험한 명령은 사용자에게 확인받아라"

### CLAUDE.md의 역할 (프로젝트 지식 주입)

- "이 프로젝트는 React + TypeScript다"
- "빌드 명령어는 `npm run build`다"
- "코드 스타일은 Prettier + ESLint 따른다"
- "테스트는 Jest를 사용한다"

## 왜 시스템 프롬프트 "아님"인가?

1. **수정 권한의 차이**: 사용자가 CLAUDE.md를 바꿔도 Claude Code의 기본 행동(안전장치, 도구 사용 규칙)은 바뀌지 않음
2. **계층 구조**: CLAUDE.md는 시스템 프롬프트의 **지침 범위 안에서** 추가 정보 제공
3. **오버라이드 불가**: CLAUDE.md로 "위험한 명령을 확인 없이 실행하라"고 써도 무시됨

## 실용적 관점

CLAUDE.md는 **프로젝트별 영구 기억장치**로 보시면 됩니다:

- 매 세션마다 "이 프로젝트는 어떤 구조"를 설명할 필요 없음
- 커밋 메시지 스타일, 금지된 패턴 등을 자동으로 Claude에게 전달
- 팀원 모두가 같은 컨텍스트로 Claude Code 사용 가능

## 관계 요약

```
시스템 프롬프트 ⊃ CLAUDE.md

(시스템 프롬프트는 항상 포함되고, CLAUDE.md는 그 안에서 "프로젝트 컨텍스트"로 주입)
```

## 참고: CLAUDE.md 작성 가이드

CLAUDE.md에 포함하면 좋은 내용:

- **프로젝트 개요**: 어떤 프로젝트인지, 주요 기술 스택
- **디렉터리 구조**: 주요 폴더의 역할 설명
- **빌드/테스트 명령어**: `npm run build`, `npm test` 등
- **코드 컨벤션**: 네이밍, 파일 구조, 포맷팅 규칙
- **금지 사항**: 특정 API 사용 금지, 특정 디렉터리 수정 금지 등
- **도메인 지식**: 비즈니스 로직, 약어, 용어집

CLAUDE.md에 포함하지 말아야 할 내용:

- 보안 정보 (비밀번호, API 키 등)
- 자주 바뀌는 정보 (현재 진행 중인 작업 상태)
- 사용자 개인 정보
