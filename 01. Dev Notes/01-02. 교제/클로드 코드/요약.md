### 1. 클로드 코드 입문과 개발 환경 구축

##### (1) 클로드 코드란
클로드 코드는 CLI상에서 클로드에 지시를 내리면서 특정 디렉터리 아래의 워크스페이스를 검색, 설명, 코딩, 계획하고 사용자의 애플리케이션 개발을 지원하는 AI 코딩 에이전트이다.

##### (2) 클로드 코드의 특징
- 멀티파일: 프로젝트 전체를 이해하고, 여러 파일을 한 번에 편집
- 병렬 실행 능력: 여러 작업을 동시에 처리하여 개발 속도가 비약적으로 향상
- 자연어를 통한 대화형 개발: 기술적인 세부 사항을 알지 못해도 자연어로 지시하면 구현 가능
- 실행 환경과의 깊을 통합: 터미널에서의 명령 실행, 테스트의 자동 실행, 배포까지 일관되게 지원
- 환경에 의존하지 않음: 터미널에서 가볍게 실행되므로 다양한 에디터 및 환경에 통합 가능

##### (3) 사용량에 영향을 미치는 요인
- 메시지 길이
- 첨부 파일의 크기
- 현재 대화의 길이
- 도구 사용(리서치, 웹 검색)
- 모델 선택
cf.) sonnet 모델 100만 토큰 분량의 컨텍스트 윈도 유지 가능(800~900페이지 책 분량)

### 2. MCP를 활용한 AI 챗봇 개발

##### (1) MCP 서버란
MCP 서버란 클로드 코드와 다른 애플리케이션 간 상호 작용을 중개하는 서버이다.
MCP 서버를 구축하고 이를 통해 데이터를 주고 받아 LLM이 특정 정보를 획득하거나 애플리케이션을 제어할 수 있게 된다. 구체적으로 Git 작업, 데이터베이스 작업, 노션과의 연동, API 연동 등 다양한 작업이 가능하며, 클로드 코드의 기능을 대폭 확장할 수 있다.
##### (2) 스코프 설정하기
- local
기본 설정이며, 현재 사용자만 클로드 코드를 실행한 디렉터리 하위에서만 사용한다.

- project
프로젝트 단위에서 MCP를 사용하고 설정을 공유할 때 사용한다.
.mcp.json에 설정이 기재되어, Git에서 관리될 경우 프로젝트의 전체 사용자에서 MCP 설정이 적용된다.

- user
사용자 홈 디렉터리의 ~/.claude에 설정이 저장되므로 해당 사용자의 모든 프로젝트에 적용된다.

- 앞으로 진행하는 프로젝트에서 이 MCP 서버를 항상 사용할 수 있게 되어 언제든지 Context7을 호출
```
claude mcp add context7 -s user -- npx -y @upstash/context7-mcp
```

##### (3) 추천 MCP 서버
###### 1) Context7
Context7은 라이브러리의 최신 공식 문서를 참조하는 MCP 서버로, 기술 스택을 선정할 때 더 정확한 정보를 제공한다. 이를 통해 클로드가 작성하는 코드에서 오류가 줄어들고, 라이브러리 명세에 정확히 맞는 코드를 생성할 수 있다.

- mcp 서버 설치는 해당 프로젝트에만 설치되므로 다른 프로젝틍데서 사용하려면 다시 claude mcp add를 실행해야 한다.

- 설치 방법
```
claude mcp add context7 -- npx -y @upstash/context7-mcp
```

- 프롬프트 예시
```
Todo 앱을 Next.js의 App Router를 사용해서 만들어주세요. 데이터는 로컬에 저장해주세요. use context7
```
이렇게 지시를 함으로써 Next.js의 최신 공식 문서의 기술이나 샘플 코드를 참고하여 만들어주므로 라이브러리 사용의 정확도가 높아진다.

- CLAUDE.md에 정리된 기술 선정 내용을 참고하여 지시
```
현재 구성 내용으로 애플리케이션을 만들어주세요. use context7
```

###### 2) Playwright
클로드 코드와 연결함으로써 LLM이 마치 눈을 뜬 것처럼 브라우저를 제어하게 하거나 프런트엔드의 작동과 오류를 확인할 수 있다.

- 프롬프트 예시
```
Chrome 브라우저에서 화면 너비가 450px 이하일 때, 3열로 표시되고 있는 "ItemList"가 화면을 벗어나고 있다. playwright MCP로 확인하여 수정해주세요.
```
```
개발 서버를 시작하고 playwright MCP 서버를 사용해서 localhost:3000에 접근하세요.
오류가 나지 않고 정상적으로 채팅할 수 있을 때까지 수정하세요.
```
브라우저의 콘솔 에러 등을 playwright MCP 서버를 통해 얻을 수 있어 백엔드에서 드러나지 않는 에러까지 잡아낼 수 있다.
###### 3) Serena
Serena는 코드의 구조와 문맥에 관한 이해를 클로드에 제공해 코드 생성 품질을 높일 수 있는 MCP이다.
Serena는 IDE처럼 LSP를 지원하여, 클로드가 코드를 더 깊이 이해할 수 있도록 돕는다.
Serena가 LSP의 분석 정보를 클로드에 전달하면, 클로드가 IDE와 같은 방식으로 프로젝트 내의 코드를 이해할 수 있게 된다.
참고로 Serena는 LSP를 프로젝트 단위로 실행해야 하므로, -s 옵션을 사용해 사용자 스코프로 설치할 수 없고 프로젝트마다 MCP 서버를 따로 설치해야 한다.

### 3. AI 챗봇 요구사항 정의

##### (1) 개발 환경 구축 및 요구사항 정의
- 프로젝트에 사용할 디렉터리 생성하고 이동한 후 프롬프트 작성
```
AI 챗봇을 만들고 싶습니다. 명세를 함께 작성하고 싶으니 필요한 정보를 질문해 줄수 있나요?
최종적으로 CLAUDE.md 파일을 출력하는 것이 목적입니다.
```

##### (2) 태스크 관리와 구현
###### 1) 실행 계획 수립과 TODO 리스트 작성
프로젝트에서는 대략적인 지시만으로 클로드가 적절히 작동할 수 있지만, <font color="#ff0000">작업 단위를 적절히 분할</font>하고 사전에 계획하면 사용자와 LLM 모두 작업을 더 쉽게 처리할 수 있다.
특히 하나의 지시로 계속 작업하면 클로드가 알아야 할 내용이 점점 많아지고, LLM이 전체 내용을 컨텍스트에 다 담지 못해 결과의 정확도가 떨어질 수 있다. 이로 인해 클로드가 일부 작업을 누락하거나 제대로 완료하지 못하는 경우도 발생한다.
따라서 실행 계획을 먼저 TODO 리스트 형태로 마크다운 파일에 정리하면, 클로드가 더 정확하게 작업을 수행할 수 있다. 또한 사용자는 각 단계의 생성 품질과 진행 상황을 쉽게 관리할 수 있다.

- 실행 계획 프롬프트 지시
프로젝트 루트에 있는 CLAUDE.md는 반드시 매번 컨텍스트에 포함되는 명세이므로 "이 애플리케이션", "현재의 애플리케이션 구성" 같은 지시 내용으로 전달된다.
```
현재 애플리케이션을 구축하기 위한 실행 계획을 세워주세요. 이를 TODO 리스트로 Markdown 파일에 정리하세요.
```

- TODO.md를 참조하도록 프롬프트에 지시
```
@TODO.md 이를 바탕으로 1단계를 구현하세요. 완료되면 체크박스에 체크하세요. use context7
```
	라이브러리를 읽어 들일 때는 use context7를 프롬프트에 추가

- 체크 박스를 활용한 진척 관리
```
완료되면 체크박스에 체크하세요.
```
이렇게 추가로 지시하지 않으면 하나의 TODO를 클로드 코드가 구현하는 것으로 끝나버러셔 어느 단계까지 구현했는지 기록할 수 없다. 세션이 길어지면 클로드가 기억하지 못할 수도 있으므로 이렇게 외부 메모리를 활용하는 편이 더 정확한 결과를 얻는 데 도움이 된다.

###### 2) 작업 내용을 돌아보게 하기
작업을 한 번 끝냈더라도 LLM이 모든 것을 구현하지 않았거나 코드의 일관성이 높지 않은 경우도 있다.
그러한 경우를 대비하여 원래 계획했던 모든 작업이 끝난 후 다음과 같이 전체 점검을 행하는 것이 바람직하다.
```
현재 코드에서 구현이 부족한 부분을 확인해서 남은 작업을 @TODO.md에 추가해주세요. ultrathink
```
