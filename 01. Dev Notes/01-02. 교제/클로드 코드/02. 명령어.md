
MCP 서버가 포함 확인
```
claude mcp list
```

클로드를 실행할 때 명령줄 인자로 설정을 전달
```
claude config set <키> <값>
```

---
## 슬래시 명령어

### 세션관리
- <font color="#245bdb">/clear</font>
	대화 기록을 초기화한다. 새로운 주제로 대화ㄹ 시작하고 싶거나 컨텍스트를 초기화하고 싶을 때 사용한다.
	주제가 변경된 시점에 사용하면 토큰을 절약하고 컨텍스트를 명확하게 할 수 있다.
	
- <font color="#245bdb">/compact</font>
	대화를 압축한다. 선택적으로 압축할 때 예를 들어 무엇을 남길지 등의 지시를 추가할 수 있다.
	
- <font color="#245bdb">/rewind</font>: 현재 세션의 채팅 기록을 확인
	대화나 코드를 되돌린다. 잘못된 변경 사항을 취소하거나 다른 구현으로 변경하려는 경우에 사용할 수 있다.
	체크 포인트 기능으로 코드 변경 사항도 되돌릴 수 있다. 다만 사용자가 변경한 것이나 명령으 변경한 것은 되돌릴 수 없다. ESC 키를 2번 연속으로 눌러서 실행할 수도 있다.
	
- <font color="#245bdb">/context</font>
	현재 컨텍스트 윈도우의 점유 상태를 시각적으로 표시한다. 대화 내용뿐만 아니라 CLAUDE.md의 메모리 파일이나 MCP 서버 등에 의해 얼마나 점유되고 있는지 파악할 수 있다.
	
- <font color="#245bdb">/export</font>
	클로드의 출력을 포함해 현재 세션의 대화 내용을 클립보드나 파일로 내보낼 수 있다.
	중요한 정보를 저장하거나, 내보낸 내용을 참조해서 다른 클로드 코드의 세션에서 사용할 수도 있다.
	
- <font color="#245bdb">/resume</font>(continue)
	과거의 세션 목록에서 선택하여 재개한다. 중단한 작업으로 돌아가고 싶거나 이전 문맥을 이어받아 작업을 계속하고 싶을 때 사용한다.
	/clear를 실행해 컨텍스트를 삭제해버린 경우에도 삭제하기 전으로 돌아가고 싶을 때 사용한다.
	
- <font color="#245bdb">/exit</font>
	대화모드를 종료한다. 클로드 코드의 세션을 닫을 때 사용한다.(세션 종료)

### 계정 및 인증
- <font color="#245bdb">/login</font>
	엔트로픽의 계정을 전환한다. 클로드 코드를 처음 실행할 때나 여러 계정을 구분해서 사용할 때 유용한다.
	
- <font color="#245bdb">/logout</font>
	현재의 엔트로픽 계정에서 로그아웃한다. 구독 및 API를 전환하고 싶을 때 등에 사용한다.
	
- <font color="#245bdb">/upgrade</font>
	클로드 맥스 플랜으로 업그레이드할 수 있는 페이지로 이동한다. 더 높은 속도 제한이나 Opus 모델의 사용량을 늘리고 싶을 때 확인하면 된다. 이미 맥스 20x 플랜을 사용 중인 경우에는 그 이상의 플랜이 없으므로 불필요한다.

### 프로젝트 설정
- <font color="#245bdb">/add-dir</font>
	추가 작업 디렉터리를 지정한다. 여러 디렉터리에 걸쳐 있는 프로젝트에서 작업할 때 사용한다. 다만 설정이 유지되지 않으므로 세션마다 추가해야 한다.
	
- <font color="#245bdb">/init</font>
	프로젝트와 관련된 특정 규칙이나 지시 사항을 클로드 코드에 제공하기 위한 구성 파일인 CLAUDE.md를 생성한다. 클로드 코드에 무언가를 지시할 때마다 CLAUDE.md의 내용이 컨텍스트에 포함된다.
	
- <font color="#245bdb">/memory</font>
	CLAUDE.md 파일을 편집한다. 프로젝트의 컨텍스트와 제약 사항을 관리하고 변경할 수 있다.
	
- <font color="#245bdb">/config</font>
	설정 인터페이스(Config 탭)을 연다. 클로드 코드의 각종 설정을 확인하고 변경할 수 있다. 다크 모드 등의 테마도 이곳에서 변경할 수 있다.
	
- <font color="#245bdb">/permissions</font>
	권한 표시나 업데이트를 수행한다. 클로드 코드에 대한 특정 명령어 실행 허가 등을 관리하고 편집한다.
	도구별 허가/거부 규칙을 설정할 수 있다.
	
- <font color="#245bdb">/ide</font>
	IDE와의 연동 설정을 확인하거나 활성화/비활성화할 수 있다. IDE와의 연동을 통해 현재 열려 있는 파일을 컨텍스트에 포함시키거나 IDE상의 에러 메시지를 연동시킬 수 있다.
	
- <font color="#245bdb"> /terminal-setup</font>
	대화 모드에서 줄바꿈을 위한 Shift+Enter 키 바인딩을 설정한다. 이는 터미널에서의 조작성을 향상시킨다. 이 기능을 지원하는 환경에서 클로드 코드를 실행했을 때만 표시된다.

### 모델, AI 기능
- <font color="#245bdb">/model</font>
	사용할 AI 모델을 선택하고 변경한다. 작업에 따라 최적의 모델을 선택할 수 있다.
	
- <font color="#245bdb">/agents</font>
	AI 서브 에이전트를 관리한다. 전문적인 작업용 서브에이전트를 정의하고 실행할 수 있다.
	
- <font color="#245bdb">/vim</font>
	 Vim 모드를 켜고 끌 수 있다. Vim을 사용하는 프로그래머에게는 친숙한 키 바인딩으로 텍스트를 입력할 수 있다. INSERT 모드와 NORMAL 모드를 번갈아 전환하며 조작한다.

### 개발 지원
- <font color="#245bdb">/review</font>
	코드 리뷰를 요청한다. 구현한 코드의 품질 점검이나 개선 제안을 받을 수 있다.
	review PR#3과 같이 인자를 붙여서 명령을 실행할 수도 있다.
	
- <font color="#245bdb">/pr-comments</font>
	풀 리퀘스트의 코멘트를 표시한다. 깃허브와 연동할 때 편리하다. 또한 그곳에서 표시된 코멘트를 바탕으로 다시 지시를 내릴 수도 있다.
	
- <font color="#245bdb">/bashes</font>
	백그라운드에서 실행되고 있는 작업을 확인하고 중지할 수 있다. 예를 들어, 개발 서버를 실행하거나 시간이 오래 걸리는 작업을 클로드 코드로 수행하는 경우, 클로드 코드가 백그라운드로 보낸 후 계속 실행 중인 작업을 확인할 수 있다.(백그라운드로 보낸 Bash의 내용을 확인)
	
- <font color="#245bdb">/hooks</font>
	대화모드에서 커스텀 훅을 추가할 수 있다. 클로드 코드의 세션 시작이나 중지, 작동 시에 임의의 명령을 실행하도록 설정한다.
	
- <font color="#245bdb">/install-github-app</font>
	클로드 코드를 깃허브 액션에서 사용하기 위한 애플리케이션을 설치 및 설정한다.
	설정이 성공하면 풀 리퀘스트가 만들어지며, 병합을 하면 깃허브에서 @맨션으로 클로드 코드를 작동시킬수 있다.
	
- <font color="#245bdb">/security-review</font>
	현재 브랜치의 보류 중인 변경 사항에 대해 보안 리뷰를 수행한다. 취약점이나 잠재적 보안 위험을 감지하고 개선 제안을 받을 수 있다. 또한 인자에 풀 리퀘스트의 번호를 지정하면 해당 풀 리퀘스트에 대해 보안 리뷰를 수행한다.(코드를 커밋하기 전에 터미널에서 보안 분석을 실행)
	
- <font color="#245bdb">/mcp</font>
	MCP 서버 연결을 관리한다. MCP 서버의 상태를 확인하거나, 인증이 필요한 서버에 로그인하는 단계를 거칠 수 있다.
	
- <font color="#245bdb">/sandbox</font>
	이 명령어는 리눅스나 맥에서 호출할 수 있다. 클로드 코드의 구현을 네트워크와 파일 시스템이 분리된 샌드박스 내부에서 안전하게 수행할 수 있다.(샌드박스 모드로 클로드 실행)
	
- <font color="#245bdb">/update</font>
	구독 한도로 인한 플랜 전환
	

### 정보 표시
- <font color="#245bdb">/todos</font>
	TODO 리스트를 표시한다. 현재 클로드 코드는 큰 작업에 대해 자신이 TODO 리스트를 만들어 하나하나 실행하도록 설계돼 있습니다. 클로드 코드가 작업 중에 만든 작업 목록이나 미완료 작업을 확인할 수 있다.
	
- <font color="#245bdb">/help</font>
	도움말을 표시한다. 명령어의 사용법을 모를 때 참조할 수 있다.
	
- <font color="#245bdb">/status</font>
	설정 인터페이스(Status 탭)를 연다. 현재 상태와 설정을 확인할 수 있다. /config나 /usage에서 볼 수 있는 설정 화면이나 사용량도 이 명령어를 통해 확인할 수 있다.
	
- <font color="#245bdb">/cost</font>
	현재 세션의 토큰 사용량을 표시한다. API 사용량을 파악하고 싶을 때 유용하다. API 사용자만 사용할 수 있다.
	
- <font color="#245bdb">/usage</font>
	구독 플랜의 사용량을 표시한다. 남아 있는 사용량도 확인할 수 있다.
	
- <font color="#245bdb">/release-notes</font>
	클로드 코드의 릴리즈 노트를 표시한다. 브라우저가 열리며, 최신 기능, 변경 사항, 버그 수정 등의 정보를 확인할 수 있다.

### UI 출력 생성
-<font color="#245bdb"> /output-style</font>
	인자에 출력 스타일을 직접 설정하거나 선택 메뉴를 통해 선택한다. 기본값으로 Default, Explanatory(설명적), Learning(학습)의 3가지가 준비돼 있다. 클로드 코드의 응답 형식과 표시 방법을 설정할 수 있다.
	
- <font color="#245bdb">/output-style:new</font>
	사용자 정의 출력 스타일을 새로 생성한다. 고유한 출력 형식을 정의하는 경우에 사용한다. 스타일 정의를 위한 서브에이전트가 실행되며, 대화식으로 클로드 코드의 출력에서 무엇을 중지할 지, 어떻게 답변해 주길 원하는지 설정할 수 있다. 설정은 사용자 수준 또는 프로젝트 수준의 ~/.claude/output-styles/에 저장된다.
	
- <font color="#245bdb">/statusline</font>
	클로드 코드의 대화 모드에서 표시되는 상태 표시줄을 설정한다. 예를 들어, 현재의 컨텍스트 사용률과 같은 특정 정보를 항상 표시하는 등 표시 내용을 수정할 수 있다.

### 플러그인 및 개인 정보 보호
- <font color="#245bdb">/plugin</font>
	클로드 코드의 플러그인을 관리한다. 특정 마켓플레이스에서 플러그인의 설치, 삭제, 활성화/비활성화 등을 설정할 수 있다.
	
- <font color="#245bdb">/privacy-settings</font>
	브라우저가 열리고 계정의 개인정보 설정을 표시 및 업데이트 한다. 데이터 처리 및 공유에 관한 설정을 관리할 수 있다.

### 유지보수 및 문제 해결
- <font color="#245bdb"> /doctor</font>
	클로드 코드의 설치 상태를 확인한다. 문제 발생 시 원인을 진단하는 데 사용한다.
	
- <font color="#245bdb">/feedback</font>
	클로드 코드에 관한 피드백을 전송한다. 버그 보고나 기능 요청 등을 앤트로픽에 전달할 수 있다.
