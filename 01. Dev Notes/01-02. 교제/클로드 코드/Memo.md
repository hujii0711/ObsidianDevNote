- Sonnet은 모델명에 1m을 붙여 100만 토큰의 컨텍스트 윈도를 유지한다.
- ripgrep은 Rust로 제작된 고속 텍스트 검색 도구이다. 클로드 코드의 파일 탐색은 ripgrep의 정규 표현식 검색에 의존한다.
- gh 명령어를 사용하면 GitHub의 원격 레포지터리를 만들거나 관리할 수 있고, 이슈 생성, 풀 리퀘스트 확인, 리뷰 등도 브라우저를 열지 않고 터미널에서 바로 처리할 수 있다.
- 클로드는 gh 명령어만 있으면 원격 레포지토리 생성과 함께 GitHub Page에 배포까지 가능하다.
- @: 특정 파일을 지정해 컨텍스트에 포함하여 클로드에 질문하거나 대화할 수 있다.
- 사용자 루트: ~/.claude 디렉터리는 모든 클로드 프로젝트에 영향을 미친다. 클로드 코드를 처음 설치했을 때 이 사용자 루트에 .claude 디렉터리가 저장되었으며, 설정이나 기록 등이 저장되어 있다.
- 설정을 사용자 루트에 저장하면 모든 프로젝트에 적용되는 공통 설정이 되고, 프로젝트 루트에 저장하면 해당 프로젝트에만 적용된다. 
- 대화 모드에서 Tab 키를 누르는 것만으로 사고 확장 모드 기능이 추가된다.
- 정적 파일 기반 애플리케이션 배포 서비스: GitHub Page, Vercel, Netlify, Cloudflare Pages, Amplify Hosting, Firebase Hosting
- 클로드 코드는 CLAUDE.md를 매번 참조하므로, 이 파일에 프로젝트를 이해하는 데 도움이 되는 정보를 최대한 적어두면 클로드가 더 쉽게 필요한 정보를 얻을 수 있고, 토큰도 절약할 수 있다.
- 클로드 코드는 세션 단위로 대화를 기억한다. 클로드는 실행된 작업 디렉터리 단위로 세션을 기록하여, $HOME/.claude/projects 디렉터리에 저장한다.
- makefile: 프로그램의 빌드를 자동화하기 위한 설정 파일이다. 확장자 없는 Makefile에 bash 기반의 명령을 정의할 수 있다. init, dev, build, deploy 등의 명령 모음이다.
- 프로젝트 관리 명령어들은 클로드가 다른 세션에서 잊을 수 있으므로, CLAUDE.md 파일에 기록해두는 것이 좋다.
- 머메이드 표기법: ER 다이어그램 같은 마크다운이더라도 머메이드(Mermaid) 표기법으로 작성돼 있으면 여러모로 편리하다. 머메이드 표기법이란 마크다운 파일 내에서 텍스트를 사용해 다이어그램을 작성할 수 있는 표기법으로, 시퀀스 다이어그램이나 플로차트, 클래스 다이어그램이나 플로 다이어그램 등의 시각적 표현을 이해하기 쉬운 텍스트로 기술함으로써 리포지토리에서 코드로 관리할 수 있다는 장점이 있다.
