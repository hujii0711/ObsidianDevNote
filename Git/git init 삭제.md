
Git 저장소를 삭제하는 방법은 아주 간단합니다. Git은 프로젝트 폴더 내의 `.git`이라는 **숨김 디렉토리**에 모든 이력과 설정을 저장하기 때문에, 이 폴더만 삭제하면 일반 폴더로 되돌아갑니다.

### 1. 터미널(커맨드 라인)에서 삭제하기

가장 빠르고 확실한 방법입니다. 해당 프로젝트 폴더 경로에서 아래 명령어를 입력하세요.

- **macOS / Linux / WSL:**
    
    Bash
    
    ```
    rm -rf .git
    ```
    
- **Windows (Command Prompt):**
    
    DOS
    
    ```
    rd /s /q .git
    ```
    
- **Windows (PowerShell):**
    
    PowerShell
    
    ```
    rm -Force .git
    ```
    

---

### 2. 탐색기에서 직접 삭제하기 (GUI)

명령어 입력이 번거롭다면 직접 폴더를 지울 수도 있습니다. 다만, `.git`은 **숨김 폴더**이므로 설정을 변경해야 보입니다.

1. 해당 프로젝트 폴더로 이동합니다.
    
2. **숨김 항목 표시**를 활성화합니다.
    
    - **Windows:** 탐색기 상단 [보기] -> [표시] -> [숨긴 항목] 체크
        
    - **macOS:** `Command` + `Shift` + `.` (마침표) 입력
        
3. 회색조로 나타나는 `.git` 폴더를 찾아 **삭제**합니다.
    

---

### 💡 주의사항 및 팁

- **이력 삭제:** 이 작업을 수행하면 그동안 쌓아온 모든 커밋 히스토리(버전 기록)가 사라집니다. 소스 코드 파일 자체는 그대로 남지만, 과거로 되돌리는 기능은 사용할 수 없게 됩니다.
    
- **확인 방법:** 삭제 후 `git status` 명령어를 입력했을 때 `fatal: not a git repository`라는 메시지가 나오면 성공적으로 삭제된 것입니다.
    
- **VS Code 사용 시:** VS Code에서 해당 폴더를 열고 있다면, `.git` 삭제 직후 소스 제어 탭의 Git 아이콘이 사라지는 것을 바로 확인할 수 있습니다.
