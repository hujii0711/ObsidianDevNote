# 학습 가이드 06: 법률 도메인 기초 지식

> 법률 상담 AI를 만들기 위해 개발자가 알아야 할 최소한의 민사법 기초 및 법률 AI 윤리

---

## 1. 왜 개발자가 법률 지식이 필요한가?

```
법률 지식 없이 개발하면 발생하는 문제:

1. 데이터 품질 판단 불가
   → 수집한 Q&A가 정확한 답변인지 검증할 수 없음

2. 잘못된 청킹
   → "제750조 (불법행위)" 를 "제750조 (불법행"과 "위)"로 자를 수 있음

3. 테스트 불가
   → 모델 응답이 올바른지 판단할 수 없음

4. 위험한 서비스
   → 모델이 위험한 법적 조언을 해도 인지하지 못함

최소한의 법률 구조와 용어를 이해해야 합니다.
```

## 2. 대한민국 법률 체계 기초

### 2.1 법의 분류

```
공법 (국가 ↔ 개인)
├── 헌법
├── 행정법
└── 형법/형사소송법     ← 이 프로젝트 범위 밖

사법 (개인 ↔ 개인)     ← 이 프로젝트 범위
├── 민법                ← 핵심
├── 상법
└── 민사소송법          ← 핵심
```

### 2.2 민법의 구조

```
민법 (총 1,118조)
├── 제1편 총칙          → 법률행위, 대리, 소멸시효
├── 제2편 물권          → 소유권, 저당권, 점유권
├── 제3편 채권          → 계약, 불법행위, 손해배상  ← 상담 빈도 최고
├── 제4편 친족          → 혼인, 이혼, 친권
└── 제5편 상속          → 상속, 유언, 유류분
```

### 2.3 자주 다루는 민사 분야

| 분야 | 핵심 법률 | 자주 묻는 질문 |
|------|----------|-------------|
| **임대차** | 주택임대차보호법 | 보증금 반환, 계약 갱신, 퇴거 |
| **손해배상** | 민법 제750조 | 교통사고, 의료사고 배상 |
| **계약** | 민법 제527조~ | 계약 해제/해지, 위약금 |
| **채권추심** | 민법, 민사집행법 | 빌려준 돈 받기, 강제집행 |
| **이혼** | 민법 제834조~ | 재산분할, 양육권, 위자료 |
| **상속** | 민법 제997조~ | 상속 순위, 유류분, 한정승인 |
| **소비자** | 소비자기본법 | 환불, 교환, 피해구제 |

## 3. 반드시 알아야 할 법률 용어

### 3.1 기본 용어

| 용어 | 의미 | 개발 시 중요한 이유 |
|------|------|---------------------|
| **원고** | 소송을 제기한 사람 | 데이터에서 당사자 식별 |
| **피고** | 소송을 당한 사람 | 데이터에서 당사자 식별 |
| **소멸시효** | 권리를 행사하지 않으면 소멸하는 기간 | 매우 빈번한 질문 |
| **내용증명** | 우체국을 통한 공식 서면 통보 | 절차 안내에 자주 포함 |
| **가처분** | 소송 전 임시 조치 | 긴급 상담에 등장 |
| **판례** | 법원의 재판 선례 | RAG 검색 대상 |
| **조문** | 법률의 개별 조항 | RAG 검색 대상 |

### 3.2 소송 절차 용어

| 단계 | 용어 | 설명 |
|------|------|------|
| 1 | 소장 | 소송을 시작하는 문서 |
| 2 | 답변서 | 피고가 제출하는 반박 문서 |
| 3 | 변론 | 법정에서의 공판 절차 |
| 4 | 판결 | 법원의 최종 결정 |
| 5 | 항소 | 판결에 불복하여 상급법원에 다시 재판 요청 |
| 6 | 강제집행 | 판결을 강제로 이행시키는 절차 |

### 3.3 금액/기간 관련

```
알아두면 데이터 검증에 도움:
- 소액사건: 3,000만원 이하 → 간이절차 (1회 변론)
- 일반 소멸시효: 10년 (민법), 채권은 보통 3~10년
- 임대차 계약갱신청구권: 최초 2년 + 갱신 (최대 10년)
- 이자제한법 최고이자율: 연 20%
```

## 4. 법률 문서 구조 이해

### 4.1 법조문 읽는 법

```
제750조(불법행위의 내용) 고의 또는 과실로 인한 위법행위로
타인에게 손해를 가한 자는 그 손해를 배상할 책임이 있다.

구조:
├── "제750조"        → 조 번호
├── "(불법행위의 내용)" → 제목 (괄호 안)
└── "고의 또는..."    → 본문

관련 표현:
├── 제750조           → "조" (기본 단위)
├── 제750조 ①        → "항" (조 내 세부 구분)
├── 제750조 ① 1.     → "호" (항 내 나열)
└── 제750조의2        → "조의2" (추가 신설 조항)
```

### 4.2 판례 읽는 법

```
대법원 2023. 5. 15. 선고 2023다12345 판결

├── "대법원"           → 법원명
├── "2023. 5. 15."    → 선고 날짜
├── "선고"            → 판결 선언
├── "2023다12345"     → 사건번호
│   ├── "2023"        → 접수 연도
│   ├── "다"          → 사건 종류 (다=민사상고)
│   └── "12345"       → 사건 번호
└── "판결"            → 재판 유형

사건 종류 약어:
  가단 = 민사 단독 1심
  가합 = 민사 합의 1심
  나   = 민사 항소
  다   = 민사 상고 (대법원)
```

> 이 구조를 알아야 판례 데이터의 메타데이터를 올바르게 파싱할 수 있습니다.

## 5. 법률 AI 윤리 및 규제

### 5.1 반드시 지켜야 할 원칙

```
1. 면책 조항 필수
   "본 답변은 참고용이며, 구체적 법률 조언은 변호사와 상담하시기 바랍니다."
   → 모든 응답에 자동 포함 (이 프로젝트에 구현됨)

2. 변호사법 위반 주의
   → 변호사가 아닌 자의 법률사무 취급은 변호사법 위반
   → AI 서비스는 "법률 정보 제공"이지 "법률 자문"이 아님
   → "~해야 합니다" 대신 "~할 수 있습니다" 표현 사용

3. 확실하지 않으면 답하지 않기
   → "관련 자료를 찾지 못했습니다" 응답
   → 추측 금지 (이 프로젝트 CLAUDE.md에 명시)

4. 개인정보 보호
   → 상담 내용에 포함된 개인정보를 로그에 기록하지 않음
   → 데이터 로컬 처리 원칙
```

### 5.2 AI 법률 서비스의 법적 지위

```
현재 대한민국 법률에서:
- AI 법률 서비스에 대한 명시적 규정은 아직 없음
- 법률 "정보" 제공은 허용 (법률 신문, 법률 포털 등과 유사)
- 법률 "자문"은 변호사만 가능 (변호사법 제109조)
- 경계가 모호하므로 보수적 접근 필요

안전한 표현:
  ✅ "~할 수 있습니다", "~이 가능합니다"
  ✅ "일반적으로 ~로 알려져 있습니다"
  ✅ "관련 법조문은 ~입니다"
  
위험한 표현:
  ❌ "~해야 합니다" (의무 부과)
  ❌ "귀하의 경우 ~입니다" (개별 자문)
  ❌ "승소 가능성이 높습니다" (결과 예측)
```
