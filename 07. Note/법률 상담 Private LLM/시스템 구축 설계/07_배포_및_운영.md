## 1. 로컬 실행 (개발/테스트)

### 1.1 API 서버 실행

```bash

source .venv/bin/activate

# config.yaml에 모델 경로 확인 (병합 모델 사용 시)
```

## 2. Docker 배포

### 2.1 이미지 빌드

```bash
# 전체 서비스 빌드
docker compose build

# 개별 빌드
docker compose build api
docker compose build ui
```

### 2.2 컨테이너 실행

```bash
# 전체 서비스 실행
docker compose up -d

# 로그 확인
docker compose logs -f api
docker compose logs -f ui

# 상태 확인
docker compose ps
```

### 2.3 볼륨 마운트 구조

```yaml
# docker-compose.yml 볼륨 설명
volumes:
  - ./data:/app/data          # 벡터DB, SQLite 데이터
  - ./models:/app/models      # 모델 파일
  - ./config.yaml:/app/config.yaml  # 설정 파일
```

> **주의**: MLX는 Apple Silicon 전용이므로, Docker 내에서 MLX 추론은 macOS Docker에서만 가능합니다. Linux 배포 시 llama.cpp 폴백을 사용해야 합니다.

### 2.4 서비스 중지

```bash
docker compose down
```

### 3.2 로그 관리

**API 서버 로그 확인 사항:**


**로그 파일 출력 설정:**

```bash

uvicorn src.api.main:app --host 0.0.0.0 --port 8000 2>&1 | tee logs/api.log
```

## 4. 데이터 백업

### 4.1 백업 대상

| 대상 | 경로 | 주기 |
|------|------|------|
| 대화 이력 DB | `data/legal_llm.db` | 매일 |
| 벡터 DB | `data/vectordb/` | 인덱싱 후 |
| LoRA 어댑터 | `models/adapters/` | 학습 후 |
| 병합 모델 | `models/fused/` | 학습 후 |
| 설정 파일 | `config.yaml` | 변경 시 |

### 4.2 백업 스크립트

```bash
#!/bin/bash
# scripts/backup.sh
BACKUP_DIR="backups/$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"

# SQLite 백업
cp data/legal_llm.db "$BACKUP_DIR/" 2>/dev/null

# 설정 백업
cp config.yaml "$BACKUP_DIR/"

# 어댑터 백업 (작은 크기)
cp -r models/adapters "$BACKUP_DIR/" 2>/dev/null

echo "백업 완료: $BACKUP_DIR"
```

## 7. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 서버 시작 시 모델 로딩 실패 | 모델 파일 누락 또는 경로 오류 | config.yaml의 model.name 확인 |
| 응답이 매우 느림 (>60초) | 메모리 스왑 발�� | 불필요 앱 종료, 메모리 확보 |
| RAG 검색 결과 없음 | 벡터DB 미구축 | `python -m src.rag.indexer` 실행 |
| Streamlit 연결 실패 | API ���버 미실행 | API 서버 먼저 실행 확인 |
| SQLite lock 에러 | 동시 쓰기 충돌 | aiosqlite WAL 모드 활성화 |
| Docker에서 MLX 실패 | Linux 컨테이너 | MLX는 macOS만 지원, llama.cpp 사용 |
