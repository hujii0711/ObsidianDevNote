# 06. 파인튜닝 실행

## 1. QLoRA 개요

### 왜 QLoRA인가?

| 방법 | VRAM 요구량 | 10.8B 모델 가능 여부 (48GB) |
|------|-----------|--------------------------|
| Full Fine-tuning | ~90GB | 불가 |
| LoRA | ~30GB | 가능 (여유 적음) |
| **QLoRA** | **~16GB** | **가능 (여유 충분)** |

QLoRA = 4bit 양자화 모델 + LoRA 어댑터 학습
- 베이스 모델은 Q4로 고정 (학습하지 않음)
- 소규모 LoRA 어댑터만 학습 (전체 파라미터의 ~1%)
- M4 48GB에서 안정적으로 학습 가능

## 3. 하이퍼파라미터 설정

### 3.1 기본 설정 (config.yaml)

```yaml
training:
  lora_rank: 64          # LoRA 행렬 랭크
  lora_alpha: 128        # LoRA 스케일링 팩터
  target_modules:        # LoRA 적용 대상 레이어
    - q_proj
    - v_proj
    - k_proj
    - o_proj
  batch_size: 4          # 미니배치 크기
  gradient_accumulation_steps: 8   # 그래디언트 누적
  learning_rate: 0.0002  # 학습률
  epochs: 3              # 에폭 수
  lora_layers: 16        # LoRA 적용 레이어 수
  max_iters: 1000        # 최대 반복 횟수
```

### 3.2 파라미터 설명

| 파라미터 | 값 | 조정 가이드 |
|---------|-----|-----------|
| `lora_rank` | 64 | 낮추면 학습 빠르지만 성능 저하 (최소 16) |
| `lora_alpha` | 128 | 보통 rank의 2배로 설정 |
| `batch_size` | 4 | 메모리 부족 시 2로 줄이고 grad_accum 증가 |
| `learning_rate` | 2e-4 | 과적합 시 1e-4로 낮춤 |
| `lora_layers` | 16 | 증가 시 성능 향상하나 메모리 증가 |
| `max_iters` | 1000 | 데이터 규모에 비례하여 조정 |

### 3.3 데이터 규모별 권장 반복 횟수

| 학습 데이터 | max_iters | 예상 시간 (M4) |
|-----------|-----------|--------------|
| 1,000건 | 500 | ~2시간 |
| 5,000건 | 1,000 | ~5시간 |
| 10,000건 | 2,000 | ~10시간 |
| 20,000건 | 3,000 | ~15시간 |

## 4. 학습 실행

### 4.1 명령어

```bash
# 학습 + 모델 병합 (전체 파이프라인)
python -m src.training.train_lora --config config.yaml

# 또는 MLX-LM 직접 실행
mlx_lm.lora \
    --model mlx-community/EEVE-Korean-10.8B-4bit \
    --train \
    --data data/processed \
    --adapter-path models/adapters \
    --batch-size 4 \
    --lora-layers 16 \
    --iters 1000 \
    --learning-rate 2e-4
```

### 4.2 학습 중 모니터링

**터미널 출력 예시:**
```
Iter 100: Train loss 2.341, Learning Rate 2.000e-04, It/sec 1.23
Iter 200: Train loss 1.876, Learning Rate 2.000e-04, It/sec 1.21
Iter 300: Train loss 1.543, Learning Rate 2.000e-04, It/sec 1.22
...
Iter 1000: Train loss 0.892, Learning Rate 2.000e-04, It/sec 1.20
```

**정상 학습 판단 기준:**
```
✅ Train loss가 지속적으로 감소
✅ It/sec가 안정적 (큰 변동 없음)
✅ 메모리 사용량이 안정적

❌ Train loss가 감소하지 않음 → learning_rate 조정
❌ Train loss가 급증 → learning_rate를 낮추거나 데이터 확인
❌ OOM 에러 → batch_size 감소
```

## 6. 학습 결과 평가

### 6.1 정성적 평가

```python
from mlx_lm import load, generate

# 병합 모델 로드
model, tokenizer = load("models/fused")

# 테스트 질문
test_questions = [
    "전세 보증금을 돌려받지 못하면 어떻게 해야 하나요?",
    "교통사고 합의금은 어떻게 산정하나요?",
    "소멸시효가 지난 채권도 받을 수 있나요?",
    "이혼 시 재산분할 기준은 무엇인가요?",
    "내용증명을 보내면 법적 효력이 있나요?",
]

for q in test_questions:
    answer = generate(model, tokenizer, prompt=q, max_tokens=300, temp=0.3)
    print(f"Q: {q}")
    print(f"A: {answer}\n{'='*60}\n")
```

### 6.2 평가 기준

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 정확성 | 법률적으로 정확한 정보인가? | 40% |
| 완결성 | 필요한 정보를 빠짐없이 포함하는가? | 25% |
| 이해도 | 일반인이 이해할 수 있는 표현인가? | 20% |
| 형식 | 단계별 안내, 구조화된 답변인가? | 10% |
| 안전성 | 위험한 조언을 하지 않는가? | 5% |

### 6.3 Before/After 비교

```
베이스 모델(학습 전)과 파인튜닝 모델(학습 후)을 동일한 질문으로 비교:

1. 법률 용어를 정확히 사용하는가?
2. 답변 형식이 일관적인가? (단계별 안내 등)
3. 한국 법률 체계에 맞는 답변인가?
4. 불필요한 영어/외국법 참조가 줄었는가?
```

## 7. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `Out of Memory` | 메모리 부족 | batch_size 2로 줄이기, lora_layers 8로 줄이기 |
| Loss가 감소하지 않음 | learning_rate 부적절 | 1e-4 ~ 5e-4 범위에서 조정 |
| Loss가 0에 수렴 | 과적합 | epochs 줄이기, 데이터 증강 |
| 학습 후 한국어 깨짐 | 토크나이저 불일치 | 베이스 모델과 같은 토크나이저 사용 확인 |
| 병합 후 모델 동작 이상 | 어댑터 호환성 | adapter_config.json의 base_model 경로 확인 |
