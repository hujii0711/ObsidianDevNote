# 01. 개발 환경 구축

## 1. 하드웨어 요구사항

### 최소 사양 (학습 + 추론)

| 항목 | 사양 | 비고 |
|------|------|------|
| CPU/SoC | Apple M4 이상 | Apple Silicon 필수 (MLX) |
| 통합 메모리 | 48GB 이상 | 10.8B Q4 모델 + QLoRA 학습 기준 |
| 저장공간 | 100GB 여유 | 모델(~8GB) + 데이터 + 벡터DB |
| OS | macOS 14 Sonoma 이상 | MLX 지원 최소 버전 |

### 메모리 사용 예상치

```
베이스 모델 (Q4)      : ~8 GB
임베딩 모델 (BGE-M3)  : ~2 GB
리랭킹 모델           : ~1 GB
ChromaDB              : ~1-3 GB (문서 규모에 따라)
QLoRA 학습 오버헤드   : ~8-12 GB
OS 및 기타            : ~5 GB
─────────────────────────────
총 예상               : ~25-35 GB (48GB 내 안전)
```

> 추론 전용 서버로 운영 시 32GB 이상이면 충분합니다.

## 2. 소프트웨어 설치

### 2.1 Python 환경

```bash
# Homebrew로 Python 3.11 설치
brew install python@3.11

# 프로젝트 디렉터리 이동
cd /path/to/LawLLM

# 가상환경 생성 및 활성화
python3.11 -m venv .venv
source .venv/bin/activate

# pip 업그레이드
pip install --upgrade pip
```

### 2.2 핵심 의존성 설치

```bash
# 전체 의존성 일괄 설치
pip install -r requirements.txt
```

**의존성이 설치되지 않을 경우 단계별 설치:**

```bash
# 1단계: MLX (Apple Silicon 추론/학습)
pip install mlx mlx-lm

# 2단계: RAG 파이프라인
pip install llama-index llama-index-vector-stores-chroma llama-index-embeddings-huggingface
pip install chromadb sentence-transformers

# 3단계: API 서버
pip install fastapi uvicorn[standard] sse-starlette aiosqlite

# 4단계: UI
pip install streamlit

# 5단계: 개발 도구
pip install pytest pytest-asyncio httpx ruff
```

### 2.3 Xcode Command Line Tools

MLX 빌드에 필요합니다:

```bash
xcode-select --install
```

### 2.4 Git 설정

```bash
git init
git add .gitignore
git commit -m "chore: initial project setup"
```

## 4. 설치 검증

### 4.1 MLX 동작 확인

```python
# python -c 로 실행
import mlx.core as mx
a = mx.array([1, 2, 3])
print(f"MLX 버전: {mx.__version__}")
print(f"디바이스: {mx.default_device()}")  # gpu 출력 확인
print(f"연산 결과: {a + 1}")
```

예상 출력:
```
MLX 버전: 0.18.x
디바이스: Device(gpu, 0)
연산 결과: array([2, 3, 4], dtype=int32)
```

### 4.2 LlamaIndex + ChromaDB 확인

```python
import chromadb
from llama_index.core import Document

client = chromadb.Client()
collection = client.create_collection("test")
print(f"ChromaDB 정상: {collection.name}")
print(f"LlamaIndex Document 정상: {Document(text='test')}")
```

### 4.3 FastAPI 서버 기동 테스트

```bash
# 모델 없이 서버만 기동 확인
uvicorn src.api.main:app --host 0.0.0.0 --port 8000

# 별도 터미널에서 헬스체크
curl http://localhost:8000/api/health
```

예상 응답:
```json
{"status":"ok","model_loaded":false,"rag_index_ready":false}
```

### 4.4 테스트 스위트 실행

```bash
pytest tests/ -v
```

## 6. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `mlx` import 실패 | Intel Mac 또는 Linux | MLX는 Apple Silicon 전용. llama.cpp 폴백 사용 |
| `chromadb` 설치 오류 | sqlite3 버전 이슈 | `brew install sqlite3` 후 재설치 |
| `pip install` 중 빌드 실패 | Xcode CLI 미설치 | `xcode-select --install` 실행 |
| 메모리 부족 경고 | 모델 + 데이터 동시 로드 | 학습과 서빙을 동시에 하지 않도록 분리 |
| `uvicorn` 포트 충돌 | 8000번 포트 사용 중 | `lsof -i :8000`으로 확인 후 프로세스 종료 |
