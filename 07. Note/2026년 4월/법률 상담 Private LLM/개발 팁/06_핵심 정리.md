#### AI 주도 개발이란?

```
기존 개발:
  개발자가 코드 작성 → AI가 보조 (자동완성, 리뷰)

AI 주도 개발:
  AI가 설계/구현/테스트 주도 → 개발자가 의사결정/검증/피드백
  
Claude Code에서의 실현:
  ┌─────────────────────────────────────────────────┐
  │  CLAUDE.md (프로젝트 규칙)                         │
  │  + MCP (외부 시스템 연동)                           │ 
  │  + Skills (반복 작업 자동화)                        │
  │  + Hooks (이벤트 기반 자동 실행)                     │
  │  + Sub-agents (병렬 작업 분산)                     │
  │  ─────────────────────────────────────          │
  │  = 개발자는 "무엇을" 지시, AI가 "어떻게" 실행            │
  └─────────────────────────────────────────────────┘
```

#### 하네스 유지보수

```
정기 점검 (2주마다):
  1. 더 이상 유효하지 않은 규칙 제거
  2. 자주 반복하는 지시를 규칙으로 승격
  3. 사용하지 않는 스킬 정리
  4. 훅 동작 확인
  5. MCP 서버 연결 상태 확인

규칙 승격 기준:
  같은 지시를 3회 이상 반복 → CLAUDE.md 규칙으로 추가
  같은 작업을 3회 이상 수동 실행 → 스킬로 자동화
  같은 검증을 매번 수동 수행 → 훅으로 자동화
```

#### 점진적 하네스 구축

```
Week 1: 기초
  ├── CLAUDE.md (핵심 규칙 20개)
  ├── .claude/settings.json (permissions + afterEdit 훅)

Week 2: 확장

Week 3: 고도화
  ├── 서브 에이전트 3개 추가
  ├── /deploy-check, /rag-test 스킬 추가
  ├── .claude/docs/ 참조 문서 4개
  └── 규칙 효과 검증 및 튜닝

Week 4: 최적화
  ├── 사용하지 않는 규칙 제거
  ├── 자주 반복하는 지시를 규칙으로 승격
  ├── 스킬/훅 조합 최적화
  └── 팀원 피드백 반영
```

#### MCP란?
<font color="#ff0000">Claude Code가 외부 시스템의 데이터를 읽고 조작할 수 있게 해주는 프로토콜</font>입니다.
MCP 서버를 연결하면 Claude가 GitHub 이슈 확인, DB 조회, 웹 검색 등을 직접 수행합니다.

MCP 조합 시나리오

```
"이번 주 GitHub 이슈 중 버그를 확인하고, 관련 코드를 수정해서 PR 올려줘"

Claude 동작:
  1. [GitHub MCP] 이슈 목록 조회 → bug 라벨 필터
  2. [Read/Edit]  관련 소스 코드 분석 및 수정
  3. [Bash]       테스트 실행 확인
  4. [GitHub MCP] PR 생성 + 이슈 링크

"피드백 DB에서 별점 1~2점인 응답을 분석하고, 프롬프트를 개선해줘"

Claude 동작:
  1. [SQLite MCP] 낮은 평점 피드백 조회
  2. [Read]       현재 prompt.py 분석
  3. [Edit]       프롬프트 템플릿 개선
  4. [Bash]       테스트 실행
```

#### 스킬이란?
`.claude/commands/` 디렉터리에 마크다운 파일로 정의하는 재사용 가능한 명령입니다.
`/명령이름`으로 실행하면 Claude가 해당 프롬프트를 따릅니다.

| /test — 테스트 실행 및 분석<br>/lint — 린트 및 자동 수정<br>/data-add — 학습 데이터 추가<br>/rag-test — RAG 검색 품질 테스트<br>/deploy-check — 배포 전 점검<br>/api-test — API 엔드포인트 테스트<br>/experiment — 모델/파라미터 실험 |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

#### 훅이란?

Claude Code의 특정 이벤트(도구 실행 전후, 알림 등)에 자동으로 실행되는 셸 명령입니다.
`.claude/settings.json`에 설정합니다.

> 권장하는 훅

	(1) 파일 수정 후 자동 린트
	**효과:** Claude가 Python 파일을 수정할 때마다 자동으로 린트 + 포맷 적용
	
	(2) 커밋 전 테스트 실행
	**효과:** Claude가 커밋하기 전에 테스트가 통과하는지 자동 확인. 실패 시 커밋 차단
	
	(3) PR 생성 전 배포 점검
	
	(4) 프롬프트 제출 시 컨텍스트 주입

> 훅 조합 전략

```
개발 중 자동화 흐름:

  Claude가 코드 수정
       │
       ▼
  [afterEdit 훅] 자동 ruff 린트/포맷
       │
       ▼
  Claude가 커밋 시도
       │
       ▼
  [preCommit 훅] pytest 자동 실행
       │
    Pass ──→ 커밋 성공
    Fail ──→ Claude가 에러 확인 → 코드 수정 → 재시도
```


#### 하네스 엔지니어링이란?

> 정의

```
하네스 엔지니어링:    AI 에이전트가 동작하는 전체 환경을 설계

하네스 = CLAUDE.md + settings.json + MCP + Skills + Hooks + Agents + docs/
        ───────────────────────────────────────────────────────────────
        이 모든 것이 Claude Code가 "어떻게 행동할지"를 결정하는 하네스
```

> 하네스 구성 요소 전체 맵

```
~/.claude/                          ← 글로벌 (모든 프로젝트)
├── settings.json                   ← 사용자 설정
└── CLAUDE.md                       ← 개인 작업 스타일

프로젝트 루트/
├── .claude/
│   ├── settings.json               ← 프로젝트 설정 (공유)
│   ├── settings.local.json         ← 로컬 설정 (비공유)
│   ├── CLAUDE.md                   ← 프로젝트 규칙 (핵심)
│   ├── docs/                       ← 참조 문서
│   │   ├── API_SCHEMA.md
│   │   ├── DATA_FORMAT.md
│   │   └── TESTING_POLICY.md
│   ├── commands/                   ← 커스텀 스킬
│   │   ├── test.md
│   │   └── deploy-check.md
│   └── agents/                     ← 서브 에이전트
│       ├── code-reviewer.md
│       └── data-validator.md
├── .mcp.json                       ← MCP 서버 연결
├── CLAUDE.md                       ← 프로젝트 최상위 규칙
└── 하위 디렉터리/CLAUDE.md          ← 디렉터리별 규칙
```

#### CLAUDE.md 규칙의 5가지 유형

| 유형         | 설명         | 예시                                                                    |
| ---------- | ---------- | --------------------------------------------------------------------- |
| **금지 규칙**  | 절대 하지 말 것  | "LangChain 사용 금지", "Any 타입 금지"                                        |
| **필수 규칙**  | 반드시 할 것    | "타입 힌트 필수", "면책 조항 필수"                                                |
| **선호 규칙**  | 가능하면 따를 것  | "가능하면 컴프리헨션 사용"                                                       |
| **조건부 규칙** | 특정 상황에서 적용 | "FastAPI 핸들러는 반드시 async def"                                          |
| **참조 규칙**  | 외부 문서 참조   | <font color="#ff0000">"API 스키마는 .claude/docs/API_SCHEMA.md 참조"</font> |

#### 에이전트 정의 문법

```markdown
<!-- .claude/agents/에이전트이름.md -->
---
name: 에이전트 식별자
description: 한 줄 설명 (에이전트 선택 시 표시)
---

## 역할
당신은 ~을 전문으로 하는 에이전트입니다.

## 입력
- 어떤 데이터를 받는지

## 처리 규칙
1. 단계 1
2. 단계 2

## 출력 형식
- 어떤 형태로 결과를 보고하는지
```
  
#### 에이전트 호출 시점

```
메인 Claude가 서브 에이전트를 호출하는 시점:

자동 호출 (description에 트리거 조건 명시):
  - "코드를 수정했을 때 이 에이전트로 리뷰"
  - "데이터 파일을 추가했을 때 이 에이전트로 검증"

수동 호출 (사용자가 명시적으로 요청):
  - "code-reviewer 에이전트로 이 변경사항 리뷰해줘"
  - "data-validator로 이 JSONL 파일 검증해줘"
```

#### 에이전트 간 역할 분리

```
┌──────────────────┐
│   메인 Claude     │ ← 전체 작업 오케스트레이션
│   (지휘자 역할)   │
└─────┬────────────┘
      │
      ├──→ [code-reviewer]     코드 규칙 준수 검증
      ├──→ [data-validator]    데이터 품질 검증
      └──→ [rag-evaluator]    검색 품질 평가

각 에이전트는:
  - 독립된 컨텍스트에서 실행 (서로 간섭 없음)
  - 전문 분야에 집중
  - 결과만 메인 Claude에 보고
```
