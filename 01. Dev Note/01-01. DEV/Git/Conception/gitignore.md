이미 `git push`를 통해 원격 저장소에 올라간 파일(Tracked)은 단순히 `.gitignore`에 파일명을 추가하는 것만으로는 `untracked` 상태가 되지 않습니다. `.gitignore`는 **한 번도 추적된 적 없는 파일**에만 작동하기 때문입니다.

이미 추적 중인 파일을 관리 대상에서 제외하고 싶다면, **Git의 인덱스(캐시)에서 해당 파일을 제거**하는 과정이 필요합니다.

---

### 해결 방법: 캐시 삭제 후 커밋

터미널에서 다음 명령어를 순서대로 실행하세요.

#### 1. 특정 파일만 제외하고 싶을 때

Bash

```
git rm --cached <파일명>
```

#### 2. 특정 폴더를 제외하고 싶을 때

Bash

```
git rm -r --cached <폴더명>
```

#### 3. 전체 적용 (이미 `.gitignore`를 수정한 경우)

`.gitignore` 파일에 제외할 목록을 모두 적은 상태라면, 아래 명령어로 캐시를 초기화하고 다시 잡는 것이 편합니다.

Bash

```
git rm -r --cached .
git add .
git commit -m "gitignore 적용: 추적 대상에서 특정 파일 제외"
git push origin <브랜치명>
```

---

### 주요 명령어 설명

- **`--cached`**: 이 옵션이 핵심입니다. **실제 로컬의 물리적인 파일은 삭제하지 않고**, Git이 관리하는 목록(Index)에서만 해당 파일을 지우겠다는 의미입니다.
    
- **`.gitignore` 업데이트**: 캐시를 삭제한 후에는 반드시 `.gitignore` 파일에 해당 경로가 등록되어 있는지 확인해야 합니다. 그렇지 않으면 다음 `git add .` 시 다시 추적 대상이 됩니다.
    

### 요약하면

1. `.gitignore`에 파일 추가
    
2. `git rm --cached <파일명>` 실행
    
3. `commit` & `push`
    

이렇게 하면 내 컴퓨터에는 파일이 그대로 남아있지만, 원격 저장소(GitHub 등)에서는 파일이 삭제된 것으로 표시되며 앞으로는 추적되지 않습니다.
