Bash와 Zsh은 모두 유닉스 계열 시스템에서 가장 널리 쓰이는 셸(Shell)이지만, 최근의 트렌드는 **사용자 편의성**과 **확장성** 측면에서 Zsh로 크게 기울어 있는 편입니다.

두 셸의 핵심적인 차이점을 정리해 드립니다.

---

## 1. 개요 및 기본 설정

- **Bash (Bourne Again Shell):** 1989년에 출시된 유서 깊은 셸입니다. 대부분의 리눅스 배포판에서 **기본 셸**로 탑재되어 있어 "표준"으로 통합니다.
    
- **Zsh (Z Shell):** ==Bash의 기능을 모두 포함하면서도 훨씬 강력한 기능을 추가한 셸==입니다. 2019년부터 **macOS의 기본 셸**로 채택되었습니다.
    

---

## 2. 주요 차이점 비교

| **기능**      | **Bash**                  | **Zsh**                                 |
| ----------- | ------------------------- | --------------------------------------- |
| **자동 완성**   | 단순 파일명/경로 완성              | 대소문자 구분 없음, 화살표 키로 선택 가능 (강력함)          |
| **맞춤법 교정**  | 지원하지 않음                   | 명령어 오타 시 `Did you mean...?` 제안          |
| **플러그인/테마** | 직접 설정해야 하며 제한적임           | **Oh My Zsh** 등을 통해 수천 개의 테마/플러그인 사용 가능 |
| **경로 이동**   | `cd /usr/local/bin` 전체 입력 | `cd /u/l/b`만 쳐도 경로 유추 및 이동 가능           |
| **프롬프트 설정** | 설정이 까다로움                  | Git 브랜치 상태 등을 실시간으로 보여주는 동적 프롬프트 지원     |

---

## 3. Zsh가 더 사랑받는 이유 (강점)

### ① 강력한 자동 완성 (Tab Completion)

Bash는 `Tab`을 누르면 단순히 리스트만 보여주지만, Zsh는 `Tab`을 한 번 더 누르면 화살표 키로 원하는 항목을 선택할 수 있는 메뉴 모드로 진입합니다.

### ② Oh My Zsh 프레임워크

Zsh의 진정한 힘은 **Oh My Zsh**라는 커뮤니티 프레임워크에서 나옵니다. 이를 설치하면 다음과 같은 기능을 바로 쓸 수 있습니다.

- **Syntax Highlighting:** 유효한 명령어는 초록색, 틀린 명령어는 빨간색으로 표시합니다.
    
- **Git 통합:** 현재 작업 중인 폴더의 Git 브랜치 이름과 수정 사항 유무를 프롬프트에 바로 띄워줍니다.
    
- **Auto-suggestions:** 과거에 입력한 명령어를 바탕으로 흐릿하게 미리보기를 보여줍니다.
    

### ③ 재귀적 와일드카드 (`**`)

특정 폴더 아래의 모든 하위 폴더에서 파일을 찾을 때, Bash는 `find` 명령어를 써야 하지만 Zsh는 `ls **/*.js` 같은 짧은 문법으로 해결할 수 있습니다.

---

## 4. Bash를 써야 할 때

사용자 환경은 Zsh가 압도적이지만, **셸 스크립트 작성(Automation)** 시에는 여전히 Bash가 중요합니다.

- 대부분의 서버 환경(Ubuntu, CentOS 등) 기본 셸이 Bash이므로, 배포 스크립트는 호환성을 위해 Bash로 작성하는 것이 안전합니다.
